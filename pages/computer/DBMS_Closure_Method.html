DBMS • CHAPTER 26 • CLOSURE METHOD

Closure Method
for Candidate Keys

Closure method helps find all candidate keys in a relation using functional dependencies. It is essential before solving 2NF, 3NF, and BCNF questions.

Functional DependencyCandidate Key Prime AttributeSuper Key
1

Meaning of Closure

The closure of an attribute set tells us all attributes that can be determined from it using the given functional dependencies.

A+ means: start with A, then keep applying functional dependencies until no new attribute can be added.
2

Example 1: One Candidate Key

Relation R(A, B, C, D) with functional dependencies:

A → BB → CC → D
Closure Flow for A+
Start
A determines itself
A+ = A
A → B
Add B
A+ = AB
B → C
Add C through B
A+ = ABC
C → D
Add D
A+ = ABCD
Since A+ contains every attribute of R, A is a candidate key.
ClosureAttributes determinedCandidate key?Reason
A+A, B, C, DYesDetermines all attributes.
B+B, C, DNoA is missing.
C+C, DNoA and B are missing.
D+DNoOnly determines itself.
Candidate key = { A } | Prime attribute = A | Non-prime attributes = B, C, D
3

Candidate Key vs Super Key

If A alone is already a candidate key, then AB can determine all attributes but is not a candidate key. It is a super key, because candidate keys must be minimal.
4

Example 2: Circular Dependencies

Relation R(A, B, C, D) with functional dependencies:

A → BB → CC → DD → A
ClosureHow it expandsCandidate key?
A+A → B → C → DYes
B+B → C → D → AYes
C+C → D → A → BYes
D+D → A → B → CYes
Candidate keys = { A, B, C, D } | Prime attributes = A, B, C, D | Non-prime attributes = NULL
5

Example 3: Must-Have Attribute Shortcut

Relation R(A, B, C, D, E) with functional dependencies:

A → BBC → DE → CD → A
Right-side check
Right side attributes are B, D, C, A. Attribute E never appears on the right side.
Important conclusion
Every candidate key must contain E, because no dependency can produce E unless E is already present.
ClosureExpansionResult
E+E → CNot a candidate key
AE+AE → B, E → C, BC → DCandidate key
DE+D → A, then AE already determines allCandidate key
BE+E → C, then BC → D, then D → ACandidate key
CE+Only C and E are available; no new FD appliesNot a candidate key
Candidate keys = { AE, DE, BE } | Prime attributes = A, B, D, E | Non-prime attribute = C
6

Prime and Non-Prime Attributes

After finding all candidate keys, prime and non-prime attributes are easy to identify.

1Closure Basics
ClosureX+ is the set of all attributes determined by X.
Candidate key testIf X+ contains every attribute of the relation, X is a super key. If X is minimal, X is a candidate key.
Self ruleEvery attribute determines itself, so A is always included in A+.

3Exam Shortcuts
Not on RHSIf an attribute never appears on the right side of any FD, every candidate key must contain it.
MinimalityIf A is a candidate key, then AB is not a candidate key. AB is a super key.
Replacement trickIf part of a candidate key appears on RHS of another FD, try replacing it with that left-side determinant and verify by closure.
