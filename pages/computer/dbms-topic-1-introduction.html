Understanding the Database System

📌 A Database System = Database + DBMS. These are two distinct components — the database is the data store, and the DBMS is the software that manages it.

What is a Database? Simply put, a database is a collection of related data. The word "related" is key — data is not randomly clubbed together, it is logically connected.

For example, the IRCTC database contains information about trains, schedules, and logged-in passengers — but it does not contain unrelated data like passport records or income tax filings. Similarly, Facebook's database holds users and posts; Flipkart's holds products and orders.

Types of Data

Data is broadly categorized into three types. The DBMS course — and exams like GATE, UGC NET, IBPS SO — focus almost exclusively on Structured Data.

📊 Three Types of Data
✓ Structured Data
📋
Stored in a predefined format — rows and columns (a Relation / Table). Just like an Excel sheet. Example: IRCTC train search results.
★ DBMS / RDBMS Focus
≈ Semi-Structured
🗂️
Has some structure but not a strict table format. Examples: JSON, XML files. Tags define structure but no fixed schema.
(Not the primary focus of this course)
◎ Unstructured Data
🖼️
No predefined format. Includes photos, videos, web pages (mix of text + image + video). 90% of Earth's data is unstructured. Managed by Big Data / Hadoop.
All three types exist, but DBMS/RDBMS focuses on structured data stored in tables (relations).

What is a Relation? In database terminology, a Relation simply means a Table — a grid of rows and columns. Each row is a record (also called a tuple), and each column is an attribute (field).

What is a DBMS?

📖 DBMS (Database Management System) — A software system or platform that allows users to easily access data and perform operations on a database.

A DBMS sits between the user and the database. It receives the user's request, processes it, fetches or modifies the required data, and returns the result — all without the user needing to know where the data physically lives.

⚙️ Key Operations in DBMS (CRUD)
INSERT
Adding new data to the database.
New student enrolled → record inserted into Students table
DELETE
Removing existing data from the database.
Cancelled train ticket → record removed from Bookings table
UPDATE
Modifying existing data.
Student changes address → existing record updated in DB
These three form the core write operations. Together with SELECT (read), they make up the full CRUD model.

The Concept of RDBMS

RDBMS (Relational Database Management System) is a DBMS specifically designed to manage structured data stored in the form of Relations (Tables). Since the entire DBMS syllabus for B.Tech, GATE, and UGC NET revolves around structured data, RDBMS is effectively the main focus.

🔵 DBMS vs RDBMS — Relationship
  • Every RDBMS is a DBMS — but not every DBMS is an RDBMS.
  • RDBMS works only on structured data stored as tables (relations).
  • DBMS is the broader category that can handle unstructured and semi-structured data too.
  • For GATE, UGC NET, IBPS SO, and B.Tech syllabus — RDBMS is the focus since the curriculum is about structured/relational data.
  • All popular platforms (MySQL, Oracle, SQL Server, DB2) are RDBMS platforms.
RDBMS ⊂ DBMS. The inner circle (RDBMS) handles structured/tabular data; the outer ring handles other data types.

DBMS vs RDBMS
DBMS
Manages all types of data (structured, semi-structured, unstructured). Broad category. Example: any data management software.
RDBMS
Manages only structured data stored in tables (relations). A subset of DBMS. Example: MySQL, Oracle, SQL Server.
Relation:RDBMS ⊂ DBMS. Every RDBMS is a DBMS; not every DBMS is an RDBMS.
Exam relevance:GATE / UGC NET / IBPS SO syllabus = structured data only → so the course = RDBMS focus.
Operations:Same as DBMS — INSERT, UPDATE, DELETE, SELECT — but specifically on relational tables.
