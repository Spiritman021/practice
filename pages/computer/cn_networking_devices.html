Overview & Classification of Networking Devices

When data leaves your laptop or mobile, it does not travel directly to the destination. It passes through a series of intermediary devices — each performing a specific role at a specific layer of the OSI model. Some devices are pure hardware; others combine hardware with software (firmware/OS) to make intelligent decisions.

2

Layer 1 Devices — Pure Hardware (No Software)

These devices operate solely at the Physical Layer (OSI Layer 1). They deal with raw bits and electrical/light signals — no logical addressing, no MAC addresses, no decision-making. They are purely mechanical or electronic amplifiers/distributors.

🔗
Cable
L1 — Physical
TYPE Pure hardware — passive medium
PURPOSE Physical medium carrying electrical or light signals between devices
TYPES Twisted Pair (UTP/STP), Coaxial, Optical Fibre
NOTE No signal regeneration; signal weakens over distance (attenuation)
🔁
Repeater
L1 — Physical
TYPE Pure hardware — active (needs power)
PURPOSE Receives a weakened/distorted signal, regenerates (amplifies + cleans) it, and retransmits it to extend network range
PORTS 2-port device — one input, one output
EXAM TIP Works on bits only — no MAC/IP awareness. Cannot filter traffic.
💠
Hub
L1 — Physical
TYPE Pure hardware — multi-port repeater
PURPOSE Broadcasts every incoming frame to ALL connected ports regardless of destination
PORTS Multi-port (4, 8, 16, 24, 32 ports)
PROBLEM Creates a single collision domain — all devices share bandwidth; very inefficient
📡
Modem
L1 — Signal Conversion
FULL NAME Modulator / Demodulator
PURPOSE Converts digital signals → analog for transmission over telephone lines, and analog → digital at the receiving end
WHY? Computers work in digital (0s & 1s), but phone/cable lines carry analog signals
EXAMPLE Home DSL modem connecting to ISP via telephone wire

🔎 Hub vs Repeater: A Repeater has exactly 2 ports — it regenerates on a single link. A Hub is a multi-port repeater — it regenerates AND broadcasts to all ports simultaneously. Both are Layer 1 — neither knows anything about MAC addresses.
3

Layer 2 Devices — Hardware + Software (Data Link Layer)

Layer 2 devices operate at the Data Link Layer. They read the MAC addresses in frame headers to make forwarding decisions. Because software (firmware) is involved in processing these addresses, they are more expensive than pure L1 devices but far more efficient.

🌟
Bridge
L2 — Data Link
TYPE Hardware + Software
PURPOSE Connects two network segments; uses a MAC address table to decide whether to forward or filter a frame
PORTS Typically 2 ports (connects 2 segments)
KEY FEATURE Reduces collision domains by separating network segments; smarter than a hub
EXAM TIP Bridge = store-and-forward device; reads full MAC address before forwarding
⏻️
Switch
L2 — Data Link
TYPE Hardware + Software — multi-port bridge
PURPOSE Forwards frames only to the specific port where the destination MAC address is located (unicast); maintains a MAC Address Table (CAM Table)
PORTS Multi-port (24, 48 ports, etc.)
KEY FEATURE Each port = separate collision domain; full bandwidth per port; much more efficient than hub
EXAM TIP Switch is a smarter, multi-port bridge. All ports share one broadcast domain (unless VLANs are configured).
🔑 Bridge vs Switch: A Bridge has 2 ports and one MAC table entry per segment. A Switch is a multi-port bridge — it has individual MAC table entries per device. Switch = smarter, faster, more scalable. Both operate at Layer 2 using MAC addresses.
4

Layer 3 Device — Router (Network Layer)

The Router operates at the Network Layer (OSI Layer 3). It uses IP addresses (logical addresses) to route packets between different networks. Unlike bridges and switches which work within a single network using MAC addresses, routers interconnect different networks including the Internet.

🛠️
Router
L3 — Network Layer
TYPE Hardware + Software (has its own OS, e.g., Cisco IOS)
PURPOSE Routes packets between different networks using IP addresses; maintains a Routing Table to determine the best path
ADDRESSING Uses IP addresses (logical/Layer 3 addresses) — not MAC addresses
KEY FEATURES Connects multiple networks; separates broadcast domains; performs NAT, DHCP; can be used as a Gateway
EXAM TIP Router is the only standard device that breaks broadcast domains. Your home Wi-Fi router = Router + Switch + Modem combined.

Gateway, IDS & Firewall — Concepts & Security Devices

🚪
Gateway
All Layers (Concept)
TYPE Not a physical device — it is a concept/term meaning “door” or entry/exit point of a network
COMPONENTS Can be implemented using a Router, Firewall, or Proxy Server — whichever is placed at the network boundary
PURPOSE Connects two different networks that may use different protocols; performs protocol translation if needed
EXAM TIP Gateway is a term/role, not a hardware box. In your home network, your router IS the default gateway.
🔒
IDS
Security Device
FULL NAME Intrusion Detection System
PURPOSE Monitors network traffic and detects suspicious activity, known attacks, or policy violations. Raises alerts.
ACTION IDS only detects and alerts — it does NOT block traffic by itself (that is IPS — Intrusion Prevention System)
EXAM TIP IDS = detection only (passive). IPS = detection + prevention (active). A key exam distinction.
🔥
Firewall
Security Device (L3–L7)
TYPE Hardware and/or Software security system
PURPOSE Monitors and controls incoming and outgoing network traffic based on predefined security rules; blocks unauthorised access
HOW? Uses packet filtering (L3), stateful inspection (L4), or application-layer (L7) filtering
EXAM TIP Firewall can act as a Gateway. It is placed at the boundary between trusted (internal) and untrusted (Internet) networks.
⚖️
IDS vs Firewall
Key Distinction
IDS Sits inside the network; monitors and alerts on suspicious traffic; passive observer
Firewall Sits at the boundary; actively blocks packets matching deny rules; enforcement point
BOTH Used together for comprehensive security: Firewall blocks known bad traffic; IDS catches anomalies that slip through
🚨 Exam Alert — Gateway is NOT a device: Many students confuse Gateway with a physical hardware box. Gateway is a concept — it refers to any device (router, firewall, proxy) that acts as the entry/exit point between two networks. Your exam question may say “which device acts as a gateway?” — the answer is: a Router (most commonly) or Firewall.

LAYER 1 DEVICES (PURE HARDWARE)
🔗 Cable
  • OSI Layer 1 — Physical
  • Passive medium — no active components
  • Types: Twisted Pair, Coaxial, Optical Fibre
  • Signal degrades over distance (attenuation)
🔁 Repeater
  • OSI Layer 1 — Physical
  • Pure hardware — 2 ports only
  • Regenerates signal (not just amplifies noise)
  • No MAC/IP awareness; cannot filter traffic
💠 Hub
  • OSI Layer 1 — Physical
  • Multi-port repeater — pure hardware
  • Broadcasts to ALL ports (no filtering)
  • Creates single collision domain; wasteful bandwidth
📡 Modem
  • OSI Layer 1 — Physical
  • Modulator + Demodulator
  • TX: Digital → Analog (MODulate)
  • RX: Analog → Digital (DEModulate)
LAYER 2 DEVICES (HARDWARE + SOFTWARE)
🌟 Bridge
  • OSI Layer 2 — Data Link
  • Hardware + Software — 2 ports
  • Uses MAC address table to forward/filter
  • Creates 2 separate collision domains
  • Store-and-forward device
⏻️ Switch
  • OSI Layer 2 — Data Link
  • Hardware + Software — multi-port bridge
  • Uses CAM/MAC table — per-port learning
  • Forwards to specific destination port only
  • Each port = separate collision domain
LAYER 3 DEVICE (HARDWARE + SOFTWARE)
🛠️ Router
  • OSI Layer 3 — Network
  • Hardware + Software (has its own OS)
  • Uses IP addresses and Routing Table
  • Connects different networks
  • Each port = separate broadcast domain
  • Most expensive of Hub/Bridge/Switch/Router set
Hub vs Bridge vs Switch vs Router
  • Hub → L1, broadcast, no address
  • Bridge → L2, 2-port, MAC filter
  • Switch → L2, multi-port bridge, MAC forward
  • Router → L3, IP routing, breaks broadcast domain
GATEWAY, IDS & FIREWALL
🚪 Gateway
  • NOT a physical device — it is a concept
  • Means “entry/exit door” of a network
  • Implemented by: Router, Firewall, or Proxy Server
  • Can perform protocol translation between networks
  • Your home router is the Default Gateway
🔒 IDS (Intrusion Detection System)
  • Security device — Hardware + Software
  • Detects & alerts on suspicious traffic
  • Does NOT block traffic (passive)
  • IDS = detect only; IPS = detect + prevent
  • Sits inside the network, monitors all traffic
🔥 Firewall
  • Security device — Hardware and/or Software
  • Operates at L3 (IP filter), L4 (port/state), L7 (app)
  • Actively blocks unauthorised traffic
  • Sits at network boundary (can act as Gateway)
  • Uses rules/policies to allow/deny packets

✅ Last-Minute Exam Checklist

  • Cable, Repeater, Hub = pure hardware (no software); all operate at L1 Physical
  • Bridge, Switch, Router = hardware + software; smarter and more expensive
  • Bridge = 2 ports, MAC-based; Switch = multi-port bridge (also MAC-based)
  • Router = only device that separates broadcast domains; uses IP addresses
  • Gateway = NOT a device — it is a concept (role played by router/firewall/proxy)
  • IDS = detects only (passive); Firewall = blocks (active enforcement)
  • Modem = MODulate (digital→analog) + DEModulate (analog→digital)
  • Hub → 1 collision domain; Switch → 1 collision domain per port; Router → 1 broadcast domain per port
