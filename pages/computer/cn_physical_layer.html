Position & Role in the OSI Model

The Physical Layer is Layer 1 — the bottommost layer of the OSI model. It holds a unique dual position: on the sender side it is the very last layer to process outgoing data (receives bits from Data Link above it, then pushes signals onto the wire). On the receiver side it is the very first layer to handle anything that arrives.

Every layer above it deals with virtual, logical, or software-based constructs — packets, frames, sessions, data. The Physical Layer is the only layer that deals exclusively with tangible hardware: real cables, real connectors, real signals. No software abstraction here.

Transmission Media, Cables & Connectors

The Physical Layer decides how signals are sent across either guided (wired) or unguided (wireless) media. The type of cable directly determines what type of signal is used — copper carries electrical signals, optical fibre carries light pulses, and wireless uses radio or infrared waves.

🔌
Cables (Guided Media)
Twisted Pair
Most common; copper wires twisted to reduce interference. UTP (Unshielded) & STP (Shielded). Carries electrical signals.
Coaxial Cable
Central copper conductor + insulation + metal shield. Higher bandwidth than twisted pair. Also electrical signals.
Optical Fibre
Glass/plastic strands that carry light pulses. Highest bandwidth, immune to EMI, very long distance.
📡
Wireless (Unguided Media) & Connectors
Infrared
Short-range wireless (e.g., TV remote). Line-of-sight required. Cannot pass through walls.
Radio / Wi-Fi
Longer range wireless. Omnidirectional. Used in Wi-Fi, Bluetooth, cellular networks.
Connectors
Physical hardware to join cables: UTP connectors (RJ-45), BNC connectors (coaxial), MGRT connectors.
4

Physical Topologies

The Physical Layer dictates how devices are physically wired together. The layout pattern is called the topology. The connection type — whether a link is dedicated or shared — also falls here.

Star Topology
All devices connect to a central hub/switch. Failure of one node doesn't affect others.
Mesh Topology
Every device has a dedicated point-to-point link to every other. Highly redundant.
Bus Topology
All devices share a single backbone cable (multipoint). Failure of backbone = entire network down.
🔗 Connection Types: Point-to-Point = dedicated link, only two devices (used in Mesh). Multipoint = shared medium, many devices on one cable (used in Bus).
5

Hardware Devices at the Physical Layer

Only two main networking hardware devices operate at the Physical Layer. Both work with raw signals — they have no knowledge of MAC addresses, IP addresses, or any higher-layer concepts.

🔁
Repeater
Signals weaken over long distances due to attenuation (loss of signal energy caused by noise and resistance in the medium). A repeater regenerates the signal back to its original strength so it can travel further. It does not interpret any data — just boosts the signal.

Layer 1 only · No filtering · Amplifies noise too (analogue) or regenerates clean (digital)
🔀
Hub
A hub creates a multipoint connection — it connects multiple machines to the same shared medium. When one device sends data, the hub broadcasts it to every connected device (not just the intended one). This is why hubs are considered "dumb" devices and have been largely replaced by switches (which operate at Layer 2).

Layer 1 · Broadcasts to all ports · No intelligence · Replaced by L2 switches

Multiplexing & Demultiplexing

If every machine needed its own dedicated physical cable/channel, networks would be prohibitively expensive. Multiplexing solves this by combining n signals from different sources into a single shared channel. At the receiver end, demultiplexing splits that combined signal back into the original n distinct signals.

Encoding

Encoding is the process of converting data into an appropriate signal format for transmission. Not all data is the same type: computers generate digital data (discrete — distinct 0s and 1s), while voice or radio generate analog data (continuous wave forms). Depending on the source data type and the medium, four encoding combinations are used:

🔡 Four Types of Encoding
Encoding TypeInputOutputExample
Digital → DigitalDigital data (bits)Digital signalManchester encoding, NRZ, RZ
Analog → AnalogAnalog data (voice/radio)Analog signalAM, FM, PM (radio modulation)
Analog → DigitalAnalog data (voice)Digital signalPCM (Pulse Code Modulation) — telephone digitisation
Digital → AnalogDigital data (bits)Analog signalASK, FSK, PSK — used in modems
Digital data = discrete (0/1). Analog data = continuous waveform. Choice of encoding depends on source type and transmission medium.
💡 For UGC NET: Manchester Encoding and Differential Manchester Encoding are the most commonly asked Digital→Digital schemes. Know how they represent 0 and 1 using signal transitions.

Signal Characteristics

The Physical Layer manages the fundamental physical properties of the signals it transmits. Three core properties define every signal:

〰️
Frequency
Number of complete wave cycles per second. Higher frequency = more data capacity.
f = 1/T  ·  Unit: Hz
📶
Amplitude
Height of the wave (peak signal strength). Determines the power/loudness of the signal.
Unit: Volts / dB
🌊
Wavelength
Physical distance between two successive wave crests. Inversely related to frequency.
λ = v/f  ·  Unit: metres
📐 Relationship: Wavelength (λ) = speed of propagation (v) ÷ frequency (f). As frequency increases, wavelength decreases. Higher frequency signals carry more data but attenuate faster over distance.

Manchester Encoding (UGC NET)
  • Type: Digital → Digital
  • Bit 0 = Low→High transition mid-bit
  • Bit 1 = High→Low transition mid-bit
  • Self-clocking — transition always present
  • Used in Ethernet (IEEE 802.3)

🔥 High-Yield Facts
  • Physical Layer = Layer 1, deals with hardware only
  • PDU at Physical Layer = Bits
  • Repeater = signal regeneration (attenuation fix)
  • Hub = multipoint, broadcasts to all ports
  • Manchester encoding used in IEEE 802.3 Ethernet
📌 Common Exam Traps
  • Hub = Layer 1 · Switch = Layer 2 · Router = Layer 3
  • Optical fibre = light signals, NOT electrical
  • WDM is for optical fibre, not copper cables
  • Simplex = NO reply possible (not half-duplex)
  • Attenuation ≠ noise — it's signal energy loss
💡 Quick Mapping
  • Mesh topology → Point-to-point links
  • Bus topology → Multipoint shared link
  • FDM → different frequencies on same wire
  • TDM → different time slots on same wire
  • WDM → different wavelengths on fibre
