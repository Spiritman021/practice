2

1NF to BCNF Summary

1NF
Rule: no multivalued attribute. Every cell should be atomic. Example: do not store C, C++ in one Course cell.
2NF
Rule: table must be in 1NF and no partial dependency. If AB is the key, only A or only B should not determine a non-prime attribute.
3NF
Rule: table must be in 2NF and no transitive dependency. Memory line: no non-prime attribute should determine a non-prime attribute.
BCNF
Rule: table must be in 3NF and left side of every FD must be a candidate key or super key.

5NF: Lossless Join

5NF says the relation should be in 4NF and decomposition should be lossless. When tables are joined again, no spurious tuples should appear.

Memory line: while decomposing vertically, the common attribute should be a candidate key of one decomposed table or of the original table, so the join remains lossless.
