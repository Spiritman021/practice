What is Packet Switching?

Packet switching is a communication technique where data is divided into small units called packets before being transmitted across the network. Each packet travels independently through the network and may take a different path to reach the destination.

This is the technology that powers modern computer networks — the internet, email, WhatsApp, web browsing, and all IP-based communication. Unlike circuit switching (which works at the Physical Layer and creates a dedicated path), packet switching uses the full OSI layer stack — primarily the Data Link Layer and Network Layer.

📨 Key Idea: In circuit switching, a dedicated path is reserved before sending. In packet switching, no reservation is needed — the data is broken into packets, and each packet finds its own way through the network using routing tables. Like walking into a restaurant and finding a free table, vs booking in advance.

OSI Layer & Two Types of Packet Switching

Packet switching operates in two forms, and they work at different OSI layers. This is a frequently tested point in GATE and UGC NET.

🌐 Datagram Service

📌 Works at Network Layer (Layer 3)
📌 Connectionless — no setup before sending
📌 Each packet is routed independently
📌 Packets may take different paths
📌 Packets may arrive out of order
📌 Most common — used in the Internet (IP)
📌 More important for exams (in syllabus)

🔂 Virtual Circuit

📌 Works at Data Link Layer (Layer 2)
📌 Connection-oriented — a logical path is set up first
📌 All packets follow the same path
📌 Packets arrive in order
📌 Combines benefits of circuit & packet switching
📌 Used in ATM, Frame Relay networks
📌 Less common in modern networks

Store-and-Forward — The Core Mechanism

The most fundamental concept of packet switching is store-and-forward. When a packet arrives at an intermediate switch or router, it is stored in the buffer, then processed, and then forwarded towards the destination. This is completely different from circuit switching, where data flows straight through without any intermediate storage.

📤
Step 1: A sends P1 to Switch S1
Device A divides its data into packets (P1, P2, P3, P4) and sends P1 to the first switch S1. No connection setup is needed — A just sends the packet directly.
💾
Step 2: S1 stores P1 in its buffer
S1 receives the packet and stores it in its buffer (memory). The buffer is necessary to hold the packet while the switch decides how to forward it.
📏
Step 3: S1 checks routing table
S1 examines the packet's destination address (in the header) and looks up its routing table to determine the best path to forward the packet to the destination B.
📤
Step 4: S1 forwards P1 to next switch S2
S1 transmits (forwards) P1 to the next switch S2 on the path towards B. S2 repeats the same store-process-forward cycle. This continues until P1 reaches B.
📨 Packet Store-and-Forward Through the Network
Each switch stores the incoming packet in its buffer, looks up the routing table, then transmits it to the next hop. This is store-and-forward. With n switches, the packet gets transmitted (n+1) times total.
📌 Why store? Because the switch needs to read the packet's destination header and decide which output port to send it through. This requires buffering the entire packet first. In circuit switching, no such decision is needed at intermediate nodes — the path is already fixed.

High Efficiency — No Wasted Bandwidth

The biggest advantage of packet switching over circuit switching is high efficiency. Because there is no dedicated reserved path, the network bandwidth is shared dynamically among all users.

🍽 Restaurant Analogy — Circuit Switching vs Packet Switching

📞 Circuit Switching = Table Reservation

📌 You book a table in advance for 5:00 PM
📌 You arrive at 7:00 PM (2 hours late)
📌 Table was reserved and empty from 5–7 PM
📌 No one else could use it during that time
📌 Wasted resource → low efficiency

📨 Packet Switching = Walk-in

📌 No reservation — you walk in and find a free table
📌 Table is only used when someone is actually sitting
📌 No idle reservation — every table is used to the fullest
📌 More people can be served overall
📌 Maximum resource usage → high efficiency
In packet switching, bandwidth is only consumed when data is actually being sent. No reserved-but-idle waste. This is why packet switching dominates modern networks.
✅ Why efficiency is high: In circuit switching, even during silence (no data being sent), the reserved bandwidth/channel stays locked. In packet switching, the moment you stop sending packets, the network bandwidth is freed for others to use. This dynamic sharing makes efficiency much higher.
5

More Delay — The Store-and-Forward Cost

The flip side of store-and-forward is higher delay. At every intermediate switch, the packet must be:

💾 Stored in buffer
→
📏 Header read
→
📋 Routing table checked
→
📤 Forwarded out

Each of these steps takes time. With more intermediate switches, the total delay increases. This is the trade-off: packet switching gains efficiency but pays with higher delay per packet.

⏳ Delay sources in packet switching:
• Transmission delay — time to push packet bits onto the link (at each node)
• Propagation delay — physical travel time through the medium
• Queuing delay — waiting in the buffer at each switch
• Processing delay — reading header, looking up routing table

Circuit switching only has transmission + propagation delay (no queuing or processing at nodes).
6

Pipelining — Parallelism to Boost Efficiency

Packet switching uses pipelining to dramatically increase efficiency and reduce total transmission time. Instead of waiting for a packet to reach its destination before sending the next one, the sender starts sending the next packet as soon as it finishes transmitting the previous one.

When S1 is forwarding P1 to S2, A simultaneously sends P2 to S1. When S1 is forwarding P2 and S2 is forwarding P1, A is sending P3 — all in parallel. This overlapping of transmissions is pipelining.

🚀 Pipelining — Parallel Packet Transmission (A sends 4 packets through S1 → S2 → B)
Who / What Time T1 Time T2 Time T3 Time T4 Time T5 Time T6
A → S1 (sends) P1 P2 P3 P4 — —
S1 → S2 (forwards) — P1 P2 P3 P4 —
S2 → B (forwards) — — P1 P2 P3 P4 ✓
📌 At T2: A sends P2 to S1, simultaneously S1 forwards P1 to S2. At T3: A sends P3, S1 forwards P2, S2 forwards P1 — all parallel! Without pipelining this would take 6 time slots; with pipelining it takes 6 but the work is done in parallel, meaning more packets get through per unit time.
Without pipelining: A waits for P1 to reach B before sending P2 → wastes time. With pipelining: A starts sending P2 as soon as it finishes P1 → links are always busy → higher throughput.
🚀 Why pipelining matters: Without pipelining, a link is only active for one packet at a time and then sits idle waiting for acknowledgement. With pipelining, multiple packets fill the links simultaneously — this maximises bandwidth utilisation and is why packet switching is so efficient for bursty computer traffic.

Total Time Formula — GATE Numericals

In packet switching, no setup or teardown time is needed (unlike circuit switching). However, because each intermediate switch must transmit the packet forward, the transmission time is multiplied by the number of links (not just one sender).

🎯 Total Time in Packet Switching (No Pipelining)
Ttotal = n × Ttx + Tprop
where n = total number of links (or switches + 1) the packet traverses
Ttx = Packet Size (bits) ÷ Bandwidth (bps)  |  Tprop = Distance ÷ Speed of Signal
📌 Why n × Ttx? In circuit switching, only sender A transmits and the data flows straight through. In packet switching, every switch must re-transmit the packet. So if there are 3 switches (S1, S2, S3) between A and B, the packet is transmitted by: A + S1 + S2 + S3 = 4 times (n = 4 links). Each transmission takes Ttx time. Hence total transmission delay = n × Ttx.
⚖ Formula Comparison — Circuit vs Packet Switching
📞 Circuit Switching
T = Tsetup + Ttx + Tprop + Tteardown
Setup and teardown phases are needed.
Only sender transmits once (Ttx × 1).
No store-and-forward at intermediate nodes.
📨 Packet Switching
T = n × Ttx + Tprop
No setup or teardown time needed.
Every switch re-transmits → n × Ttx.
Store-and-forward at each hop adds delay.
✎ Worked Example — Total Time in Packet Switching
Problem: A sends a 1000-bit packet to B through 3 switches (4 links). Bandwidth = 1 Mbps. Distance = 2000 km. Signal speed = 2×108 m/s.
GIVEN
Packet Size = 1000 bits
Bandwidth = 1 Mbps = 106 bps
Switches = 3 ⇒ n (links) = 4
Distance = 2000 km = 2×106 m
Signal Speed = 2×108 m/s
SOLUTION
Ttx = 1000 ÷ 106 = 0.001 s
Tprop = 2×106 ÷ 2×108 = 0.01 s
Ttotal = 4 × 0.001 + 0.01
= 0.004 + 0.01 = 0.014 s
📌 Note: This is the time WITHOUT pipelining (one packet). With pipelining and multiple packets, the total time changes — see the pipelining formula covered in numericals.
