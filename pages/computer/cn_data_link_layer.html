DATA LINK LAYER

Layer 2 is the hop-to-hop manager of Computer Networks. It takes packets from the Network Layer, converts them into frames, controls local link flow/errors, manages channel access, and uses MAC addresses inside a LAN.

GATE UGC NET B.Tech / BCA Exam Notes
1

Position in the OSI Model

The Data Link Layer is the second layer from the bottom in the OSI model. It sits between the Network Layer and the Physical Layer. The Network Layer gives it packets, and the Data Link Layer prepares those packets as frames for transmission over the Physical Layer.

🔗 Layer 2 Context
Exam line: Network Layer thinks end-to-end; Data Link Layer thinks next hop.
2

Hop-to-Hop / Node-to-Node Delivery

The most important responsibility of the Data Link Layer is hop-to-hop delivery, also called node-to-node delivery. If host A4 wants to send data to host B1 in another network, Layer 2 does not directly handle A4 to B1 as one complete journey. It handles each smaller jump: A4 to R1, R1 to R2, R2 to the next network, and so on.

📡 Hop-by-Hop View
Each arrow is a Data Link Layer responsibility. The full source-to-destination path is Network Layer responsibility.
Key idea: Within one LAN, communication can be handled using the Data Link Layer and MAC addresses. Across different networks, routing and IP addressing from the Network Layer are required.
3

Flow Control at Every Hop

Flow control means controlling the speed at which frames are sent. If A4 sends frames too fast and R1 cannot process or store them, R1's buffer may overflow and frames may be lost. Data Link Layer flow control checks the capacity of the next node, not only the final destination.

Data Link Layer Flow Control

Controls flow between two adjacent nodes: sender to next hop, hop to next hop, and so on.

Hop-to-hopLayer 2

Transport Layer Flow Control

Controls flow between the original source and the final destination machine.

End-to-endLayer 4
Flow-control protocols: Stop-and-Wait, Go-Back-N, and Selective Repeat are the important methods used to manage how many frames can be sent before waiting for confirmation.
4

Error Control: Detect Early, Recover Faster

While frames travel through a medium, bits may change. A 0 may become 1, a 1 may become 0, or multiple bits may change together as a burst error. The Data Link Layer performs error control hop by hop so that an error can be detected early at an intermediate node instead of waiting until the final receiver.

LayerError Check MethodScopeWhy Important?
Data Link LayerCRC, parity, hamming codeHop-to-hopDetects errors earlier and avoids wasting full-path transmission time.
Transport LayerChecksumSource-to-destinationConfirms final end-to-end correctness after the message reaches the destination.
Exam trap: Error control exists in both Data Link and Transport layers, but their scope is different. Layer 2 checks link by link; Layer 4 checks end to end.
5

Access Control on a Shared Channel

In a shared medium such as a bus topology or common channel, multiple devices may try to send data at the same time. If two devices transmit simultaneously, their frames can collide and become useless. Access control decides who gets the channel and when.

⚠ Shared Channel Collision Picture
CSMA/CD, ALOHA, Token Ring and Token Bus are ways to control shared-medium access.

Collision-Based Methods

CSMA/CD and ALOHA-style methods are used where devices may compete for access and collisions must be handled.

CSMA/CDALOHASlotted ALOHA

Token-Based Methods

Token Ring and Token Bus allow transmission only when a device has the token, reducing random collisions.

Token RingToken Bus
Simple meaning: Access control is like asking, "Who can use the channel now?" Without it, two messages may collide on the same wire.
6

MAC Address and Framing

The Data Link Layer uses the physical address, also called the MAC address. A MAC address is generally 48 bits and is used for communication inside a local network. When a packet comes from the Network Layer, the Data Link Layer converts it into a frame by adding a header and trailer.

📦 Frame Structure
Data Link Layer PDU = Frame. Network Layer PDU = Packet.
Train analogy: Data is like coaches, the header is like the engine, and the trailer is like the last guard section. The complete unit travels properly as a frame from one hop to the next.
ONE-PAGE REVISION
Layer Position
  • OSI Layer: 2
  • Between Network Layer and Physical Layer
  • Input from L3: Packet
  • Output to L1: Frame
Main Responsibility
  • Data Link Layer = hop-to-hop delivery
  • Network Layer = source-to-destination delivery
  • Transport Layer = process-to-process / end-to-end reliability controls
  • Inside one LAN, MAC-based communication is possible
Flow Control
  • Controls sending speed on each link
  • Prevents next-node buffer overflow
  • Important protocols: Stop-and-Wait, Go-Back-N, Selective Repeat
  • Scope: node to node
Error Control
  • Detects bit errors and burst errors
  • Layer 2 commonly uses CRC
  • Transport Layer commonly uses Checksum
  • Early error detection improves performance
Access Control
  • Controls who can transmit on shared media
  • Prevents or handles collision
  • Methods: CSMA/CD, ALOHA, Token Ring, Token Bus
  • Common in LAN/shared-channel discussions
MAC & Frame
  • MAC address = 48-bit physical address
  • Used mainly inside local network communication
  • Frame = Header + Data + Trailer
  • Header/trailer help proper delivery and checking

✅ Last-Minute Summary

  • Data Link Layer is Layer 2 and works between Network and Physical layers.
  • Its golden phrase is hop-to-hop delivery, not final source-to-destination delivery.
  • Flow control at L2 checks the capacity of the next hop; Transport Layer checks the final destination.
  • Error control at L2 detects errors earlier using methods like CRC.
  • Access control decides who can use the shared channel and prevents collision.
  • MAC address is a 48-bit physical address; Data Link Layer PDU is a frame.
