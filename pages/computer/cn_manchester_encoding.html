Where Does Encoding Fit? — Physical Layer Context

The Physical Layer (OSI Layer 1) is responsible for transmitting raw data from one node to another. Data from upper layers arrives as a stream of 1s and 0s. However, the Physical Layer can encode those bits into a different signal form before sending them on the wire.

Manchester & Differential Manchester are Digital-to-Digital encoding schemes — purely signal-level transitions, no modulation.
🎯 Why not just send raw 1s and 0s? Raw NRZ signals suffer from synchronisation problems — long runs of identical bits produce no transitions, making it hard for the receiver to track the clock. Manchester encoding guarantees a transition in every bit period, solving this entirely.
2

Manchester Encoding

In Manchester encoding each bit is represented by a signal transition at the mid-point of the bit period. There is never a flat, ambiguous period — the signal always changes direction at the centre of every bit cell, guaranteeing clock synchronisation.

⚠️ Two Conventions Exist! Exams sometimes specify which to use. If none is mentioned, IEEE 802.3 is the safe default.

Dr. Thomas Convention

1 High→Low transition (↓) at mid-bit — Z-shape
0 Low→High transition (↑) at mid-bit — reverse-Z shape
Boundary edge only appears when adjacent bits differ in value.

IEEE 802.3 Convention — Default

1 Low→High transition (↑) at mid-bit
0 High→Low transition (↓) at mid-bit
IEEE is the exact reverse of Dr. Thomas.
📈 Manchester Encoding — Example 1: Data = 1 0 1 0 0 (Dr. Thomas convention)
DATA BITS:  1  ·  0  ·  1  ·  0  ·  0
─── Waveform ┌┐ Mid-bit marker ↓↑ Mid-bit transition (encodes data)
Dr. Thomas: 1 = H→L (↓) at mid, 0 = L→H (↑) at mid. Bits 4&5 (0,0): boundary transition added automatically at the border.
📈 Manchester Encoding — Example 2: Data = 0 0 1 1 1 (Dr. Thomas convention)
DATA BITS:  0  ·  0  ·  1  ·  1  ·  1
When two same-value bits appear consecutively, a boundary transition is inserted automatically so the next bit can start at the correct level.
🖉 Step-by-Step: Drawing Manchester Waveforms
  1. 1Draw a time axis divided into equal bit-period cells.
  2. 2For each bit, determine the mid-bit transition direction using your convention (Dr. Thomas or IEEE 802.3).
  3. 3Set the signal level at the start of the bit period: for a 1 (Dr. Thomas) begin HIGH; for a 0 begin LOW.
  4. 4At the mid-point flip the level, then stay flat to the end of the period.
  5. 5Move to the next bit. If it requires a different starting level from where you ended, add a boundary transition (vertical line at the bit border).
  6. 6Connect all segments. Never leave a gap.
3

Differential Manchester Encoding

Differential Manchester takes a fundamentally different approach. A mid-bit transition is always present in every bit period — but it is used only for clock synchronisation. The actual bit value is determined by whether a transition occurs at the boundary between adjacent bits.

The Two Golden Rules

0 Edge at the boundary — a transition occurs at the START of the bit period (before the mandatory mid-bit transition).
1 No edge at the boundary — signal continues from where it left off. No extra transition at the start. (Mid-bit transition still occurs.)
MID A mid-bit transition is always present in every bit period — this is the sync pulse.
START Your starting level (H or L) is a free choice. The first mid-transition direction follows logically.

Memory Tricks 🧠

📌 0 = Edge — Zero “has a stroke”, needs an extra edge at the boundary.
📌 1 = Continue — One goes straight, no extra edge.
📌 A 0 gives 2 transitions per bit (boundary + mid). A 1 gives exactly 1 transition (mid only).
📌 Once you fix the starting level, all subsequent transitions are forced by logic — you have no choice.
📈 Differential Manchester — Example 1: Data = 1 0 1 0 0 (starting HIGH)
DATA BITS:  1  ·  0  ·  1  ·  0  ·  0   (starting HIGH)
─── Waveform ┌┐ Mid-bit sync (always present) EDGE = bit 0 NO EDGE = bit 1
Bit 1 (1): no boundary edge — signal continues. Bit 2 (0): boundary edge at start. Bit 3 (1): no boundary edge. Bits 4&5 (0,0): both have boundary edges. Mid-bit sync transition appears in every bit period.
📈 Differential Manchester — Example 2: Data = 0 1 1 1 0 (starting LOW)
DATA BITS:  0  ·  1  ·  1  ·  1  ·  0   (starting LOW)
Three consecutive 1s (bits 2–4): no boundary edges — waveform simply alternates direction at each mid-bit sync. Bits 1 and 5 (0s) have clear boundary edges.
🖉 Step-by-Step: Drawing Differential Manchester Waveforms
  1. 1Choose any starting level (H or L) — this is your free choice.
  2. 2For each bit check: 0 → add a boundary transition first; 1 → no boundary transition.
  3. 3After handling the boundary (or not), travel flat to the mid-point of the bit period.
  4. 4At the mid-point always add a transition (sync pulse). Its direction is forced by your current level.
  5. 5Travel flat to the end of the bit period. Repeat for the next bit from step 2.
  6. 6Verification: count transitions — each 1 has exactly 1, each 0 has exactly 2.

🔑 Key advantage of Differential Manchester: Because bit values are encoded as changes relative to the previous state (not absolute levels), swapping the two wires does not corrupt the data. This polarity-inversion immunity is called differential signalling.

Draw the waveform for a given bit string Check which convention is specified. If none → use IEEE 802.3. Don't forget boundary transitions between same-value adjacent bits in Manchester.
Decode a given waveform back to bits Manchester: look at mid-transition direction. Differential: look at whether a boundary transition exists — edge = 0, no edge = 1.
Count signal transitions Differential: 1s always have 1 transition, 0s always have 2. Manchester: count varies with adjacent-bit patterns.
Baud rate / bandwidth Both schemes: minimum baud rate = 2× bit rate. Signal rate = 2 × data rate in worst case.
Compare with NRZ / RZ NRZ has no guaranteed transitions → clock sync problem. Manchester and Differential Manchester fix this with guaranteed mid-bit transitions.
Which uses relative encoding? Differential Manchester uses relative (differential) encoding → polarity-safe. Manchester uses absolute voltage levels.
🚨 Most common mistake: In Differential Manchester, students often forget the mid-bit sync transition when drawing a long run of 1s. Even a stream of all 1s still has a mid-bit transition in every single bit period!
💡 IEEE 802.3 vs Dr. Thomas memory trick: IEEE is the exact vertical flip of Dr. Thomas. In IEEE: 1 = L→H at mid, 0 = H→L at mid. If you know one, you know the other.

MANCHESTER RULES
Dr. Thomas Convention
  • 1 = H→L transition at mid-bit (↓ Z-shape)
  • 0 = L→H transition at mid-bit (↑ reverse-Z)
  • Mid-transition carries the actual bit value
  • Boundary edge only appears between different adjacent bits
IEEE 802.3 Convention (Exam Default)
  • 1 = L→H transition at mid-bit (↑)
  • 0 = H→L transition at mid-bit (↓)
  • Exact reverse of Dr. Thomas
  • Used in Ethernet — default when unspecified
Drawing Tips
  • Guaranteed transition at every mid-bit
  • Same adjacent bits → extra boundary transition added automatically
  • Never leave a gap in the waveform
  • Baud rate = 2 × bit rate
DIFFERENTIAL MANCHESTER RULES
The Two Core Rules
  • 0 = Edge at boundary (transition at START of bit)
  • 1 = No edge at boundary (continue signal)
  • Mid-bit sync transition: always present
  • Memory: 0 = Edge, 1 = Continue
Transitions Per Bit
  • 1 → exactly 1 transition (mid only)
  • 0 → exactly 2 transitions (boundary + mid)
  • Waveform is polarity-inversion immune
  • Starting level (H or L) is a free choice
Common Pitfalls ⚠️
  • Forgetting mid sync transition for a run of 1s
  • Using wrong convention (always check the question)
  • Confusing boundary vs mid-bit transitions when decoding
  • Thinking Diff. Manchester encodes using mid-transition direction — it does not

QUICK DECODE SHORTCUTS
Decode Manchester Waveform
  • Find the mid-point of each bit period
  • ↓ at mid (H→L) → 1 (Dr. Thomas) or 0 (IEEE)
  • ↑ at mid (L→H) → 0 (Dr. Thomas) or 1 (IEEE)
  • Boundary transitions: ignore — they carry no bit information
Decode Differential Manchester
  • Find each bit boundary
  • Transition at boundary → 0
  • No transition at boundary → 1
  • Mid-transitions: ignore — sync only, no data meaning
Exam Checklist ✅
  • Identify which encoding and convention is used
  • Draw bit-period grid before starting waveform
  • Mark mid-bit points with dotted lines
  • Verify: every bit has ≥1 mid-bit transition

✅ Last-Minute Summary

  • Manchester — 1 = Z-shape (Dr. Thomas), 1 = reverse-Z (IEEE); mid-transition encodes the bit
  • Differential Manchester — 0 = edge at boundary, 1 = no boundary edge; mid-transition is sync only
  • Every bit in both schemes has exactly 1 guaranteed mid-bit transition
  • Default convention = IEEE 802.3 unless the question says otherwise
  • Baud rate = 2 × bit rate for both Manchester and Differential Manchester
  • Differential Manchester is polarity-inversion immune; standard Manchester is not
  • Both schemes solve the NRZ clock-sync problem via guaranteed transitions
