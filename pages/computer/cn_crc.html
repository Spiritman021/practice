CYCLIC REDUNDANCY CHECK

Detailed teacher-style notes on CRC: why it is powerful, how to convert generator polynomial to binary, append zeroes, perform modulo-2 XOR division, pick the CRC remainder, send the codeword, and verify using zero remainder at receiver.

1

What Is CRC?

CRC stands for Cyclic Redundancy Check. It is an error detection method used heavily in real communication systems. It sends redundant bits with the message so that the receiver can check whether any bit changed on the path.

The teacher calls CRC powerful because it can detect single-bit errors, odd-numbered errors, many double-bit errors, and burst errors up to the generator polynomial degree as discussed in the lecture. In real-life applications, CRC is one of the most widely used error detection methods.

🛡 CRC Adds Redundant Bits
CRC bits are redundant bits calculated from modulo-2 division.
Power point: If the generator polynomial has degree 4, then CRC can detect burst error of length up to 4 in the lecture's explanation.
2

Polynomial to Binary Divisor

CRC questions may give the divisor directly in binary form or as a polynomial. The dividend/message may also be given in binary form or polynomial form. If polynomial form is given, convert it into binary using coefficients from highest degree down to zero degree.

For example, x^4 + x^3 + 1 becomes 11001. Why? Coefficient of x^4 is 1, x^3 is 1, x^2 is missing so 0, x^1 is missing so 0, and x^0 is 1.

🔢 Generator Polynomial Conversion
Missing powers get coefficient 0.
3

How Many Zeroes to Append?

If the generator polynomial has maximum degree r, append r zeroes to the message before division. If the divisor is given in binary, append divisor length - 1 zeroes.

In the lecture example, maximum polynomial degree is 4, so append four zeroes. If the divisor is directly given as 11001, it has 5 bits, so append 5 - 1 = 4 zeroes. Do not append 5 zeroes.

4

Modulo-2 Division Uses XOR

CRC division is binary division using XOR, not normal subtraction. Same bits give 0, different bits give 1. Quotient is not important; only the remainder is used.

The teacher's shortcut: always start division from a leading 1. If the current part does not have enough bits to match the divisor length, bring down bits from above until the divisor-length group is formed. This keeps the division shorter and avoids unnecessary work.

✖ XOR Division Rule
After division, take the last r bits of the remainder as CRC bits.
Full form: XOR means Exclusive OR. In CRC, modulo-2 subtraction is performed using XOR.
5

Make the Codeword and Check at Receiver

The sender replaces the appended zeroes with the calculated CRC remainder. In the lecture example, the final remainder area may look longer during division, but only the last r bits are used. Since r = 4, take the last four bits from the LSB side. The CRC bits are 0010.

The complete transmitted codeword is original message + CRC bits. In the example, message bits are m = 10 and redundant bits are r = 4, so the receiver gets 14 bits.

At the receiver side, the receiver divides the complete 14-bit codeword by the same divisor. If the final remainder is all zero, no error is detected. If any bit changes from 0 to 1 or 1 to 0, the division normally produces a non-zero remainder and receiver detects an error.

✅ Sender and Receiver CRC Flow
Both sender and receiver use the same divisor/generator.
6

Efficiency / Channel Utilization

Efficiency here means how many transmitted bits are actual message bits. If message length is m and CRC length is r, total transmitted bits are m + r. This can also be understood as channel utilization for useful message bits.

Efficiency Formula

Useful bits divided by total transmitted bits.

m / (m + r) x 100

Lecture Example

m = 10 data bits, r = 4 CRC bits.

10 / 14 x 100

Total Sent

The receiver gets the complete codeword, not only the original message.

14 bits
7

Teacher-Style Complete Steps

Step 1: Identify the message/dividend and the divisor/generator. If the divisor is polynomial, convert it into binary. Example: x^4 + x^3 + 1 becomes 11001.

Step 2: Find how many zeroes to append. Polynomial degree 4 means append 4 zeroes. Binary divisor 11001 has 5 bits, so append 5 - 1 = 4 zeroes.

Step 3: Divide the message with appended zeroes by divisor 11001 using XOR. Same values give 0 and different values give 1. Ignore quotient.

Step 4: Pick the last r bits of the remainder. In the lecture example, the CRC bits are 0010. Replace the appended zeroes with these bits.

Step 5: Send the full codeword to receiver. Receiver divides it by the same divisor. Remainder 0000 means no error detected; non-zero remainder means error detected.

Exam note: Most CRC questions are numerical. Theory is short, but mistakes happen in zero-appending, polynomial-to-binary conversion, and remainder selection.
