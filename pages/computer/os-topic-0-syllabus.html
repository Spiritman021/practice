Why Study This Syllabus Video?

Before diving into Operating Systems, having a clear roadmap saves you from wasting effort on low-yield topics. This overview does three things:

🎯 Prioritised Roadmap
Clearly marks which topics are star-marked mandatory vs. overview-level, so you know exactly where to spend the most time regardless of how much preparation time you have.
📊 Exam-Pattern Based
Every topic priority is decided based on previous year questions from GATE and UGC NET. The pattern for both exams overlaps significantly — GATE questions often repeat in UGC NET.

💡 Hard Work + Smart Work

Hard work is necessary, but smart work multiplies its effect. Focusing your preparation on high-weightage, easier topics first (like CPU Scheduling and Page Replacement Algorithms) is the smartest use of your limited study time — especially for GATE where accuracy and negative marking matter more than coverage.

📌 Key principle: In GATE, it doesn't matter how many questions you attempt — what matters is how many you answer correctly. Prioritise correctness over coverage.

Topic 2 — Process Scheduling (CPU Scheduling) ⭐⭐ MANDATORY

🔥 This entire topic is mandatory. Every single year there is at least one question from CPU Scheduling in both GATE and UGC NET. You cannot skip this.

Process Scheduling covers a variety of algorithms. Practice 1–2 numericals for each algorithm and you will be able to solve any exam question from this topic.

⚙️ CPU Scheduling Algorithms
AlgorithmTypeKey Concept
FCFS (First Come First Served)Non-preemptiveJobs run in arrival order; simple but can cause convoy effect
SJF (Shortest Job First)Non-preemptiveShortest burst time goes first; optimal average waiting time
SRTF (Shortest Remaining Time First)PreemptivePreemptive version of SJF; context switch on shorter arrival
Round RobinPreemptiveFixed time quantum; fair CPU sharing; used in time-sharing OS
Priority SchedulingBoth versionsHighest priority runs first; can cause starvation (solved by aging)
Practice Gantt charts and calculate Average Waiting Time & Turnaround Time for each algorithm.
📌 What to calculate: Average Waiting Time (AWT) and Average Turnaround Time (ATT). These are the standard numerical outputs asked in exams.

Topic 3 — Process Synchronization ⭐ MANDATORY

Process Synchronization deals with coordination between concurrent processes sharing resources. It is somewhat complex but highly rewarding once understood.

⭐⭐ Binary Semaphore — MOST IMPORTANT
  • Binary Semaphore (value: 0 or 1)
  • Wait (P) and Signal (V) operations
  • Mutual exclusion using semaphore
  • Producer-Consumer problem solution
  • Most exam questions come from here
📖 Peterson's Solution — Overview
  • Software-based synchronization for two processes
  • Uses flag[] array and turn variable
  • Know the basic working principle
  • Fewer exam questions — overview sufficient
💜 Exam focus: Questions primarily come from Binary Semaphore. Study it deeply. Peterson's solution is good to know at a conceptual level.

Topic 4 — Deadlock & Threads ⭐ DO IT

Deadlock is largely theoretical with one important numerical algorithm. Threads are mostly conceptual for exams.

🔒 Deadlock (Easy + Theory)
  • 📌 Definition, necessary conditions (Mutual Exclusion, Hold & Wait, No Preemption, Circular Wait)
  • 📌 Deadlock Prevention & Avoidance strategies
  • ⭐ Banker's Algorithm — the one numerical in this topic; learn it well
  • 📌 Resource Allocation Graph (RAG) — detect cycles
🧵 Threads (Conceptual)
  • 📌 User-Level Threads — managed by user-space thread library
  • 📌 Kernel-Level Threads — managed by OS kernel directly
  • 📌 Advantages and differences between both types
  • ℹ️ Light on numericals; mostly definition-based questions
🏦 Banker's Algorithm tip: It's surprisingly easy once practised. The algorithm checks if a system is in a "safe state." Run through 2–3 examples and you'll get it right every time in the exam.

🎯 Page Replacement — Why It's Your Best Friend

Page replacement algorithms are easy to learn, easy to calculate, and asked every year. If you practice just 2–3 problems each for FIFO, Optimal, and LRU, you will never get this wrong in the exam. This is where smart work pays off most in OS.

Topic 6 — Disk Scheduling ⭐⭐ MANDATORY

Disk Scheduling is a numerical-heavy topic with very easy-to-learn algorithms. Like CPU Scheduling, just 1–2 practice problems per algorithm is enough.

💿 Disk Scheduling Algorithms
AlgorithmHead Movement PatternKey Point
FCFSIn order of request arrivalSimple but not optimal; high seek time possible
SSTF (Shortest Seek Time First)Closest cylinder request nextReduces average seek time; can cause starvation
SCAN (Elevator)Moves one direction, then reversesLike an elevator; serves requests in both directions
C-SCAN (Circular SCAN)One direction only, jumps back to startMore uniform wait time; fairer distribution
LOOK / C-LOOKLike SCAN/C-SCAN but goes only to last requestEfficient variant of SCAN; fewer unnecessary head movements
Calculate total head movement (total cylinders traversed) for each algorithm given a request queue and starting head position.
📌 Practice tip: Draw the head movement on a number line for each algorithm. Calculate total seek distance by adding all moves. That's what the exam asks.

1
Basic Introduction
OS Definition
System software acting as intermediary between user/apps and hardware; manages resources; provides execution environment for programs.
Types of OS
Batch OS, Multiprogramming OS, Multitasking OS, Real-Time OS (Hard & Soft RTOS), Time-Sharing OS, Distributed OS.
Process State Diagram ⭐
New → Ready → Running → Waiting/Blocked → Terminated. Also: Suspended Ready, Suspended Blocked states. Know every transition cause.
System Calls
Interface between user program and OS kernel. Fork() creates child process, Exec() replaces process image, Wait() makes parent wait for child.
2
CPU Scheduling ⭐⭐ MANDATORY
FCFS
Non-preemptive. Process runs to completion or blocks. No priority. Can cause convoy effect (short jobs wait behind long jobs).
SJF
Non-preemptive. Pick shortest burst time job. Minimum average waiting time (optimal). May starve long jobs.
SRTF
Preemptive SJF. Preempt if new arrival has shorter remaining time. Best for minimizing average waiting time.
Round Robin
Preemptive. Each process gets time quantum Q. After Q expires → go to end of ready queue. Used in time-sharing systems.
Priority
Highest priority runs first. Preemptive or non-preemptive version. Starvation solved by Aging (increase priority over time).
Key metrics
Arrival Time (AT), Burst Time (BT), Completion Time (CT). Turnaround Time = CT − AT. Waiting Time = TAT − BT.
3
Process Synchronization ⭐
Race Condition
Outcome depends on order of execution of concurrent processes accessing shared data. Must be prevented.
Critical Section
Code segment accessing shared resources. Requirements: Mutual Exclusion, Progress, Bounded Waiting.
Binary Semaphore ⭐⭐
Value 0 or 1. Wait(S): if S=0, block; else S=S−1. Signal(S): S=S+1, wake blocked process. Ensures mutual exclusion.
Counting Semaphore
Value can be any non-negative integer. Used to control access to a pool of identical resources.
Peterson's Solution
Software solution for 2 processes. Uses flag[] array and turn variable. Satisfies all 3 critical section requirements.
4
Deadlock & Threads ⭐
4 Necessary Conditions
Mutual Exclusion, Hold & Wait, No Preemption, Circular Wait — ALL four must hold for deadlock.
Prevention
Eliminate at least one of the four necessary conditions. E.g., allow preemption, require all resources upfront.
Avoidance
Banker's Algorithm: check if granting a request keeps system in safe state. Safe state = safe sequence exists.
Banker's Algo ⭐
For each process: check Need ≤ Available → allocate → add to Work → mark finished → repeat. If all finish = safe state.
User-Level Threads
Managed by thread library (not OS). Fast context switch. If one blocks, all block (OS sees single process).
Kernel-Level Threads
Managed by OS kernel. True parallelism possible. Slower creation/management. One blocking doesn't stop others.
5
Memory Management ⭐⭐ MANDATORY
Paging ⭐
Divide logical memory into fixed-size pages; physical memory into frames. Page Table maps page → frame. Logical addr = (page#, offset).
Segmentation ⭐
Divide into variable-size segments (code, data, stack). Segment Table stores (base, limit). Physical addr = base + offset.
Internal Fragmentation
Wasted space INSIDE allocated memory block (paging — last page may be partially used).
External Fragmentation
Wasted space OUTSIDE allocated blocks (segmentation — free memory exists but not contiguous enough).
Virtual Memory
Run programs larger than physical RAM. Demand Paging loads pages only when needed. EAT = (1−p)×mem_time + p×page_fault_time.
FIFO Page Replacement
Oldest page in memory is replaced first. Simple. Suffers from Belady's Anomaly (more frames → more page faults).
Optimal Page Replacement
Replace page not used for longest time in future. Theoretical minimum faults; not implementable (needs future knowledge).
LRU Page Replacement ⭐
Replace least recently used page. Approximates optimal. No Belady's Anomaly. Most commonly asked in exams.
Thrashing
Process spends more time in paging than execution. Cause: too many processes competing for too few frames.
6
Disk Scheduling ⭐⭐
FCFS
Serve requests in arrival order. Simple, fair, but high seek time. No starvation.
SSTF
Serve nearest cylinder next. Reduces seek time. Risk of starvation for far requests.
SCAN (Elevator)
Head moves in one direction, serves all requests, reaches end, reverses. No starvation.
C-SCAN
Head moves in one direction only, jumps back to beginning (without serving on return). More uniform wait.
LOOK / C-LOOK
Like SCAN/C-SCAN but head only goes to last request in each direction, not disk edge. More efficient.
Metric
Total head movement = sum of absolute differences between consecutive cylinder positions visited.
7
UNIX Commands & File Management
Key Linux Commands
ls, mkdir, cd, chmod, rm, cp, mv, pwd, cat, grep, ps, kill, top, man
chmod
Change file permissions. Format: chmod [permissions] [file]. E.g. chmod 755 file.sh → rwxr-xr-x
File System Calls
open(path, flags, mode) → fd; read(fd, buf, count); write(fd, buf, count); close(fd). Know parameter meanings.
Sequential Access
Records read one by one in order. Simple. Cannot go backwards easily. Good for tape storage.
Direct/Random Access
Jump to any record by position/block number. Good for disks. O(1) access time.
Security Basics
Virus (self-replicating), Vulnerability (exploitable weakness), DoS attack, Encryption for confidentiality.
