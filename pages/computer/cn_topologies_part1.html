INTRO

What is a Network Topology?

A network topology defines the physical arrangement of devices and how they are connected to one another. It determines how cables are laid out, how data flows between nodes, and what happens when one component fails.

The primary topology types are: Mesh, Star, Bus, Ring, and Hybrid. Each has distinct mathematical properties (number of cables and ports required), reliability characteristics, costs, and security implications. This lecture covers Mesh and Star in depth.

Mesh Topology
Every device connects to every other device directly. Dedicated point-to-point links.
Star / Hub Topology
All devices connect to a central Hub. No direct device-to-device links.
MESH TOPOLOGY

Mesh Topology — Full Analysis

In a mesh topology, every single device is directly connected to every other device in the network via a dedicated cable. There is no shared link — each pair of devices has its own private physical wire.

📐 Mesh Topology — Key Formulas (n = number of devices)
Number of Cables
n × (n−1) / 2
Also written as nC2. Same as the number of edges in a complete graph Kn.
n=10 → 10×9/2 = 45 cables
Ports per Device
n − 1
Each device needs one port for each other device it connects to directly.
n=10 → 9 ports per device
Total Ports (all devices)
n × (n−1)
Sum of all ports across every device in the network.
n=10 → 10×9 = 90 total ports
Note: Number of cables = n×(n−1)/2 but total ports = n×(n−1). Each cable uses 2 ports (one on each end), so total ports = 2 × cables.
🧮 Graph Theory Connection: The cable formula n×(n−1)/2 is identical to finding the number of edges in a complete graph Kn. This is also nC2 — choosing 2 nodes from n to connect. This is a commonly asked exam connection.
⚖️ Mesh Topology — Properties at a Glance
Reliability
Very High ✓
If one cable fails, data takes an alternative path. No single point of failure. System keeps running.
Cost
Very High ✗
Massive cabling required. 10 devices = 45 individual cables. Scales catastrophically with more devices.
Security
High ✓
Every link is dedicated. A→D message cannot be seen by B or C. Private point-to-point channels.
Maintenance
Very Difficult ✗
With 100 devices = 4,950 cables to manage. Identifying a faulty cable among thousands is extremely hard.
Communication Type
Point-to-Point
Dedicated link per pair. Does NOT support multicast or multipoint sharing.
Suitable for Large LAN?
No ✗
Cost and maintenance make mesh impractical for large LANs. Used in small, critical networks only.
✅ Advantages of Mesh
  • Highest reliability — multiple redundant paths
  • No single point of failure anywhere
  • High security — dedicated private links
  • Simultaneous transmission on all links possible
  • Easy fault isolation — each link is independent
❌ Disadvantages of Mesh
  • Extremely high cost — n×(n−1)/2 cables
  • Very difficult maintenance at scale
  • Not practical for large networks
  • High number of ports needed per device
  • No multicast/broadcast support
STAR / HUB TOPOLOGY

Star Topology — Full Analysis

In a star topology, no device connects directly to another. Instead, every device connects to a centralised multi-port hardware device called a Hub. When device A wants to send a message to device C, it cannot do so directly — the message must pass through the Hub first.

The Hub is a pure hardware device with no intelligence — it does not know which device is the intended recipient. It simply receives a signal on one port and immediately broadcasts it out of every other port. This is the defining characteristic that differentiates a Hub from a Switch (which operates at Layer 2 and can target specific ports).

📐 Star Topology — Key Formulas (n = number of devices)
Number of Cables
n
Simply one cable per device, each running directly to the Hub. Much simpler than Mesh.
n=10 → 10 cables only
Ports per End-Device
1
Each end-device only needs a single port — the one that connects to the Hub.
n=10 → 1 port per device
Total Ports (end devices)
n
n devices × 1 port each. The Hub itself requires n ports (one per device).
n=10 → Hub needs 10-port Hub
Cables: Mesh needs n×(n−1)/2. Star needs just n. For 10 devices: Mesh = 45 cables vs Star = 10 cables.
⚖️ Star Topology — Properties at a Glance
Reliability
Very Low ✗
Single point of failure. Hub breaks → entire network stops instantly. All communication halts.
Cost
Normal / Low ✓
Far fewer cables than Mesh. Only n cables needed. Hub itself adds some hardware cost.
Security
Low ✗
Hub broadcasts to ALL devices. A message intended for device C is also seen by A, B, D, E. No privacy.
Maintenance
Easy ✓
Only n cables. Easy to add/remove a device. A faulty device can be unplugged without affecting others.
Communication Type
Point-to-Point
Each device has a dedicated cable to the Hub. Not a shared wire (multipoint). But Hub still broadcasts.
Suitable for LAN?
Yes ✓
Very commonly used for home/office LANs due to low cable count and easy management.
🔄 Hub vs Switch — Critical Distinction: A Hub broadcasts to ALL ports (no intelligence, Layer 1). A Switch sends data only to the intended port (uses MAC address table, Layer 2). Modern networks use switches, not hubs — but the topology is still called "star" even with a switch at the centre.
✅ Advantages of Star
  • Low cable count — just n cables
  • Easy to install and maintain
  • Adding or removing a device doesn't affect others
  • Fault in one device is isolated easily
  • Widely used — very practical for LANs
❌ Disadvantages of Star
  • Hub failure = entire network fails (SPOF)
  • Low security — Hub broadcasts to everyone
  • Hub = additional hardware cost
  • All traffic must pass through central hub
  • Hub creates bottleneck under heavy load

Key Exam Facts
Mesh — Must Know
  • Every device connects to every other device
  • Cables: n×(n−1)/2 = ⁿC₂
  • Ports per device: n−1
  • Total ports: n×(n−1)
  • Same formula as complete graph Kn
  • Communication: Point-to-Point only
Star / Hub — Must Know
  • All devices connect to central Hub
  • Cables: n (one per device)
  • Ports per device: 1
  • Hub = pure hardware, NO intelligence
  • Hub failure = entire network down
  • Hub broadcasts to ALL ports always
Hub vs Switch (Exam Trap)
  • Hub = Layer 1, broadcasts to all
  • Switch = Layer 2, sends to target MAC
  • Both used in Star topology physically
  • Modern LANs use Switch, not Hub
  • Topology name = Star regardless of Hub or Switch
Worked Example (n=4)
  • Mesh cables: 4×3/2 = 6
  • Mesh ports/device: 3
  • Mesh total ports: 4×3 = 12
  • Star cables: 4
  • Star ports/device: 1
  • Hub needs: 4-port hub

⚡ Last-Minute Must-Know Points

🔥 High-Yield Facts
  • Mesh formula: n×(n−1)/2 cables
  • Star: only n cables, 1 port per device
  • Mesh = highest reliability, highest cost
  • Star = lowest reliability (SPOF = Hub)
  • Hub broadcasts = low security in star
📌 Common Exam Traps
  • Total ports ≠ cables: Mesh ports = 2 × cables
  • Star port-per-device = 1, not n−1
  • Hub is Layer 1, Switch is Layer 2
  • Both Mesh AND Star use point-to-point
  • Mesh doesn't support multipoint/multicast
💡 Quick Memory Hooks
  • Mesh = complete graph = ⁿC₂ edges
  • Star = spider web — hub at centre
  • Hub down → star collapses entirely
  • Mesh: more nodes → cables grow quadratically
  • Star: more nodes → cables grow linearly
