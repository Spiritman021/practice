1.1

What Is an Operating System?

An Operating System (OS) is system software — a program that lives inside your computer and has one core job: act as an interface (intermediary) between the user and the hardware.

📌 One-line exam definition: "An Operating System is system software that acts as an interface between the user and the hardware, manages system resources, and provides an environment for executing programs."
🏗️ Abstract View of a Computer System — 4 Layers
L3

👤 Users

People who use the computer — students, programmers, data entry operators, etc. They interact via applications or command prompt.

↕️
L2

📱 System & Application Programs

Compilers, assemblers, databases, Microsoft Word, browsers, games — these define how resources solve user problems.

↕️
L1

⚙️ Operating System ← THE INTERFACE

Controls and coordinates hardware use among all programs. This is the layer that does the actual management work.

↕️
L0

🖥️ Computer Hardware

CPU, RAM, Hard Disk, Keyboard, Mouse, Monitor, Printer, Scanner — the physical computing resources.

Request path: User → Application → OS → Hardware → result flows back up the same chain.
1.2

Hardware Components the OS Manages

Taking a laptop/desktop as an example — all of these physical components are hardware. The user never directly controls them; it is always the OS that mediates.

🖥️ Hardware Components — What the OS Controls
🧠
CPU
Brain of the system. Executes all instructions. OS controls which process gets CPU time and for how long.
⌨️
I/O Devices
Keyboard, Mouse (input), Monitor, Printer, Scanner (output). OS manages all device drivers and requests.
💾
RAM
Main memory. Fast but limited in size. All processes must be loaded here before CPU can execute them.
💿
Hard Disk
Secondary memory. Large, permanent storage. OS manages how data is stored and retrieved from it.
The user accesses all these through the OS — never directly. The OS is the gatekeeper for all hardware.
1.3

Why Do We Need an OS? (Without OS Scenario)

Imagine a world without any operating system. What would happen?

❌ Without OS
  • 🔴 To use any hardware, the user must write a separate program for every device interaction.
  • 🔴 Want to print a file? Write a program for the printer first, then print.
  • 🔴 Different programs needed for each device — printer, keyboard, CPU, disk — every single time.
  • 🔴 No authority to take back hardware from a user who has grabbed it — no fairness.
  • 🔴 User interaction with hardware becomes extremely complex.
✅ With OS
  • 🟢 OS hides all hardware complexity — user just clicks buttons.
  • 🟢 Press Ctrl+P in Word → OS triggers print system call → printer works automatically.
  • 🟢 OS provides common services so apps don't have to control hardware from scratch.
  • 🟢 OS is the authority — allocates and de-allocates hardware fairly among multiple users.
  • 🟢 OS coordinates multiple programs running simultaneously without conflict.

1.4

Two Major Goals of an OS

The two primary goals of any operating system are Convenience and Throughput. Every design decision in OS revolves around balancing these two.

🎯 Primary Goals of an Operating System
😊

CONVENIENCE

The user can access hardware in the easiest possible way. No technical knowledge needed. Just click, type, or tap — the OS handles the rest.

🏆 Windows dominated 95% of the market for years primarily because of convenience.
⚡

THROUGHPUT

Number of tasks executed per unit time. The more tasks completed in a given time, the higher the throughput. Users want their problems solved fast.

🐧 Linux maximises throughput — used in servers, cloud computing, and high-performance systems.
⚠️ Note: Linux also provides convenience — it is not inconvenient, it is just that Windows was optimised for convenience first, whereas Linux was built for high throughput servers and technical systems.
Modern OS (like Windows 11, macOS) try to achieve both goals simultaneously.

1.5

5 Key Functionalities of an Operating System

These functionalities are essentially the reasons why we use an OS. Each one solves a real problem that would otherwise burden the user or crash the system.

1

Resource Manager / Resource Governor

Hardware + Software Resources

The OS acts as a resource manager — it controls and allocates both hardware and software resources among all competing programs and users in a fair and efficient way.

  • Hardware resources: CPU time, RAM, Hard Disk, I/O devices, system buses (data bus, address bus, control bus).
  • Software resources: Files, access permissions, semaphores, shared services, authorization mechanisms.
  • Most critical in parallel access scenarios — e.g., on a server where hundreds of users send requests simultaneously. The OS decides who gets how much resource and for how long, then takes it back when done.
  • Example: Open Task Manager in Windows → you can see CPU %, RAM usage, Disk I/O — all managed and monitored by the OS in real-time.
👥 Government analogy: A government manages shared national resources (land, water, minerals) by deciding who can use them, how much, and under what rules. The OS does the same for computer hardware — it is the authority that enforces resource policies.
2

Process Management

CPU Scheduling

The OS manages all processes running on the system. A process is a program in execution. When you save a .C file and run it — that program becomes a process.

  • Multiple processes can run simultaneously: Microsoft Word + VLC Media Player + a game, all at once.
  • OS decides how and when each process gets CPU time using CPU Scheduling Algorithms.
  • Algorithms used: FIFO/FCFS, SJF, Round Robin, Priority Scheduling, SRTF, etc.
  • The OS ensures no single process monopolises the CPU, and all processes eventually get executed.

3

Storage Management

Hard Disk / Secondary Memory

Storage management deals with the Hard Disk (secondary memory) — how data is permanently stored, organised, and retrieved. This is different from memory management (which is about RAM).

  • The OS manages how data is written to tracks and sectors on the disk.
  • This is achieved through File Systems.
  • File systems used in Windows: NTFS (New Technology File System), FAT32
  • File systems used in Linux: ext4, NFS (Network File System), CIFS (Common Internet File System)
  • The file system organises data into files and directories, manages free space, and ensures data integrity.
📁 Why separate from Memory Management? Hard disk has no size constraint in the way RAM does — you can add more storage easily. RAM is strictly limited, which is why its management is a distinct and critical OS function.
4

Memory Management

RAM — Limited & Critical

Memory management deals with RAM (main memory). RAM is limited in size, and all processes must pass through RAM before the CPU can execute them. The OS must carefully manage this scarce resource.

RAM is full with P1, P2, P3. When P1 finishes executing, the OS deallocates its memory (0–100) and brings P4 into RAM. This cycle of allocation and deallocation is memory management.

RAM is a shared, limited resource. Without OS memory management, finished processes would sit in RAM forever, blocking new ones.
  • Multi-programming / Multi-tasking: Multiple processes are loaded into RAM simultaneously. OS switches CPU among them rapidly, giving the illusion of parallel execution.
  • Allocation: OS assigns a memory region to a process when it starts.
  • Deallocation (Swapping): When a process finishes or is paused, OS reclaims its RAM so another process can be loaded.
  • Key concepts covered in detail in separate topics: Paging, Segmentation, Virtual Memory, Page Replacement, Fragmentation.
💡 Why OS must manage RAM: If a finished process holds onto its RAM allocation and never gives it back, the system runs out of memory. New processes can't load, and the system freezes. The OS prevents this by forcefully reclaiming memory.
5

Security & Privacy

Authentication + Process Isolation

The OS provides two levels of security: user authentication (who can log in) and process isolation (preventing processes from interfering with each other).

🔐 User Authentication
  • 🔑 Password protection at login — only authorised users can access the system.
  • 🔑 Windows uses the Kerberos security protocol to store passwords in secured, encrypted files.
  • 🔑 Kerberos ensures that even if someone accesses the password file, they cannot read the actual passwords.
🛡️ Process Isolation
  • 🛡️ Each process is given a specific memory segment (e.g., P1 gets 0–100, P2 gets 101–200).
  • 🛡️ If P2 tries to access memory outside its range (101–200), the OS immediately blocks that process.
  • 🛡️ Prevents malicious or buggy processes from corrupting other processes' data.
🛡️ Process Memory Isolation — Security in Action

Process isolation prevents data corruption, privacy breaches, and system crashes caused by rogue or buggy programs.
1.6

How Users Access the OS — Two Paths

Users interact with the OS through two main channels. In both cases, the OS ultimately processes the request using system calls.

📱 Path 1: Through Applications (GUI)

Most users go through applications: Microsoft Word, VLC, games, browsers. The application provides a Graphical User Interface (GUI) — icons, menus, buttons.

Example: Press Ctrl+P in Word → Word asks OS to print → OS triggers the write() or print system call → Printer gets the data.

The user doesn't even know which system call happened — the OS is completely transparent.

💻 Path 2: Through Shell (CLI)

More technical users can bypass the application and talk to the OS directly via the shell (Command Line Interface).

  • 🖥️ Windows: Command Prompt (cmd)
  • 🐧 Linux/Mac: Terminal (bash, zsh)

In the terminal, you type commands (which are essentially system calls) directly — like open file.txt, rm file.txt, etc.

1.7

System Calls — The OS Language

Everything the OS does, it does via system calls. A system call is the programmatic way an application requests a service from the OS kernel. The user never sees them directly, but they happen constantly.

🔧 Common System Calls & When They Trigger
open

open() — File Opening

When you double-click a file in Explorer/Finder, the OS internally calls open("filename", flags) to open the file and return a file descriptor (handle).

read

read() — Reading Data

When you open a document and its contents appear on screen, the OS uses read(fd, buffer, count) to read data from the file into memory.

write

write() — Writing Data / Print

When you press Ctrl+S (save) or Ctrl+P (print), the OS calls write(fd, buffer, count) to write data to disk or send it to the printer device.

fork

fork() — Create Child Process

When a program spawns a new process (like a browser opening a new tab as a separate process), the OS uses fork() to create an identical copy (child) of the calling (parent) process.

close

close() — Closing Resources

When you close a file or application, the OS calls close(fd) to release the file descriptor and free up OS-level resources associated with the file.

System calls are the invisible backbone of every user interaction. Every click, keystroke, and button press eventually becomes one or more system calls.

🔑 Key Takeaway — What to Remember for Exams

OS = System Software = Interface between User and Hardware. It has 2 goals (Convenience, Throughput) and 5 functionalities (Resource Management, Process Management, Storage Management, Memory Management, Security). All OS actions happen through System Calls. Users access the OS either through Applications (GUI) or directly through the Shell/Terminal (CLI).

5 Functionalities — Quick Reference
① Resource Manager
Controls CPU time, RAM, I/O devices, files among all users/programs. Critical in multi-user server environments.
② Process Management
Manages process execution via CPU scheduling algorithms (FCFS, SJF, Round Robin, Priority). Multi-tasking support.
③ Storage Management
Manages hard disk via file systems. Windows: NTFS, FAT32. Linux: ext4, NFS, CIFS. Organises data into files/directories.
④ Memory Management
Manages RAM (limited). Allocates memory to process → deallocates when done (swapping). Enables multi-programming.
⑤ Security & Privacy
Login passwords (Windows uses Kerberos). Process isolation — if P2 accesses P1's memory → OS blocks P2 instantly.
