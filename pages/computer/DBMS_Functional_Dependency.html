Functional Dependency
in DBMS

Functional dependency describes how one attribute determines another attribute. It is the base idea behind keys, closure, and normalization rules.

Meaning of Functional Dependency

Functional dependency describes the relationship between attributes of a table. The general form is X → Y.

X → Y Diagram

X determines Y. Equivalently, Y is determined by X.
Example: SID → SName means Student ID determines Student Name. If confusion occurs in the name, SID clears it.
2

SID → SName Example

If two students have the same name, SID tells whether they are actually the same student or two different students.

3

Trivial and Non-Trivial FD

Trivial Functional Dependency

X → Y is trivial when Y ⊆ X.

Example: SID → SID or SID, SName → SID.

Always valid, because it is like an attribute determining itself.

Non-Trivial Functional Dependency

X → Y is non-trivial when Y is not a subset of X.

Example: SID → SName, EmpID → EmpName.

These need checking for valid/invalid cases.

Shortcut from the transcript: in a trivial FD, the intersection of left side and right side is not empty. In non-trivial examples like SID → SName, the sides are separate attributes.
4

Properties of Functional Dependency

These properties are used to derive new functional dependencies from existing ones.

Reflexivity
If Y ⊆ X, then X → YTrivial dependency. Always valid.
Augmentation
If X → Y, then XZ → YZAdd the same extra attributes to both sides.
Transitivity
If X → Y and Y → Z, then X → ZVery important for closure and normalization.
Union
If X → Y and X → Z, then X → YZSame determinant can combine dependent attributes.
Decomposition
If X → YZ, then X → Y and X → ZOnly the right side can be decomposed.
Pseudo Transitivity
If X → Y and WY → Z, then WX → ZUsed when the middle dependency has an extra attribute.
Composition
If X → Y and Z → W, then XZ → YWCombine left sides and right sides together.
Important Warning
XY → Z does not imply X → Z and Y → ZNever decompose the left side.
Most-used properties for upcoming topics: reflexivity and transitivity. Also remember: decompose the right side, not the left side.
