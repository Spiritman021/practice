Analyze R1

R1 is 1:M
In a one-to-many relationship, the relationship table's Primary Key comes from the many side. Here, E2 is on the many side, so the Primary Key of R1 is B.
PK(R1) = B
R1 can merge with E2
Entity E2 already has Primary Key B. Since R1 also uses B as Primary Key, R1 and E2 can be merged into one table.
T2 = E2 + R1
4

Analyze R2

R2 is M:N
In a many-to-many relationship, the relationship table needs a composite Primary Key from both participating entity keys.
PK(R2) = A + B
R2 cannot merge
R2 has composite key (A, B). It does not match the Primary Key of E1 alone or E2 alone, so R2 must remain a separate table.
T3 = R2
