SINGLE PARITY BIT

Detailed teacher-style notes on the least-expensive error detection method: add one redundant parity bit to make the number of 1s even or odd, understand what errors it detects/misses, and use Hamming distance to explain the detection limit.

1

What Is Single Parity Bit?

Single Parity Bit is a simple and least-expensive error detection method. Least expensive does not mean money here; it means we add very few redundant bits. If the message has m bits, we add only 1 redundant bit, so the transmitted codeword has m + 1 bits.

The teacher explains that in methods like double parity, CRC, or checksum, more redundant bits may be added. But in single parity, only one extra bit is added, so overhead is low. This method is useful where the application can tolerate a little error or wants a very cheap checking method, for example some audio communication where a tiny sound disturbance may be bearable.

➕ m Bits + 1 Parity Bit
It is cheap because only one extra bit is added.
2

Even and Odd Parity

Even parity makes the total number of 1s even. Odd parity makes the total number of 1s odd. Even parity is used more commonly in basic examples.

Data words are often discussed in fixed binary sizes such as 4 bits, 8 bits, 16 bits, 32 bits, or 64 bits because data communication is based on binary 0 and 1. In the lecture, examples use 4-bit data words.

🔢 Choosing the Parity Bit
Sender and receiver must follow the same parity rule.
Example: Data 1010 has two 1s, so even parity bit is 0. Data 1110 has three 1s, so even parity bit is 1, making codeword 11101.
3

What It Detects and What It Misses

Single parity can detect every single-bit error. More generally, it detects errors when an odd number of bits change. It cannot detect errors when an even number of bits change, because parity remains the same.

Suppose sender sends 11101, where the last bit is the parity bit. If one bit changes and receiver gets 01101, the number of 1s becomes odd, so receiver detects that something is wrong. But receiver does not know exactly which bit changed, so it cannot correct the error.

Now suppose two bits change and receiver gets 00101. The total number of 1s is still even. Receiver thinks it is valid, even though sender had sent a different codeword. This is why even-count errors are missed. If three bits change, parity flips again, so the error is detected.

⚠ Odd Errors Detected, Even Errors Missed
Parity detects odd-count errors: 1, 3, 5, 7... but misses even-count errors.
Important: Single parity detects error but cannot locate the wrong bit, so it cannot correct the error.
4

Hamming Distance

Hamming Distance between two codewords is the number of positions where they differ. To calculate it, perform XOR between the two codewords and count the number of 1s in the result.

In the transcript, the teacher uses XOR examples. XOR gives 0 when bits are same and 1 when bits are different. So after XOR, the number of 1s directly tells the Hamming distance.

📐 XOR and Count 1s
XOR gives 1 wherever bits are different.
5

Minimum Hamming Distance and Detection Power

If the minimum Hamming distance among valid codewords is d, then the code can detect up to d - 1 bit errors. For single parity, minimum Hamming distance is 2, so it detects 1-bit errors.

Why not detect 2-bit errors? If a valid codeword like 00000 changes by two bits, it may become another valid codeword such as 11000. Receiver sees a valid codeword and cannot know that an error happened. But if only one bit changes, such as 10000, that pattern is not a valid even-parity codeword, so receiver detects the error.

✅ Valid Codewords and d - 1 Rule
Changing two bits can turn one valid parity codeword into another valid codeword, so 2-bit errors may be missed.
6

Teacher-Style Complete Summary

Step 1: Take a message of m bits and add one redundant bit. Codeword size becomes m + 1. This is why single parity is called least expensive.

Step 2: In even parity, make total 1s even. Example: 1010 already has two 1s, so parity is 0. Data 1110 has three 1s, so parity is 1 and codeword becomes 11101.

Step 3: If one bit changes, parity becomes invalid, so error is detected. If two bits change, parity may remain valid, so the error may be missed.

Step 4: Single parity detects all odd-numbered bit errors: 1, 3, 5, 7 and so on. It misses even-numbered bit errors: 2, 4, 6 and so on.

Step 5: Hamming distance explains the limit. Minimum Hamming distance for single parity is 2, so it can detect d - 1 = 1 bit error.

Full form: XOR means Exclusive OR. In Hamming distance, XOR gives 1 at positions where two codewords differ.
