ERROR DETECTION & CORRECTION

Introduction to errors in transmitted data: how receiver detects a mismatch, how correction goes one step further, single-bit vs burst error, burst length numericals, detection methods, Hamming code, and redundancy.

Single-bit ErrorBurst ErrorCRCHamming Code
1

What Is an Error?

An error occurs when the data received is different from the data sent. If sender transmits 101 but receiver gets 100, the meaning changes. In decimal, 101 is 5 while 100 is 4.

⚠ Sent Data vs Received Data
Error detection first tells us that something changed; correction identifies and fixes the changed bit.
2

Detection vs Correction

Error detection tells the receiver that the data contains an error. Error correction is the advanced step: it also identifies where the error occurred, so the bit can be corrected.

🔍 Two Levels of Error Handling
Correction requires more information than detection.
Layer point: Error detection/correction appears in Data Link Layer and Transport Layer. In this topic, the focus is Data Link Layer bit-level checking.
3

Types of Error: Single Bit and Burst

Single-bit error means only one bit changes. Burst error means two or more bits change. Burst errors are more common because noise usually lasts longer than the duration of one bit.

🔢 Single-bit Error vs Burst Error
Burst length includes unchanged bits between the first and last corrupted bits.
4

Duration Numericals

If a channel bandwidth is 1 Gbps, it transmits 10^9 bits per second. Therefore one bit duration is 1 / 10^9 second = 1 nanosecond. If noise lasts longer, many bits may be corrupted.

Bit Duration

For 1 Gbps channel, one bit takes 1/10^9 second.

1 ns

Corrupted Bits

Corrupted bits = bandwidth in bps x error duration.

B x T

Example

1/1000 sec error at 1 Gbps corrupts 10^9 x 10^-3 bits.

10^6 bits
Why burst is common: real noise usually lasts for much more than 1 ns, so it affects many bits, not only one.
5

Detection and Correction Methods

Common error detection methods are Simple Parity, 2-Dimensional Parity Check, Checksum, and CRC. For correction, the important method is Hamming Code.

🛠 Methods Map
CRC is especially important at Data Link Layer; checksum is commonly discussed at Transport Layer too.
6

Redundancy: The Base Idea

All these methods depend on redundancy. If we send only the original data bits, the receiver has no extra information to verify correctness. So we add redundant bits that help detect or correct errors.

➕ Extra Bits for Checking
Redundant bits are overhead, but they make error detection/correction possible.
