Purpose of ON DELETE CASCADE

ON DELETE CASCADE is used with a Foreign Key to maintain referential integrity automatically. When a row is deleted from the parent table, all matching rows in the child tables are deleted automatically.

Simple meaning: Delete the parent record once, and the database automatically deletes all dependent child records that reference it.
2

Employee Example

Assume Employee is the parent table. Two child tables, Salary_Detail and Dependent, store information using employee IDs from the parent table. Their ID values must come from Employee.ID.

SQL Setup

The child tables define their ID column as a Foreign Key referencing Employee(ID). The important keyword is ON DELETE CASCADE.

CREATE TABLE Salary_Detail (
  ID NUMBER,
  Salary NUMBER,
  FOREIGN KEY (ID) REFERENCES Employee(ID)
  ON DELETE CASCADE
);

CREATE TABLE Dependent (
  ID NUMBER,
  Relation_Name VARCHAR2(20),
  FOREIGN KEY (ID) REFERENCES Employee(ID)
  ON DELETE CASCADE
);
Invalid Child Insert
Integrity Constraint Violated
If Employee has IDs 1 and 2 only, inserting ID = 10 into Salary_Detail or Dependent fails because 10 does not exist in the parent table.
Valid Child Insert
1 Row Inserted
Inserting ID = 1 is allowed because employee 1 exists in the parent table. The FK rule is satisfied.
-- This fails because employee 10 does not exist
INSERT INTO Salary_Detail VALUES (10, 50000);
Integrity constraint violated

-- This works because employee 1 exists
INSERT INTO Salary_Detail VALUES (1, 50000);
1 row inserted

INSERT INTO Dependent VALUES (1, 'Father');
1 row inserted
4

Deletion Behavior

Deleting from child tables is usually not a problem. The special case is deleting from the parent table. Without a proper action, child records can become orphaned. With ON DELETE CASCADE, the DBMS automatically removes child rows first.

After Deleting Employee ID 1
Before DELETE
Employee: IDs 1, 2
Salary_Detail: ID 1
Dependent: ID 1
→
After DELETE FROM Employee WHERE ID = 1
Employee: ID 2 remains
Salary_Detail: No data found
Dependent: No data found
Only one parent row was manually deleted. The matching child rows were automatically deleted by cascade.
DELETE FROM Employee
WHERE ID = 1;

1 row deleted from Employee
Matching rows automatically deleted from Salary_Detail and Dependent
Final exam line: ON DELETE CASCADE preserves referential integrity by automatically deleting child records when their referenced parent record is deleted.

ON DELETE CASCADE:A Foreign Key action that automatically deletes matching child rows when the referenced parent row is deleted.
Purpose:To maintain referential integrity and prevent orphan child records.
Used with:FOREIGN KEY (...) REFERENCES Parent(...)
