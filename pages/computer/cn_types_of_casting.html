Types of Casting

How data is addressed and delivered on a network — One-to-One (Unicast), One-to-All (Broadcast), and One-to-Group (Multicast). Broadcast is further split into Limited and Direct.

👤 Unicast — 1 to 1 📣 Broadcast — 1 to All 👥 Multicast — 1 to Group 255.255.255.255 (Limited) IGMP • Class D
1

Overview — Three Types of Casting

The main purpose of a network is sharing data and files. The way that data is addressed — i.e., who receives it — defines the type of casting. There are three fundamental types:

👤
Unicast
1 TO 1
  • One sender, one specific receiver
  • Most common type in daily use
  • e.g. M1 sends data to M2 only
  • HTTP, FTP, SSH, Telnet
  • Unique source → unique destination
📣
Broadcast
1 TO ALL
  • One sender → ALL hosts receive
  • Two sub-types: Limited & Direct
  • Limited: within same network
  • Direct: to all in another network
  • ARP, DHCP, RIP use broadcast
👥
Multicast
1 TO GROUP
  • One sender → specific group
  • Group = similar kind of people
  • Uses IGMP protocol
  • Uses Class D IP addressing
  • e.g. live streaming, video conf
💡 Gate Smashers Key Line: Unicast = 1→1, Broadcast = 1→All, Multicast = 1→Group. Broadcast sends to everyone whereas Multicast targets only a specific group of similar people. Broadcast is a subset concept of Multicast in the sense that multicast is more targeted.
2

Unicast — One-to-One Communication

Unicast is the most common form of communication in computer networks. Here, one machine (source) sends data to exactly one other machine (destination). Every day when you browse a website, send an email to one person, or SSH into a server — that is unicast.

👤 Unicast — M1 sends data to M2 only (Network A, Class A: 90.0.0.0)
M1 (90.0.0.1) sends data only to M2 (90.0.0.2). M3 does not receive it. This is 1-to-1 unicast communication — the most common type used in daily internet traffic.
👤 Unicast Examples: HTTP/HTTPS (web browsing), FTP (file transfer), SSH (remote login), Telnet, most email (SMTP to specific server). When you send a WhatsApp message to one person — that is unicast.
3

Broadcast — One-to-All (Two Sub-Types)

In broadcast, a message is sent from one source to all hosts. Broadcast has two important sub-types: Limited Broadcast (within same network) and Direct Broadcast (to all hosts of a specific other network).

📢 Limited Broadcast

Sends a message to all hosts within the same (local) network. The message stays inside the network — routers do not forward it.

BROADCAST ADDRESS
255.255.255.255

All bits = 1 in the IP address. Any packet with this destination is delivered to every host on the local network segment.

Used by: DHCP Discover, ARP (within subnet)

📥 Direct Broadcast

Sends a message to all hosts in a specific remote network. Routers can forward this across networks. The host portion of the IP is all 1s.

BROADCAST ADDRESS FORMAT
NetworkID . 255.255.255

Example: For Network B = 90.0.0.0, the direct broadcast address is 90.255.255.255

Host part replaced by all 1s (255 in each octet)
📣 Broadcast Types — Limited (within Network A) vs Direct (Network A to Network B)
Limited Broadcast (255.255.255.255): M1 → all hosts in Network A only — routers block it.  |  Direct Broadcast (92.255.255.255): M1 in Network A → all hosts in Network B — router forwards it.
💡 Direct Broadcast Address Formula: Take the Network ID and replace the entire host portion with all 1s (255 each).
Example: Network = 90.0.0.0 (Class A, 1 network octet) → Direct Broadcast = 90.255.255.255
Example: Network = 192.168.1.0 (Class C, 3 network octets) → Direct Broadcast = 192.168.1.255
4

Multicast — One-to-Group Communication

Multicast is used when you want to send data to a specific group of similar people rather than everyone. Think of a live research seminar online — only subscribed members receive it, not the entire internet.

👥 Multicast — One Sender, Data Delivered to Group Members Only
The sender transmits once to a multicast group address (Class D: 224.x.x.x). Only subscribed group members (M2, M5, M8) receive the data. Non-members (M3, M9) are excluded. Managed by IGMP.
👥 Multicast Key Facts:
• Protocol: IGMP (Internet Group Management Protocol) manages group memberships
• Address: Uses Class D IP addresses (224.0.0.0 to 239.255.255.255)
• Real-world: Live video streaming, IPTV, online meetings, routing protocols (OSPF uses 224.0.0.5)
• More targeted than broadcast — only group members receive, not everyone

Real-life analogy from Gate Smashers: Imagine a research group where only the subscribed members get emails. If you broadcast, everyone gets it (like a TV channel). If you multicast, only the research group members get it. If you unicast, you send one email to one specific person.

🎯 Most Likely Exam Points

  • Three types of casting: Unicast (1→1), Broadcast (1→All), Multicast (1→Group)
  • Broadcast has two sub-types: Limited Broadcast and Direct Broadcast
  • Limited Broadcast address = 255.255.255.255 (all bits = 1) — stays within local network, router blocks it
  • Direct Broadcast = NetworkID + host part all 1s — router forwards it to the target network
  • For Network 90.0.0.0 (Class A): Direct Broadcast = 90.255.255.255
  • For Network 192.168.1.0 (Class C): Direct Broadcast = 192.168.1.255
  • Multicast uses IGMP (Internet Group Management Protocol) for group management
  • Multicast uses Class D IP addresses: 224.0.0.0 to 239.255.255.255
  • Key distinction: Broadcast sends to ALL; Multicast sends to a specific group of similar users
  • OSPF routing protocol uses multicast address 224.0.0.5 (All OSPF routers)
  • Unicast is the most common type used in daily internet traffic (HTTP, FTP, SSH)
  • Multicast is efficient for one-to-many without flooding every node like broadcast
