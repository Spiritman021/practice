2.1

Scheduling Algorithms — Introduction

What is a Scheduling Algorithm? It is a method (policy) used by the OS to select a process from the Ready Queue and assign it to the CPU for execution. In a uni-processor system (single CPU), only one process can run at a time — the scheduler decides which process, for how long, and in what order.

📌 Ready Queue lies in RAM. The goal of multiprogramming is to keep as many processes as possible in the Ready Queue so that the CPU is never idle. Scheduling Algorithms choose which of these waiting processes gets the CPU next.

Preemptive vs Non-Preemptive Scheduling

🔵 Non-Preemptive

Once a process is given the CPU, it runs to completion (until its full Burst Time is done). It cannot be forcibly removed from the CPU while it is running.

Analogy: Teacher answers all 5 questions of Student 1 completely, then moves to Student 2.

  • FCFS — First Come First Serve
  • SJF — Shortest Job First
  • LJF — Longest Job First
  • HRRN — Highest Response Ratio Next
🟢 Preemptive

A running process can be stopped mid-way and placed back in the Ready Queue. Another process then gets the CPU. The preempted process resumes later.

Analogy: Teacher takes 2 questions from Student 1, then switches to Student 2 for 2 questions, and so on — round-robin style.

  • SRTF — Shortest Remaining Time First
  • LRTF — Longest Remaining Time First
  • Round Robin (Time Quantum)
  • Priority (mostly Preemptive)
▶ DIAGRAM 2.1-B — Why Does Preemption Happen? (3 Reasons)
1
⏱ Time Quantum Expired
CPU is set to execute each process for a fixed time slice (e.g., 2 ms). When the timer fires, the running process is preempted and placed back in the Ready Queue — even if it has remaining burst time. Used in Round Robin.
2
⭐ Higher Priority Process Arrives
A process with higher priority arrives while a lower-priority process is running. The CPU immediately switches to the high-priority process, preempting the current one. Used in Priority Scheduling (Preemptive).
3
⚡ Shorter Process Arrives
A new process arrives whose remaining burst time is less than the current process's remaining time. CPU switches to the shorter one immediately. Used in SRTF (Shortest Remaining Time First).
Any of these 3 events can trigger preemption — the running process returns to the Ready Queue.

Scheduling Criteria (What We Optimise For)

Different algorithms optimise for different goals. Here are the key criteria used to evaluate a scheduling algorithm:

⚙️
CPU Utilization
Keep CPU as busy as possible (maximize)
📦
Throughput
Processes completed per unit time (maximize)
⏱
Turnaround Time
Total time from arrival to completion (minimize)
⌛
Waiting Time
Time process waits in Ready Queue (minimize)
💬
Response Time
Time until first CPU response (minimize)
📌 Key Rule for Exams: CPU Utilization and Throughput → Maximise. Turnaround Time, Waiting Time, Response Time → Minimise.

CPU Scheduling Times — All Formulas

Before solving any CPU scheduling numerical, you must know these 6 time concepts. Questions in GATE, UGC NET, and college exams always ask for Turnaround Time, Waiting Time, or Response Time — all derived from these definitions.

🚪
Arrival Time (AT)
Point of time
Definition: The time at which a process enters the Ready Queue (arrives in RAM and is ready to run).

Given directly in problem

📌 It is a point of time, not a duration.
Example: Process arrived at time 0, 2, 4…
⚙️
Burst Time (BT)
Duration
Definition: The time a process needs on the CPU to complete execution. Also called CPU Time or Execution Time.

Given directly in problem

📌 It is a duration (not a point of time).
Example: Process needs 5 ms of CPU time.
✅
Completion Time (CT)
Point of time
Definition: The time at which a process finishes execution on the CPU.

Derived from Gantt Chart

📌 It is a point of time, not a duration.
Example: Process finished at time 12 ms.
🔄
Turnaround Time (TAT)
Duration
Definition: Total time a process spends in the system — from arrival to completion.

TAT = CT − AT

📌 It is a duration.
Includes waiting time + burst time.
⌛
Waiting Time (WT)
Duration
Definition: Time a process spends waiting in the Ready Queue — time spent not being executed.

WT = TAT − BT

📌 Also: WT = CT − AT − BT
This is the "wasted" time inside the system.
💬
Response Time (RT)
Duration
Definition: Time from arrival until the process gets the CPU for the very first time.

RT = First CPU time − AT

📌 For non-preemptive, RT = WT.
For preemptive (RR), RT < WT (shorter wait for first slice).

▶ DIAGRAM 2.2-B — The Bank Analogy (Gate Smashers Example)
🏦 Scenario: You entered the bank at 11:00 AM. The clerk takes 15 minutes to deposit your money. You came out at 12:00 PM (due to queues, form-filling, interruptions).
🚪 Arrival Time = 11:00 AM
When you entered the bank — the moment you joined the queue (Ready Queue). This is a point of time.
⚙️ Burst Time = 15 minutes
How long the clerk (CPU) takes to actually process your work. Depends on the CPU, not on you.
✅ Completion Time = 12:00 PM
When you walked out of the bank — your process is done. Point of time.
🔄 Turnaround Time = 60 min
CT − AT = 12:00 − 11:00 = 60 minutes. Total time you spent "inside the bank system."
⌛ Waiting Time = 45 min
TAT − BT = 60 − 15 = 45 minutes. Time spent waiting in queue, filling forms, interruptions — NOT being served by the clerk.
💬 Response Time
From your arrival (11:00 AM) until the clerk first attended to you. You may have waited in the queue before first contact.
The bank analogy maps perfectly to OS scheduling: YOU = process, BANK = system, CLERK = CPU.

D

CPU Bound vs I/O Bound — Why Waiting Time > 0

⚙️ CPU Bound Process

Most of the time is spent executing instructions on the CPU. Very few I/O requests. High CPU demand — they contribute to high burst times.

Example: Scientific simulation, video encoding, matrix multiplication.

🖨️ I/O Bound Process

Frequently needs Input/Output operations (read file, print, use scanner). CPU gives it up to wait for I/O — goes to Waiting State — then comes back to Ready Queue.

Example: Word processing, database queries, web server requests.

💡 Why is Waiting Time not just "queue time"? For I/O Bound processes, the waiting time also includes time spent in the Waiting State (doing I/O). They come back to the Ready Queue after I/O completes — this extra time all contributes to their total Waiting Time = TAT − BT.

Scheduling Algorithms — Core Points
DefinitionMethod to select a process from Ready Queue and assign it to CPU. In uni-processor: one process at a time.
Ready Queue locationLies in RAM. OS tries to keep as many processes as possible here (degree of multiprogramming).
Non-Preemptive ruleOnce CPU is assigned, process runs to completion. Cannot be forcibly removed during execution.
Non-Preemptive algorithmsFCFS, SJF (Non-preemptive), LJF, HRRN, Priority (NP variant)
Preemptive ruleRunning process can be stopped mid-execution and returned to Ready Queue; another process gets CPU.
Preemptive algorithmsSRTF, LRTF, Round Robin, Priority (Preemptive variant)
3 reasons to preempt① Time Quantum expired (Round Robin)  ② Higher-priority process arrives  ③ Shorter process arrives (SRTF)
Preemptive advantageBetter responsiveness — even the last process gets CPU sooner (no one process hogs CPU)
Preemptive disadvantageHigher overhead due to frequent context switches; more complex implementation
⭐ Exam Hot AlgorithmsSRTF and Round Robin are asked the most in GATE / UGC NET. FCFS for concept questions. Priority for theory.

RT vs WT (for RR)In Round Robin, RT < WT because process gets CPU slice early (first response fast) but waits longer overall.
