SELECTIVE REPEAT ARQ

A sliding-window protocol where both sender and receiver have equal windows, out-of-order frames are accepted and buffered, and only the missing frame is retransmitted using NAK. These notes follow the teacher's m = 2 and m = 3 examples.

GATEUGC NET2^(m-1)Out-of-Order Accepted
1

What Is Selective Repeat?

Selective Repeat is the next category of sliding window protocol after Go-Back-N. The main change is on the receiver side. In Go-Back-N, receiver window size was only 1, so out-of-order frames were rejected. In Selective Repeat, receiver window is also large.

Here, both sender and receiver maintain windows of the same size. Because the receiver window is bigger, the receiver can accept frames that arrive out of order and keep them in buffer. Later, when the missing frame arrives, the complete ordered data can be delivered upward.

🪑 Equal Sender and Receiver Windows
Selective Repeat uses a larger receiver window, so out-of-order frames can be accepted.
2

Window Size Formula

If sequence numbers are represented using m bits, then total sequence numbers are 2^m. In Selective Repeat, the maximum sender window and receiver window are both 2^(m-1), which is half of the sequence-number space.

The teacher stresses the exponent carefully: in Go-Back-N, the sender window was 2^m - 1, where minus 1 is outside the power. But in Selective Repeat, the formula is 2^(m-1), where m - 1 is inside the power. That difference is a common exam trap.

Example: if m = 2, total sequence numbers are 2^2 = 4, so the sequence range is 0, 1, 2, 3. Selective Repeat window size is 2^(2-1) = 2. So sender window is 0,1 and receiver window is also 0,1.

3

Why Window Cannot Be Greater Than 2^(m-1)

If the Selective Repeat window becomes larger than half the sequence-number space, duplicate old frames can fall inside the receiver's current window after wraparound. Then an old frame may be accepted again as if it were new.

Safe case with m = 2: sequence numbers are 0,1,2,3 and correct window size is 2. Sender sends 0 and 1. Receiver accepts 0 and 1, then its window moves to 2 and 3. If ACKs are lost and sender again sends old 0, receiver can reject it because its current window wants 2 and 3.

Wrong case: if window size is wrongly increased to 3, sender sends 0,1,2. Receiver accepts all three and its window moves to 3,0,1. Now if ACKs are lost and sender retransmits old 0 or 1, those old frames are inside the receiver's current window, so duplicates may be accepted. That is why Selective Repeat window must not be greater than 2^(m-1).

⚠ Duplicate Ambiguity When Window Is Too Large
The half-window rule prevents old and new copies of the same sequence number from overlapping.
4

Out-of-Order Frames Are Accepted

In Go-Back-N, if frame 1 is missing, frames 2 and 3 are discarded. In Selective Repeat, the receiver window is larger, so frames 2 and 3 can be accepted and buffered even if frame 1 is missing.

Teacher's example: with m = 3, total sequence numbers are 0 to 7 and window size is 2^(3-1) = 4. So the receiver window can be 0,1,2,3. If frame 0 arrives, it is accepted and the receiver moves forward. If frame 1 is lost but frames 2 and 3 arrive, receiver accepts 2 and 3 because they are inside its window.

But accepting does not mean delivering incomplete data to the upper layer. The receiver still needs frame 1. Like a file, if 90% has arrived but one middle part is missing, the file cannot be opened properly. So frames 2 and 3 are buffered until frame 1 arrives.

📦 Buffering Out-of-Order Frames
Selective Repeat accepts out-of-order frames, but keeps them buffered until the missing frame fills the gap.
5

NAK and Selective Retransmission

When the receiver notices that frame 1 is missing while later frames have arrived, it sends NAK 1. NAK means Negative Acknowledgement: "I need this missing frame." The sender retransmits only frame 1, not every frame after it.

This is why it is called Selective Repeat. It does not go back and resend all later frames like Go-Back-N. It selectively repeats only the frame that is missing or damaged.

🔁 Only Missing Frame Is Repeated
Selective Repeat repeats selectively: only the missing or damaged frame is resent.
Important: If the receiver sent ACK 4 instead of NAK 1, the sender would assume 0,1,2,3 are all accepted. NAK tells exactly which frame is missing.
6

Teacher-Style Complete Example

Step 1: Take m = 3. Total sequence numbers are 2^3 = 8, so range is 0 to 7. Selective Repeat window size is 2^(3-1) = 4. Sender and receiver both have window size 4.

Step 2: Sender sends frame 0. Receiver accepts 0, shifts its window, and sends ACK 1 to show that 0 is received and it can now accept from the next window range.

Step 3: Sender sends frame 1, but frame 1 is lost. Sender also sends frame 2 and frame 3. Unlike Go-Back-N, receiver accepts 2 and 3 because its receiver window is large enough.

Step 4: Receiver still cannot pass the data upward because frame 1 is missing. So it sends NAK 1. Sender retransmits only frame 1. After frame 1 arrives, the receiver can arrange 1,2,3 in order and move its window forward to accept later frames such as 4,5,6,7.

Full forms: ARQ = Automatic Repeat reQuest, ACK = Acknowledgement, NAK = Negative Acknowledgement.
