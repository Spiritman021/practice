FRAMING IN DATA LINK LAYER

Detailed teacher-style notes on framing: how Data Link Layer packs bits into frames, separates one frame from another using delimiters, handles FLAG/ESC inside data using byte stuffing, and handles delimiter-like bit patterns using bit stuffing.

FrameDelimiterByte StuffingBit Stuffing
1

What Is Framing?

One major responsibility of the Data Link Layer is framing. The bits coming from the upper layer are packed into separate units called frames. In the Network Layer, the PDU is called a packet; in the Data Link Layer, the PDU is called a frame.

The teacher explains framing with a postal example. If letters in a postal service were not placed inside envelopes, it would be difficult to distinguish where one letter ends and another begins. An envelope separates letters. Similarly, framing separates one message/frame from another.

To separate frames, special markers called delimiters are used. A delimiter tells the receiver where a frame starts and where it ends.

📦 Bits Packed into Frames
Data Link Layer PDU is called a frame.
Exam point: Network Layer PDU = packet; Data Link Layer PDU = frame. PDU means Protocol Data Unit.
2

Character-Oriented Framing

In character-oriented framing, framing is done using characters/bytes. A special character called FLAG is placed at the beginning and end of a frame. The receiver treats everything between two FLAGs as data.

Example: if data is ABCD, the transmitted frame can be written as FLAG ABCD FLAG. The first FLAG marks the start, the second FLAG marks the end, and ABCD is accepted as the actual data.

⚑ FLAG as Delimiter
FLAG does not belong to user data unless it is protected by stuffing.
3

Byte Stuffing: When FLAG Appears in Data

If the data itself contains the word FLAG, the receiver may wrongly treat it as the end of the frame. For example, suppose data from the upper layer is A FLAG C. If sender sends FLAG A FLAG C FLAG, the receiver may read only A between the first two FLAGs and treat the middle FLAG as the ending delimiter.

To prevent this, the sender inserts an ESC byte before a data FLAG. So data A FLAG C becomes FLAG A ESC FLAG C FLAG. The receiver rule is: if FLAG is preceded by ESC, then that FLAG is not a delimiter; it is actual data.

📝 Stuff ESC Before Data FLAG
Byte stuffing is also called character stuffing. ESC means Escape character.
4

Byte Stuffing: When ESC Appears in Data

If the data itself contains an ESC character, the sender inserts another ESC before it. This allows the receiver to identify that the ESC belongs to user data. So if data contains FLAG, sender transmits ESC FLAG; if data contains ESC, sender transmits ESC ESC.

The key point is that sender and receiver already know the same rules. The teacher compares this with games: in childhood or gully cricket, rules are decided before playing. Similarly, sender and receiver agree in advance that ESC before FLAG means FLAG is data, and ESC before ESC means ESC is data.

Shared rule: Sender stuffs extra bytes; receiver de-stuffs them using the same known rule. These extra ESC bytes are overhead, but they protect the actual data.
5

Bit Stuffing

In bit-oriented framing, stuffing is done at bit level, not character level. If five continuous 1s occur in data, the sender inserts a 0 after them. This prevents the data from accidentally looking like a special delimiter pattern.

The transcript explains that patterns like 0 followed by five 1s are generally used around the header/tail area as a delimiter or SFD. If similar bits appear inside actual data, the receiver may get confused and think data bits are part of a delimiter. So after five 1s, sender stuffs one 0 before a sixth 1 can appear.

🔢 Insert 0 After Five Continuous 1s
Receiver removes the stuffed 0 during de-stuffing.
Rule: whenever five consecutive 1s appear in data, insert a 0 immediately after them. SFD means Start Frame Delimiter.
6

Teacher-Style Complete Example

Character stuffing example: Data is A FLAG C. If sent directly with starting and ending FLAG, receiver may wrongly stop at the data FLAG. So sender transmits FLAG A ESC FLAG C FLAG. Receiver sees ESC before the middle FLAG and keeps that FLAG as data.

ESC example: If actual data contains ESC, sender sends ESC ESC. Receiver removes one ESC and keeps one ESC as actual data.

Bit stuffing example: If data contains 0111111, after five continuous 1s sender inserts 0, making it 01111101. Receiver later removes the stuffed 0 and restores the original data.

Final memory: Byte stuffing protects special characters; bit stuffing protects special bit patterns.
