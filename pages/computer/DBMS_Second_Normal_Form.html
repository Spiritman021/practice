Second Normal Form
2NF in DBMS

Second Normal Form removes partial dependency: every non-prime attribute must be fully functionally dependent on a candidate key.

Must be in 1NFNo Partial Dependency Prime AttributeCandidate Key
1

Rules of Second Normal Form

According to E. F. Codd, a relation is in Second Normal Form when it satisfies two rules.

Rule 1
The relation must already be in First Normal Form.
Rule 2
The relation must not contain any partial dependency.
Important form: all non-prime attributes should be fully functionally dependent on the candidate key, not on only a part of it.
2

What is Partial Dependency?

Partial dependency happens when a proper subset of a composite candidate key determines a non-prime attribute.

Partial Dependency Pattern
Candidate KeyProper SubsetsProblem FDWhy Problem?
ABA or BA → CC is non-prime and depends on part of the key.
In 2NF, a non-prime attribute should depend on the full candidate key, not only A or only B.
3

Customer-Store Example

In the customer table, the candidate key is composite: CustomerID + StoreID. The non-prime attribute is Location.

Table Not in 2NF

53MumbaiLocation depends on StoreID.
Location depends on StoreID only, which is a part of the composite key. So partial dependency exists.
Since StoreID → Location and StoreID is only part of CustomerID + StoreID, this table is not in 2NF.

Check Partial Dependencies

Use this test for every functional dependency: LHS should be a proper subset of candidate key and RHS should be non-prime.
