FCFS — First Come First Serve

🔵 FCFS — Algorithm at a Glance
Selection CriteriaArrival Time — who arrived first gets CPU first
ModeNon-Preemptive — once started, runs to full completion
StarvationNo — all processes execute in strict arrival order
Response Time= Waiting Time (CPU given exactly once per process)

2.4

SJF — Shortest Job First (Non-Preemptive)

🟩 SJF — Algorithm at a Glance
Selection CriteriaBurst Time — smallest BT among ARRIVED processes gets CPU
ModeNon-Preemptive — once started, runs to full completion
Step 1 Always!Check which processes ARRIVED first, then pick smallest BT among those
StarvationYes — long processes may starve if short ones keep arriving
🔗 SJF Tie-Breaking Rules (Exam Critical!): When BT is equal: (1) Pick smaller BT — tied. (2) Tie-break 1 — pick lower Arrival Time. (3) Tie-break 2 — pick lower Process ID (P1 before P2).

SJF selects smallest Burst Time among arrived processes. Non-preemptive.
1
At time 0 — CPU IDLE: No process has arrived (P1 and P3 arrive at 1; P2 at 2; P4 at 4). CPU idle 0 → 1.
2
At time 1: P1(BT=3) and P3(BT=2) both arrive. Apply SJF: P3 BT=2 < P1 BT=3 → Select P3. Runs 1 → 3.
3
At time 3: P3 done. Arrived: P1(BT=3, waiting since t=1) and P2(BT=4, arrived at 2). P1 BT=3 < P2 BT=4 → Select P1. Runs 3 → 6.
4
At time 6 — TIE! Arrived: P2(AT=2,BT=4) and P4(AT=4,BT=4). Both BT=4 — EQUAL! Tie-break: P2 AT=2 < P4 AT=4 → Select P2. Runs 6 → 10.
5
At time 10: P2 done. Only P4 remains → Select P4. BT=4 → runs 10 → 14. All done at t=14.

5-Step Method to Solve Any CPU Scheduling Numerical

1
Read the given data: Note AT and BT for all processes. Identify algorithm and mode (preemptive/non-preemptive).
2
Draw the Gantt Chart: Start from t=0. At each decision point, check which processes ARRIVED, apply selection criterion, handle CPU idle if needed.
3
Fill Completion Time (CT): Read the right-edge of each process block from the Gantt chart.
4
Calculate TAT: TAT = CT − AT for each process. Then Avg TAT = ΣTAT / n.
5
Calculate WT and RT: WT = TAT − BT. For non-preemptive: RT = WT. Then Avg WT = ΣWT / n.

Algorithm typeNon-Preemptive
Selection criterionBurst Time — smallest BT among ARRIVED processes gets CPU
CRITICAL step orderStep 1: check which ARRIVED. Step 2: pick smallest BT among those.

★
Universal Formulas & Star Points
CTRead from Gantt Chart (right edge of process block)
TATCT − AT = WT + BT
WTTAT − BT = CT − AT − BT
RT (non-preemptive)First CPU time − AT = WT (same value always)
★ CPU Idle ruleNo process in queue → IDLE on Gantt. Jump to next arrival. Idle does NOT add to WT.
★ RT matters only inPreemptive scheduling (Round Robin). In non-preemptive: RT = WT always.
★ SJF optimalSJF gives minimum avg WT — this exact statement appears in GATE MCQs.
★ Convoy effectFCFS: long process blocks all shorter ones behind it (slow truck analogy). SJF solves this.
★ SJF limitationNeeds BT info in advance — impractical in real OS. Used in batch systems only.
