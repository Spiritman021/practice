Learn how to prove whether two FD sets are equivalent by checking both directions: X covers Y and Y covers X.

1

Meaning of Equivalence

Two functional dependency sets X and Y may look different, but they are equivalent if they imply the same dependencies.

Two-way Cover Test
X
FD Set 1
X covers Y
and
Y covers X
Y
FD Set 2
If both directions are true, then X and Y are equivalent.
To check whether X covers Y, take the left side of every FD in Y, compute its closure using X, and verify that the right side appears.
2

Example 1: Equivalent Sets

Given:

X = { A → B, B → C }Y = { A → B, B → C, A → C }
CheckClosure usedDependency verifiedStatus
X covers YA+ under X = A,B,CA → B and A → CCovered
X covers YB+ under X = B,CB → CCovered
Y covers XA+ under Y = A,B,CA → BCovered
Y covers XB+ under Y = B,CB → CCovered
Both directions are true, so X and Y are equivalent.
3

Example 2: Not Equivalent

Given:

X = { AB → CD, B → C, C → D }Y = { AB → C, AB → D, C → D }
DirectionClosure testResult
X covers YAB+ under X = A,B,C,D, so AB → C and AB → D are covered. C+ under X gives C,D.True
Y covers XAB+ under Y gives A,B,C,D, so AB → CD is covered. C+ gives C,D.Partly true
Y covers XB+ under Y = B. But X contains B → C.False
Because B → C is not covered by Y, the two sets are not equivalent.
4

Exam Algorithm

StepWhat to do
1To test X covers Y, pick every FD in Y.
2For each FD A → B in Y, compute A+ using X.
3If every RHS from Y appears in the corresponding closure, X covers Y.
4Repeat the reverse direction: test whether Y covers X.
5If both directions are true, the sets are equivalent. If even one direction fails, not equivalent.
1Definition

X ≡ Y only when X covers Y and Y covers X.
