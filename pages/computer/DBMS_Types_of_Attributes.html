What is an Attribute?

An attribute is a property or characteristic of an entity. If Student is an entity, then Name, Roll_No, Registration_No, Father_Name, Address, and Mobile_No can be its attributes.

Registration_No Name Age Mobile_No Age from DOB Address
In ER diagrams, normal attributes use an oval. Special attribute types modify that oval.
2

Main Attribute Types

Single-Valued Attribute
Has only one value for an entity instance. Example: a university Registration_No for a student.
Registration_No
Normal single oval representation.
Multivalued Attribute
Can have more than one value. Example: Mobile_No or Address when a student has multiple numbers or addresses.
Mobile_No
Represented with a double oval.
Simple Attribute
Cannot be divided further. Example: Age as a single value like 20.
Age
Atomic and indivisible.
Composite Attribute
Can be divided into smaller parts. Example: Name can become first, middle, and last name.
NameFirstMiddleLast
Also called compound attribute.
Stored Attribute
Its value is stored directly. Example: Date_of_Birth.
Date_of_Birth
No special ER notation beyond normal oval.
Derived Attribute
Its value is calculated from another attribute. Example: Age derived from Date_of_Birth.
Age
Represented with a dotted oval.
Key Attribute
Uniquely identifies each row/entity instance. Example: Registration_No or Roll_No.
Registration_No
Represented by underlining the attribute name.
Non-Key Attribute
Does not guarantee uniqueness. Example: Name, Age, Date_of_Birth, or sometimes Mobile_No.
NameAge
Normal oval representation.

Required vs Optional

Required attributes are mandatory, like a student name during admission. Optional attributes can be left blank depending on the database requirement. This is mostly handled in SQL using constraints like NOT NULL, not by a special ER symbol.

Important: Required vs Optional is not a core ER notation topic. In SQL, the DBA decides it while creating table constraints.
5

Complex Attribute

A complex attribute is both composite and multivalued. Example: a student may have multiple addresses, and each address can be broken into street, city, state, and pin code.

Complex Attribute = Composite + Multivalued
Composite
Address can be divided into Street, City, State, Pin_Code.
+
Multivalued
Student can have permanent address and correspondence address.
Complex attributes combine both ideas at once.
Exam memory: Multivalued = double oval, Derived = dotted oval, Key = underlined attribute.

Important Examples
Registration_No:Single-valued and key attribute.
Mobile_No:Can be multivalued; may not be key because siblings can share it.
Student_Name:Composite: First_Name, Middle_Name, Last_Name.
