Overall Strategy & Foundation

The entire Computer Networks syllabus for both UGC NET and GATE revolves around the 7 layers of the OSI Model. Before diving into any specific topic, build a solid understanding of what each layer does and why it exists — this mental model makes every other concept far easier to retain.

Whether you have months to prepare or just days, the OSI model is the skeleton on which everything else hangs. Know the PDU (Protocol Data Unit) at each layer, the key protocols, and the hardware devices associated.

🎯 Golden Rule: For every layer, know its basic functionality, the protocols that live there, and the data unit (PDU) used. This alone covers ~40% of UGC NET questions.

Physical Layer

The Physical Layer is the foundation — it deals with the actual transmission of raw bits over a physical medium. For UGC NET, this layer is highly important because it produces many straightforward theory questions. For GATE, it is comparatively less critical. Importantly, no numerical problems come from this layer.

Focus your study on the types of cables (coaxial, twisted pair, fiber optic), network topologies (bus, star, ring, mesh, hybrid), and the different modes of transmission — simplex, half-duplex, and full-duplex.

💡 Key Concepts: Encoding schemes like Manchester Encoding and Differential Manchester Encoding are classic UGC NET question topics — understand how they represent 0s and 1s, not just the names.

Also understand the basic distinction between LAN, MAN, WAN, and Wireless LAN, and know which hardware devices operate at this layer (hubs, repeaters) versus at higher layers.

Data Link Layer (DLL)

The Data Link Layer has one of the heaviest topic loads in the entire syllabus. GATE expects deep mathematical knowledge of efficiency and throughput, but for UGC NET, your focus should be on conceptual understanding and — critically — memorising frame formats and header fields, as these are very heavily tested.

For Transmission Protocols, understand the three sliding window protocols: Stop and Wait (sender waits for ACK before sending next frame), Go-Back-N (retransmits all frames from the errored one), and Selective Repeat (retransmits only the errored frame).

📊 Sliding Window Protocol Comparison
ProtocolSender WindowReceiver WindowOn Error
Stop & Wait11Retransmit 1 frame
Go-Back-N (n-bit)2ⁿ − 11Retransmit ALL from error
Selective Repeat (n-bit)2ⁿ⁻¹2ⁿ⁻¹Retransmit ONLY errored

For MAC Protocols, study Pure Aloha (max efficiency ~18.4%), Slotted Aloha (~36.8%), CSMA/CD (used in wired Ethernet — detects and aborts on collision), and CSMA/CA (used in wireless — avoids collision). These efficiency figures are commonly asked directly.

For Error Control, CRC (Cyclic Redundancy Check) is the most important — understand the concept of polynomial division over binary data. Also know Checksum and Parity.

🔑 Crucial Tip: Memorise the Ethernet frame format (IEEE 802.3) completely — Preamble (7B), SFD (1B), Dest MAC (6B), Src MAC (6B), Type/Length (2B), Data (46–1500B), CRC (4B). Frame formats are very heavily tested in UGC NET.
3

Network Layer

The Network Layer is responsible for logical addressing and routing packets from source to destination across multiple networks. Two areas dominate: IP Addressing and Routing Protocols.

For IP Addressing, you must understand both Classful (Class A/B/C/D/E) and Classless (CIDR) addressing. UGC NET keeps this at a basic level — you don't need the complex variable-length subnetting required by GATE.

For Routing Protocols, understand Distance Vector Routing (each router shares its table with direct neighbours — e.g., RIP) and Link State Routing (each router broadcasts link states to all routers, builds a global map — e.g., OSPF).

🔑 Crucial Tip: Study the IPv4 header fields (Version, IHL, TTL, Protocol, Checksum, Source/Dest IP, etc.) and IPv6 header fields thoroughly — these are absolutely essential for UGC NET and GATE both.

Transport Layer

The Transport Layer ensures end-to-end communication between processes on different hosts. TCP (Transmission Control Protocol) is connection-oriented, reliable, and uses flow control and error recovery. UDP (User Datagram Protocol) is connectionless, faster but unreliable, used where speed matters more than reliability.

For GATE, advanced topics like congestion control, timers, and the three-way handshake are critical. For UGC NET, focus only on TCP and UDP header fields and port numbers.

Session, Presentation & Application Layers

The Session Layer manages sessions between applications — establishing, maintaining, and terminating them. The Presentation Layer handles data translation, encryption/decryption, and compression. For UGC NET, just knowing their basic functions is sufficient.

The Application Layer is where user-facing protocols live. DNS is the single most important protocol here and is asked very frequently. Beyond DNS, know HTTP, SMTP, and FTP — and their port numbers by heart.

Network Security

Network Security is an essential and high-yield part of the syllabus. Core concepts revolve around cryptography — specifically the distinction between symmetric and asymmetric key systems.

🔐 Symmetric vs Asymmetric Cryptography
PropertySymmetric KeyAsymmetric Key
Keys usedSame key for encrypt + decryptPublic key encrypts, Private key decrypts
SpeedFastSlow
Key distributionDifficult (must share securely)Easy (public key is open)
AlgorithmsDES, 3DES, AES, RC4RSA, DSA, ECC
Use caseBulk data encryptionKey exchange, digital signatures
🔑 RSA Algorithm: Choose large primes p and q → n = p×q → φ(n) = (p−1)(q−1). Public key = (e, n), Private key = (d, n), where e·d ≡ 1 mod φ(n). Security relies on the computational difficulty of factorising large numbers.

MAC Protocols — Efficiency
Pure ALOHA
  • Max efficiency: 18.4% (1/2e)
  • Transmit anytime; collision → wait random time
  • Vulnerable time: 2 × frame time
Slotted ALOHA
  • Max efficiency: 36.8% (1/e)
  • Transmit only at slot boundaries
  • Vulnerable time: 1 × frame time
CSMA/CD
  • Used in wired Ethernet (IEEE 802.3)
  • Sense before transmit; detect collision mid-send
  • Abort + jam signal + exponential back-off
CSMA/CA
  • Used in wireless (IEEE 802.11)
  • Avoid collision (can't detect in wireless)
  • Uses ACK + IFS + random back-off

TCP & UDP Headers
TCP Header (20 bytes min)
  • Source Port 16 bits
  • Destination Port 16 bits
  • Sequence Number 32 bits
  • ACK Number 32 bits
  • Header Length 4 bits
  • Flags: SYN, ACK, FIN, RST, PSH, URG
  • Window Size 16 bits
  • Checksum 16 bits
UDP Header (8 bytes — fixed)
  • Source Port 16 bits
  • Destination Port 16 bits
  • Length 16 bits
  • Checksum 16 bits
  • No sequence/ACK — no connection
  • Used: DNS, DHCP, VoIP, Video streaming
Network Security — Quick Reference
Symmetric Key
  • Same key for encrypt + decrypt
  • Fast, but key distribution is hard
  • Algorithms: DES, 3DES, AES, RC4
Asymmetric Key
  • Public key encrypts; private key decrypts
  • Slow, but easy key distribution
  • Algorithms: RSA, DSA, ECC
RSA Algorithm
  • Pick primes p, q → n = p × q
  • φ(n) = (p−1)(q−1)
  • Public key: (e, n)
  • Private key: (d, n) where e·d ≡ 1 mod φ(n)
  • Security: hard to factorise large n
Digital Signature
  • Sender hashes message, encrypts with private key
  • Receiver decrypts with sender's public key
  • Provides: Authentication + Non-repudiation

💡 Quick Memory Tricks
  • PDU mnemonic: Bit→Frame→Packet→Segment→Data
  • TTL decremented at EVERY router hop
  • Protocol field: TCP=6, UDP=17, ICMP=1
  • DNS = always port 53 (TCP + UDP)
  • UDP header = exactly 8 bytes always
