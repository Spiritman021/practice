OVERVIEW

Bus & Ring — At a Glance

Both Bus and Ring topologies belong to the multipoint / shared-medium family — all devices share a single communication channel rather than having dedicated point-to-point links. This makes them cheaper but introduces the problem of data collisions and low security.

Bus Topology
Backbone + drop lines + taps + terminators. All devices share one wire.
Ring Topology
Bus topology bent into a loop. Unidirectional signal flow. Monitor device manages the ring.
BUS TOPOLOGY

Bus Topology — Full Analysis

In a bus topology, all devices share a single central communication line called the backbone cable (also called a coaxial cable or thick Ethernet wire). This backbone has a very high bandwidth. Each device connects to the backbone via an individual drop line, and the physical connection point between the drop line and the backbone is an electrical device called a tap.

At both ends of the backbone, terminators are placed to absorb the signal so it does not bounce back and cause interference. Without terminators, signals would reflect and corrupt incoming transmissions.

🔌 4 Key Hardware Components: (1) Backbone cable — the shared highway. (2) Drop lines — individual device connections. (3) Taps — electrical junction points on the backbone. (4) Terminators — signal absorbers at both ends of the backbone.
📐 Bus Topology — Key Formulas (n = number of devices)
Number of Cables
n + 1
n drop lines (one per device) + 1 backbone cable. The backbone counts as a single cable.
n=10 → 10 + 1 = 11 cables
Ports per Device
1
Each device needs only 1 port — to connect its drop line. No direct device-to-device ports needed.
n=10 → 1 port per device
Total Ports
n
n devices × 1 port each = n total ports across all end devices.
n=10 → 10 total ports
Bus uses far fewer cables than Mesh (n+1 vs n×(n−1)/2) but all share a single medium — leading to collision problems.
⚖️ Bus Topology — Properties at a Glance
Reliability
Very Low ✗
Single point of failure. Backbone breaks → entire network instantly stops. No alternative path exists.
Security
None ✗
Backbone cannot filter data. A→B message travels to C and D too. Devices check the destination address themselves to decide whether to process it.
Cost
Low ✓
Significantly fewer cables than Mesh. Simple backbone + n drop lines. No expensive central device needed.
Communication Type
Multipoint
Multiple devices share the single backbone wire. This is the defining characteristic of Bus topology.
Collision Problem
High ✗
If n devices transmit simultaneously, signals collide on the shared backbone. Must use CSMA/CD or token system to manage this.
Extensibility
Yes (with Repeater) ✓
Cables have distance limits (~500m). A repeater placed at the end boosts the signal and attaches a new segment, extending range to ~1km+.
💥 The Collision Problem: Since all n devices share one wire, simultaneous transmissions result in signal collision. The two main solutions are: CSMA/CD (Carrier Sense Multiple Access / Collision Detection) — used in Ethernet — and a Token system which grants permission to one device at a time.
✅ Advantages of Bus
  • Very cheap — only n+1 cables
  • Easy to install for small networks
  • Works well for small, temporary LANs
  • Extendable with repeaters
  • Adding a device is simple (just tap the backbone)
❌ Disadvantages of Bus
  • Backbone failure = entire network down (SPOF)
  • Zero security — all devices see all traffic
  • High collision rate on shared medium
  • Performance degrades as more devices are added
  • Terminators required at both ends
RING TOPOLOGY

Ring Topology — Full Analysis

A ring topology is formed by taking the two loose ends of a bus topology and connecting them together to form a closed loop. The result is a continuous electrical circuit where every device is connected to exactly two neighbours — one on each side.

One specific device in the ring is designated as the monitor. The monitor manages the network — it ensures the ring is functioning correctly and oversees data circulation. All other devices in the ring are regular transmission devices.

🔁 Ring = Bus in a Loop: Conceptually, take a bus topology and bend it until the two terminators meet and connect. The backbone becomes a circular loop with no terminators needed. The hardware maths (cables, ports) remain the same as bus topology.
📐 Ring Topology — Key Formulas (n = number of devices)
Number of Cables
n + 1
Same as Bus: n drop lines + 1 backbone loop. The ring backbone counts as 1 cable.
n=10 → 11 cables
Ports per Device
1
Same as Bus: each device only needs 1 port to connect its drop line to the ring backbone.
n=10 → 1 port per device
Directionality
Uni
Data travels in only ONE direction around the ring. This is a key distinguishing feature of Ring topology.
A → B → C → D → E → A
Ring and Bus have identical hardware formulas (n+1 cables, 1 port per device, n total ports) — but differ in shape, directionality, and collision method.
⚖️ Ring Topology — Properties at a Glance
Reliability
Low ✗
Relies on a continuous electrical circuit. Any break in the ring wire snaps the entire circuit — no device can communicate.
Security
Low ✗
A message sent by A passes through B, C, D on its way to E. All intermediate devices can see it in transit.
Cost
Low ✓ (≈ Bus)
Same cable count as Bus (n+1). Comparable cost to Bus topology with no expensive central devices needed.
Directionality
Unidirectional
Messages travel in only one direction around the ring. There is no reverse path — data always follows the loop order.
Collision Handling
Token Ring ✓
Uses a Token Ring system. A "token" (special signal) circulates around the ring; only the device holding the token can transmit. Reduces collisions compared to Bus.
Communication Type
Multipoint
Like Bus, the ring backbone is a shared medium. All devices share the same loop circuit.
🪙 Token Ring System: A special control signal called a token continuously circulates around the ring. A device can only transmit when it holds the token. After transmitting, it passes the token to the next device. This ensures only one device transmits at a time, preventing collisions — a significant improvement over the free-for-all of Bus topology.
✅ Advantages of Ring
  • Token system greatly reduces collisions
  • Orderly, deterministic data transmission
  • Low cost — same as Bus (n+1 cables)
  • No terminators required (closed loop)
  • Easy to add devices (just break the loop and insert)
❌ Disadvantages of Ring
  • Single point of failure — any wire break = down
  • Low security — messages pass through all devices
  • Unidirectional: if one path breaks, no alternate route
  • Slower than Bus for small networks (token wait time)
  • Monitor device is a dependency / complexity

Key Exam Facts
Bus — Must Know
  • Backbone + n drop lines + taps + terminators
  • Cables: n + 1, Ports: 1 per device
  • Backbone failure = entire network down
  • Zero security — backbone cannot filter
  • Communication type: Multipoint
  • Repeater extends range beyond cable limit
Ring — Must Know
  • Bus bent into a loop — ends joined
  • Cables: n + 1, Ports: 1 per device
  • One device acts as Monitor
  • Data travels unidirectionally only
  • Collision handling: Token Ring system
  • Wire break anywhere = entire circuit broken
Bus vs Ring — Key Differences
  • Bus = open ends · Ring = closed loop
  • Bus needs terminators · Ring doesn't
  • Ring is unidirectional · Bus is bidirectional
  • Ring uses Token Ring · Bus uses CSMA/CD
  • Ring has monitor device · Bus doesn't
  • Hardware formulas (cables, ports) = identical
Collision Methods
  • CSMA/CD — Bus topology (Ethernet)
  • Detect collision → abort → backoff → retry
  • Token Ring — Ring topology
  • Token circulates; only token-holder transmits
  • Token Ring = fewer collisions than CSMA/CD

  • Ring is unidirectional — always exam asked

  • Bus backbone failure ≠ one device failure
  • Token Ring reduces but doesn't eliminate collisions
💡 Quick Memory Hooks
  • Bus = highway — everyone drives the same road
  • Ring = roundabout — traffic flows one way only
  • Tap = where device plugs into the backbone
  • Repeater = signal boost station on the highway
  • Token = permission slip to talk on the ring
