Introduction to TCP/IP

TCP/IP — also called the Internet Protocol Suite — is the actual, practical model implemented in the real-world internet. While the OSI model is invaluable for theoretically understanding how networks function, TCP/IP is what the internet truly runs on.

TCP/IP was developed by ARPANET and funded by DARPA (the US Defense Advanced Research Projects Agency). In contrast, the OSI model was developed by ISO (International Organization for Standardization). This origin story explains why TCP/IP became the real internet standard — it was built by the people who actually built the internet.

🏛 Origin: TCP/IP → ARPANET / DARPA (USA Defense).   OSI → ISO (International Standards Body). One is practical; one is theoretical. Both are essential for exams.

Architecture: 4-Layer vs 5-Layer Models

Unlike the rigid 7-layer OSI model, TCP/IP is described using either a 4-layer or 5-layer architecture depending on the textbook. Both describe the same network — the difference is only in how the bottom layer is divided.

Tanenbaum (Computer Networks) typically uses the 4-layer model. Forouzan (Data Communications and Networking) uses the 5-layer model. Both are valid for exam purposes.

Each layer in the TCP/IP model has a well-defined responsibility, its own set of protocols, and a specific PDU (Protocol Data Unit) name. Understanding these is essential for both conceptual clarity and exam questions.

Data Flow, Encapsulation & Routers

When a user sends data, it travels top-down through the TCP/IP layers on the sender's side. At each layer, a header specific to that layer is prepended to the data — this process is called encapsulation. Think of it like putting a letter into an envelope, then that envelope into a bigger one, layer by layer.

The layers behave like a stack (LIFO — Last In, First Out): the last header added on the sender side is the first one removed on the receiver side, working bottom-up during decapsulation.

Receiver does the reverse: bottom-up decapsulation — each layer strips its own header and passes the rest up.
🔀 Routers only process the bottom two layers: Network Access and Internet. They strip the frame header, read the IP packet to determine the next hop, create a new frame header, and forward. They do not touch Transport or Application data.

Supported Network Architectures

TCP/IP supports two primary types of network architecture. The choice of architecture affects how data flows, where control is centralised, and how the network scales.

Origin & Key Facts
TCP/IP vs OSI — Who Made What
  • TCP/IP developed by: ARPANET / DARPA
  • OSI developed by: ISO
  • TCP/IP = Practical (real internet)
  • OSI = Theoretical (reference model)
  • TCP/IP also called: Internet Protocol Suite
4-Layer vs 5-Layer
  • 4-Layer: Tanenbaum textbook
  • 5-Layer: Forouzan textbook
  • Difference: 5-layer splits Network Access into Physical + Data Link
  • Functionally: identical
  • OSI = 7 layers, TCP/IP = 4 or 5
Router Behaviour
  • Router uses only bottom 2 layers
  • Network Access + Internet Layer
  • Strips frame header → reads IP → creates new frame
  • Does NOT touch Transport or Application data
Encapsulation Rule
  • Sender: top → bottom (7→1 / 4→1)
  • Receiver: bottom → top (1→7 / 1→4)
  • Each layer adds its own header
  • Stack = LIFO architecture
  • Last added = first removed (decapsulation)

Network Architectures
Client-Server
  • Centralised — one main server
  • All clients connect to server
  • Server = single point of authority
  • Easy to manage, single point of failure
  • Example: company intranet, web server

  • Decentralised — no central authority
  • All peers mutually connected
  • Each device can be both client and server
  • Resilient, but harder to manage
  • Example: BitTorrent, blockchain

⚡ Last-Minute Must-Know Points

🔥 High-Yield Facts
  • TCP/IP = Internet Protocol Suite
  • DARPA (not ISO) funded TCP/IP
  • 4-layer: App + Transport + Internet + Net Access
  • App layer absorbs OSI layers 5, 6, 7
  • Net Access layer = OSI L1 + L2 combined
📌 PDU Names (Memorise)
  • Application → Data
  • Transport → Segment
  • Internet → Packet
  • Network Access → Frame / Bits
  • Same as OSI PDUs (just 4 instead of 7)
💡 Protocol Quick Map
  • App: SMTP · FTP · Telnet · DNS
  • Transport: TCP · UDP · SCTP
  • Internet: IPv4 · IPv6 · ICMP · IGMP
  • Net Access: Ethernet · Wi-Fi · ARP
  • Router = only layers 1 + 2 (Net Access + Internet)
