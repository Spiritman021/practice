Basic Identity — Router = Internet

A router is used to connect two or more different networks. The moment you hear "Router", think Internet — a collection of many different networks. While hubs, repeaters, bridges, and switches deal with LANs, routers deal with WANs (Wide Area Networks) and the Internet.

💡 Key mental model:
📡 Hub / Repeater → Physical Layer device, LAN only
📡 Bridge / Switch → Data Link Layer device, LAN only
🌐 Router → Network Layer device, WAN / Internet

Router Network Diagram — Multiple Networks

Unlike a bridge (typically 2 ports), a router can have multiple interfaces, each connecting to a different network. Each interface gets an IP address that belongs to the network it is connected to.

🌐 Router Connecting Multiple Networks — IP Packet Flow
Router interface If1 gets an IP from Network A (192.168.1.1); If2 gets an IP from Network B (10.0.0.1). Each interface IP is the default gateway for hosts on that network. Routers can connect 3, 4, or more networks simultaneously.
4

Key Functionalities — Forwarding, Flooding & Filtering

📤
Forwarding
When a packet arrives, the router looks up its Routing Table to find which interface (port) leads toward the destination IP. It then forwards the packet out of the correct interface. Example: HA sends to HB → router checks routing table → HB is reachable via If2 → packet forwarded to Network B.
✓ YES — Router CAN forward (uses Routing Table)
📢
Flooding
If the router cannot find the destination in its routing table, it sends the packet out of all interfaces (except the one it arrived on). This is similar to broadcasting — used as a last resort when the route is unknown. This is called flooding.
📢 If route unknown → flood all interfaces
🚫
Filtering — ARP Example
Routers can stop (filter) packets that should not cross network boundaries. Classic example: an ARP Request originates inside Network A to find the MAC address of the router's interface. ARP works only within a LAN. The router recognises this and does not forward the ARP packet to Network B — it filters it right there. Filtering is controlled by the routing table and protocol awareness.
✓ YES — Router CAN filter (e.g. stops ARP at boundary)
🔑 ARP Filtering explained: A host inside Network A knows the router's IP (default gateway) but does not know its MAC address. It sends an ARP Request broadcast: "Who has IP 192.168.1.1? Tell me your MAC." The router responds with its MAC — but it does not forward this ARP broadcast into Network B. ARP is scoped to the local LAN. Routers filter it at the boundary.

Routing Table & Routing Protocols

Every router maintains a Routing Table — a map of which destination networks are reachable via which interface and how far away they are. This is analogous to a bridge's MAC Address Table, but works at Layer 3 with IP addresses.

When a packet arrives, the router matches the destination IP against this table and sends the packet out of the appropriate interface. If no match → use default route or flood.

Routing tables are built and maintained using Routing Protocols:

🔗
RIP — Routing Information Protocol
A Distance Vector Routing Protocol. Each router shares its routing table with neighbours. Routes are chosen based on hop count (number of routers to cross). Simple but slow to converge; used in small networks.
Distance Vector — hop count metric — UGC NET / GATE tested
📝 For UGC NET / GATE: Know that routers use a Routing Table (like bridge uses a MAC table). The main protocol names to remember: RIP (Distance Vector) and the concept of Routing Protocols in general. Detailed protocol mechanics are covered at Network Layer depth.

Store-and-Forward, Collision Domain & Interface IP

💾
Store-and-Forward (No Collision)
A router stores each incoming packet in its buffer (memory), processes it against the routing table, then forwards it. If two packets arrive simultaneously, both are queued — no collision. This is exactly the same principle as a bridge's store-and-forward strategy.
✓ Buffer / Memory present — collisions do not occur inside a router
🌐
Router Interface IP Assignment
A router does not bring IP addresses from outside. Each router interface must be given an IP address that belongs to the network it is connected to. If Interface If1 connects to Network A (192.168.1.0/24), then If1 gets an IP like 192.168.1.1. This IP becomes the default gateway for all hosts in that network. Similarly, If2 gets an IP from Network B.
🌐 Interface IP = one address from the connected network

IDENTITY & OSI LAYER

  • Layers: L1 + L2 + L3 (Physical + DataLink + Network)
  • Type: Hardware + Software (full OS)
  • Connects: Multiple different networks
  • Scope: WAN / Internet (NOT just LAN)
  • Think "Router" → think Internet
Addresses Used
  • Primary: IP Address (Layer 3)
  • Can also read: MAC Address (Layer 2)
  • LAN: uses MAC; WAN/Internet: uses IP
  • Routing decisions are based on destination IP
Device vs Scope
  • Hub/Repeater: L1, LAN, no filtering
  • Bridge/Switch: L2, LAN, MAC table
  • Router: L3, WAN/Internet, IP routing
  • Cost: Hub < Bridge < Switch < Router (highest)
FORWARDING, FLOODING & FILTERING
📤 Forwarding
  • YES — router can forward packets
  • Uses Routing Table to find best exit interface
  • Multiple networks connected → router decides direction
📢 Flooding
  • YES — as a fallback
  • If destination NOT in routing table → flood all ports
  • Like broadcasting; used when route is unknown
🚫 Filtering (ARP example)
  • YES — router can filter/stop packets
  • ARP Request: works only within one LAN
  • Router does not forward ARP to other networks
  • Filtering is done using routing table + protocol rules
ROUTING TABLE & PROTOCOLS
📈 Routing Table
  • Router's equivalent of a Bridge's MAC table
  • Maps: Dest Network → Exit Interface → Hop Count
  • Used to decide: forward, flood, or filter
  • Directly connected nets: hop = 0
🔗 Routing Protocols
  • RIP = Routing Information Protocol
  • Type: Distance Vector Routing Protocol
  • Metric: Hop Count
  • Routers share their tables with neighbours to learn routes
  • GATE/UGC NET: know name + type + metric
🌐 Interface IP Assignment
  • Each router interface = separate network connection
  • Interface gets one IP from that connected network
  • If1 → Net A → IP = 192.168.1.1 (default gateway for Net A)
  • If2 → Net B → IP = 10.0.0.1 (default gateway for Net B)
  • Router does NOT import IP from elsewhere
STORE-AND-FORWARD & COLLISION
💾 Store-and-Forward
  • Router stores packet in buffer first
  • Processes routing table → decides direction
  • Simultaneous arrivals → queued, not crashed
  • Result: NO collision inside router
💥 Collision Domain
  • Hub: collision = N (all devices)
  • Bridge/Router: no collision internally
  • Both use store-and-forward buffer
  • Router separates broadcast domains too (unlike Bridge)
