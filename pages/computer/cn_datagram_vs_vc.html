Recap — Two Types of Packet Switching

As covered in Topic 18, packet switching divides data into small packets and sends them over the network. But this can be done in two fundamentally different ways:

🌐 Datagram Switching

📌 Connectionless — no setup before sending
📌 No resource reservation in the network
📌 Each packet routed independently
📌 Packets may take different paths
📌 Works at Network Layer (L3)
📌 Used by: Internet (IP), IP network

🔂 Virtual Circuit

📌 Connection-oriented — reservation before sending
📌 Resources reserved along the path (CPU, Buffer, BW)
📌 All packets follow the same fixed path
📌 Mix of circuit + packet switching (50%+50%)
📌 Works at Data Link Layer (L2)
📌 Used by: ATM, Frame Relay, X.25
📌 Virtual Circuit = 50% Packet + 50% Circuit: It still divides data into packets (packet switching part). But before sending, it reserves resources along the path (circuit switching part). This is why it is called a hybrid approach.
2

Connectionless vs Connection-Oriented

This is the most fundamental difference between the two. In Datagram, A sends packets directly to B with no prior setup — everything is on demand. In Virtual Circuit, a global packet (setup packet) is first sent from A to B, which reserves all resources (Buffer, CPU, Bandwidth) at every switch along the path.

🔌 Datagram (Connectionless) vs Virtual Circuit (Connection-Oriented)
🌐 DATAGRAM — On Demand
🔂 VIRTUAL CIRCUIT — Fixed Reserved Path
In Datagram, P1/P2/P3 explore different routes based on routing tables. In Virtual Circuit, a global packet first reserves one path, and then all data packets follow that exact path.
3

Property-by-Property Deep Dive

🔌
1. Connection Setup
🌐 Datagram Connectionless — No setup. Packets go directly. No reservation needed.
🔂 Virt. Circ. Connection-Oriented — A global packet first reserves path. Setup phase required.
💾
2. Resource Reservation
🌐 Datagram On Demand — No prior reservation. Buffer, CPU, Bandwidth allocated as packet arrives. If busy, packet waits.
🔂 Virt. Circ. Pre-reserved — Buffer, CPU, Bandwidth reserved along path before any data packet is sent.
📏
3. Routing / Path
🌐 Datagram Independent — Each packet uses routing tables and may follow a different path. Better traffic management; all paths explored.
🔂 Virt. Circ. Fixed path — The global packet establishes one route. All data packets follow this same route.
🔁
4. Order of Delivery
🌐 Datagram Out-of-Order possible — Packets take different paths with different lengths → P4 may arrive before P1.
🔂 Virt. Circ. Always In-Order — Fixed path means P1 → P2 → P3 always arrive sequentially.
📜
5. Header Overhead
🌐 Datagram HIGH overhead — Every single packet must carry a full header (source IP, dest IP, TTL, fragmentation…) because packets travel independently.
🔂 Virt. Circ. LOWER overhead — First packet carries the global/full header. Subsequent data packets carry only a smaller local header (VCI label).
💥
6. Packet Loss
🌐 Datagram More packet loss — Packets on different paths have higher chance of getting lost in transit (different routes, different congestion levels).
🔂 Virt. Circ. Less packet loss — Known, fixed route with reserved resources → fewer chances of drops.
✅
7. Efficiency
🌐 Datagram HIGH efficiency — On-demand resources. Idle = resources freed for others. No wasted reservation.
🔂 Virt. Circ. LOWER efficiency — Reserved resources locked even during idle moments, like a pre-booked restaurant table you don't use.
💰
8. Cost
🌐 Datagram Less cost — No reservation overhead. Resources used only when needed. Better value for money.
🔂 Virt. Circ. More cost — Every session requires a new connection setup + reservation. Repeated overhead cost per session.
⏳
9. Delay
🌐 Datagram More delay — On-demand: if switch is busy, packet waits in buffer. Like walking into a full restaurant and waiting for a table.
🔂 Virt. Circ. Less delay — Resources pre-reserved: packets don't wait at switches. Like sitting straight at a pre-booked table.
🌐
10. Real-World Usage
🌐 Datagram Internet (IP) — All IP networking relies on datagram service. Email, web, WhatsApp, streaming all use IP datagrams.
🔂 Virt. Circ. ATM, Frame Relay, X.25 — Older/specialised networks. ATM = Asynchronous Transfer Mode (NOT Automated Teller Machine).

Header Overhead — Why Datagram Has More

In packet switching, a header is like the envelope on a courier package — it carries addressing and routing information. The header is extra data that must be sent alongside the actual payload, adding overhead.

In Datagram switching, since each packet may follow a completely different route, every single packet must independently carry its full header (source IP, destination IP, TTL, fragmentation info, sequence number, etc.) so that each intermediate router knows what to do with it.

In Virtual Circuit, only the first global packet (setup packet) carries the full header to establish the path. Subsequent data packets carry only a small VCI (Virtual Circuit Identifier) label — much smaller than a full IP header.

📌 What's in a header? Source Address, Destination Address, TTL (Time To Live), Fragment Offset, Sequence Number, Protocol type, Checksum — all required in datagram. In VC, once the path is set up, a tiny VCI label is enough since all switches already know the path.

Restaurant Analogy — Delay, Efficiency & Cost

The restaurant analogy beautifully captures three properties at once:

🍽 Restaurant Analogy — Datagram vs Virtual Circuit
🌐 Datagram = Walk-in
📌 You walk into the restaurant without a booking.
📌 Tables are used on demand — only when you arrive.
📌 You might have to wait if all tables are full → more delay.
📌 Tables never sit empty reserved → high efficiency.
📌 No booking cost → less cost.
🔂 Virtual Circuit = Pre-booked
📌 You book a table at 5 PM in the morning.
📌 Resources pre-reserved → you sit down straight away.
📌 No waiting → less delay.
📌 Table may sit empty from 5–7 PM if you're late → low efficiency.
📌 Booking cost every time → more cost.
The same analogy from circuit/packet switching applies here — reservation (VC) gives faster access but wastes resources; on-demand (datagram) is cheaper and more efficient but may have wait times.

🎯 Golden Summary:
• Datagram: Connectionless, on-demand, different paths, out-of-order, more overhead & loss, high efficiency, low cost, more delay — powers the Internet.
• Virtual Circuit: Connection-oriented, pre-reserved, same path, in-order, lower overhead & loss, lower efficiency, more cost, less delay — used in ATM, Frame Relay.
⚡ Revision Mode — Flash-card summaries for GATE, UGC NET & PSU exam quick prep.
DATAGRAM (CONNECTIONLESS)
🌐 Datagram — Identity
  • Type: Connectionless
  • Layer: Network Layer (L3)
  • No setup packet — data goes directly
  • Resources allocated on demand
  • Examples: Internet (IP), IP network
🌐 Datagram — Disadvantages
  • Packets may arrive out of order
  • HIGH header overhead — full header on every packet
  • More packet loss — different paths, unpredictable
  • More delay — may wait if switch is busy
🌐 Datagram — Advantages
  • HIGH efficiency — on-demand, no idle waste
  • Less cost — no reservation overhead
  • Better traffic management — all paths explored
  • Dominant in today's networks — powers the Internet
VIRTUAL CIRCUIT (CONNECTION-ORIENTED)
🔂 Virtual Circuit — Identity
  • Type: Connection-Oriented
  • Layer: Data Link Layer (L2)
  • Global packet → reserves path before data
  • Mix: 50% packet switching + 50% circuit switching
  • Examples: ATM, Frame Relay, X.25
🔂 Virtual Circuit — Advantages
  • Packets always arrive in order
  • Less packet loss — fixed known path
  • Less delay — resources pre-reserved, no waiting
  • Lower header overhead — only VCI label on data packets
🔂 Virtual Circuit — Disadvantages
  • Lower efficiency — reserved resources may sit idle
  • More cost — setup cost per session
  • Less used in modern networks
  • ATM = Asynchronous Transfer Mode (not ATM machine!)
