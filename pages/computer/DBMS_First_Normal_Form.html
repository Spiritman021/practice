First Normal Form
1NF in DBMS

First Normal Form says a table must not contain multivalued attributes. Every cell should hold a single atomic value.

E. F. Codd RuleNo Multivalued Attribute Composite KeyForeign Key
1

Rule of First Normal Form

According to E. F. Codd, the father of DBMS, a table is in First Normal Form when it does not contain any multivalued attribute.

1NF rule: For a single record, one attribute should not contain more than one value. Each cell should be atomic.
2

Table Not in 1NF

A student may enroll in more than one course, but storing multiple courses inside one cell is the wrong representation.

Three Ways to Convert into 1NF

Method 1: Repeat Rows
Split each multivalued course into separate rows. Primary Key becomes Roll No + Course.
Method 2: Add Columns
Create Course1, Course2, etc. Roll No can be Primary Key, but many NULL values may appear.
Method 3: Split Tables
Create a base student table and a referencing course table. This is the better representation.
