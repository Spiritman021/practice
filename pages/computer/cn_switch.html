Basic Identity & OSI Layer

A switch is one of the most important and widely used networking devices in modern LANs. It operates at Layer 2 — the Data Link Layer. This means it understands both the Physical Layer (signals and bits) and the Data Link Layer (MAC addresses and frames).

A switch is fundamentally a multiport bridge. A bridge connects two LANs using two ports; a switch does the same job but with many ports — which is why it is called a multiport bridge. You have likely seen a switch in a computer lab or office: it is the box with many Ethernet ports that connects all devices in a room.

Switch vs Bridge — Why Switch is Called a Multiport Bridge

Both bridge and switch are Layer 2 devices. The key difference is the number of ports. A bridge has exactly two ports and is used to connect two LANs. A switch has many ports (8, 24, 48, 52+) and can connect many devices — computers, laptops, printers, wireless access points — all in one.

🔃 Bridge

📌 2 ports only — Port 1 and Port 2
📌 Connects LAN1 (Port 1) to LAN2 (Port 2)
📌 Layer 2 device — reads MAC addresses
📌 Has MAC address table (forwarding table)
📌 Connects only two network segments
📌 Used to separate two LANs / reduce traffic

💠 Switch (Multiport Bridge)

📌 Multiple ports (8, 24, 48, 52+ ports)
📌 Connects many devices: PC, laptop, printer, wireless AP
📌 Layer 2 device — reads MAC addresses
📌 Has MAC address table / CAM table per port
📌 Used as the central device in a LAN
📌 Full duplex — no collisions between any ports

3

Role of Switch in a Real Network

In a real network, devices do not connect directly to the router. Instead, all devices in a LAN connect to a switch, and the switch connects to the router. This is how internet access works in a computer lab, office, or home.

🌐 Real-World Network Topology — Switch as Central Hub of LAN
In a computer lab or office: Laptop connects via blue Ethernet cable → Switch → Router → Internet. The switch is the central device of the LAN.
📱 Lab Scenario: You enter a computer lab, plug your Ethernet cable into the lab switch. The switch connects to the router. The router connects to the internet. This is the standard LAN architecture used everywhere — switch sits between the devices and the router.
4

Full Duplex Links & Zero Collision Domain

Every link between a device and a switch is a full-duplex link. This means data can flow in both directions simultaneously on the same wire — a device can send and receive at the same time without waiting.

Because of full-duplex operation, there is no collision on any port. When A sends to the switch and the switch sends back to A, both happen at once — they use different pairs of wires (TX and RX) so they never interfere. The switch also creates a separate, dedicated circuit for each active conversation (e.g., A→B and C→D happen independently and simultaneously).

✅ Key Exam Fact: The number of collision domains in a switch = number of ports. Each port is its own collision domain. Because links are full-duplex, the actual number of collisions = 0 (zero). Switch completely eliminates collisions.

MAC Address Table & Unicast Forwarding

A switch works on Physical Addresses (MAC Addresses). It maintains a MAC Address Table (also called a CAM table — Content Addressable Memory table) that maps each MAC address to the port it is connected to.

When a frame arrives, the switch checks the destination MAC address and looks it up in its MAC table. It then forwards the frame only to the port where the destination device resides. Devices on other ports do not receive the frame. This is called unicast forwarding — sending to one specific destination.

📜 Switch MAC Address Table (CAM Table)
MAC Address Port Device
AA:BB:CC:DD:01:01 Port 1 Laptop A
AA:BB:CC:DD:02:02 Port 2 Computer B
AA:BB:CC:DD:03:03 Port 3 Printer C
AA:BB:CC:DD:04:04 Port 4 Wireless AP D
💡 How it works: When A sends a frame to C, the switch reads destination MAC AA:BB:CC:DD:03:03, finds Port 3 in the MAC table, and sends the frame only to Port 3. Devices A, B, and D never receive this frame — this keeps traffic minimal.
📤
Forwarding (Unicast)
A switch forwards frames only to the intended destination port based on the destination MAC address in its table. This is unicast forwarding — unlike a hub which broadcasts to everyone, the switch sends to one specific port.
✓ YES — Forwards to correct port only
🚫
Filtering
A switch can filter. Because it knows which device is on which port (via the MAC table), it suppresses the frame on all other ports. Only the destination port receives the frame — all others are filtered out.
✓ YES — Switch CAN filter
📢
Broadcasting (Special Case)
A switch does broadcast in special cases: (1) When the destination MAC is unknown (not yet in the MAC table), it floods the frame to all ports to learn the destination. (2) For broadcast frames (destination = FF:FF:FF:FF:FF:FF), it forwards to all ports. But for known unicast traffic, it does not broadcast.
📢 Broadcasts only for unknown MACs / ARP
6

Traffic — Switch vs Hub

Traffic in a switch-based network is minimal compared to a hub. In a hub, every frame goes to every device — creating enormous unnecessary traffic. In a switch, only the destination device receives the frame.

📢 Traffic Comparison — Hub (Broadcast) vs Switch (Unicast)
💥 HUB — A sends, ALL receive
✅ SWITCH — A sends, only B receives
Hub wastes bandwidth sending to all devices. Switch sends only to the intended recipient, drastically reducing network traffic.
📊 Traffic comparison: With N devices on a hub, each frame consumes (N−1) unnecessary bandwidth slots. With a switch, each frame uses exactly 1 bandwidth slot — only to the destination. This is why switches replaced hubs in all modern networks.
7

Default Layer 2 & Layer 3 Smart Switches

By default, a switch is a Layer 2 device. It only understands MAC addresses and operates at the Data Link Layer. However, modern switches can be upgraded to Layer 3 (smart switches) — they gain IP routing capabilities, effectively combining switch and router functionality.

TypeOSI LayerAddress UsedFunctionAlso Called
Standard Switch Layer 2 MAC Address Forwards frames within a LAN using MAC table Multiport Bridge
Smart / Layer 3 Switch Layer 3 MAC + IP Address Can route packets between different networks (like a router) Multilayer Switch
Router Layer 3 IP Address Routes packets between different networks Network Layer Device
💡 Exam Tip: When a question says "switch", always assume Layer 2 by default unless "Layer 3 switch" or "multilayer switch" is explicitly mentioned. The router is the standard Layer 3 network device.
