Topic 6 — Key UNIX Commands

6A

chmod — Change Mode (File Permissions)

chmod (Change Mode) is used to set or modify the read, write, and execute permissions on a file or directory for three categories of users.

👥 Three Categories & Three Permission Types
📁 Three User Categories
U — User (Owner)
The person who created the file. Has the most control over it.
G — Group
Users who belong to the same group as the owner (e.g., same department).
O — Others
Everyone else on the system who is neither the owner nor in the group.
🔑 Three Permission Types
r — Read (4)
Can open and read the file contents. Octal value = 4.
w — Write (2)
Can modify or delete the file contents. Octal value = 2.
x — Execute (1)
Can run the file as a program/script. Octal value = 1.
Total 9 permission bits = 3 categories × 3 permission types. Always written in the order: U-G-O, and within each: r-w-x.

Always read left to right: first 3 = User, middle 3 = Group, last 3 = Others. Within each group: r (read), w (write), x (execute).
6B

Symbolic Notation — chmod with Letters

📝 chmod Symbolic Notation — Syntax & Examples
/* Syntax: chmod [who][operator][permission] filename */
who = u (user) | g (group) | o (others) | a (all = ugo)
operator = + (add) | - (remove) | = (set exactly)
permission= r (read) | w (write) | x (execute)
CommandWhat It DoesExplanation
chmod ugo+r note Give read permission to User, Group, Others on file "note" ugo = all three categories, +r = add read. This is the answer to "assign only read to all three categories".
chmod a=r note Set ONLY read for all (removes write & execute too) a = all (same as ugo), =r means set EXACTLY to read only — strips write and execute.
chmod u+rwx note Give User all permissions (read + write + execute) User gets full access. Group and Others not affected.
chmod go+rw note Give read & write to Group and Others go = group and others only (not user), +rw = add read and write.
chmod o-x note Remove execute from Others o = others, -x = take away execute permission. The minus sign removes.
chmod a-rw note Remove read & write from ALL a = all, -rw = remove read and write. A command that restricts access.
chmod ugo+rw note Give read & write to User, Group, Others Equivalent to octal 666. All three get read and write but NOT execute.
NOTE: Multiple categories and permissions can be combined in one command. "ugo+rw" and "a+rw" are equivalent.

chmod 000 file --- (0)--- (0)--- (0) No permissions for anyone (not even owner without sudo).
The two most exam-asked: 444 = read-only for all three, 666 = read+write for all three, 777 = all permissions for all three.
📝 Exam Questions from Lecture — Solved:

Q1: "Which command assigns ONLY read to all three categories of file 'note'?"
→ chmod ugo+r note (symbolic)  OR  chmod 444 note (octal) ✅

Q2: "Represent chmod ugo+rw in octal notation."
→ rw = 4+2 = 6. Applied to U, G, O → chmod 666 note ✅

Why not "o-x" in Q1? The minus sign removes permission. If the question says "assign read", you must add (+), not subtract. "o-x" removes execute from others — that's a different operation.
6D

lseek — Random Access in Files

lseek is a system call (not just a command) used to move the read/write head to a specific position within an open file. By default, the head starts at position 0 (the very beginning of the file).

📌 Purpose: Sequential access reads byte-by-byte from start. lseek enables random access — jumping to any position in the file instantly, like a seek head on a disk. Use it when you don't want to read from the beginning every time.
📄 lseek — Three Modes (SEEK_CUR, SEEK_SET, SEEK_END)
SEEK_CUR
Move relative to the current position.
+n = move n bytes forward.
-n = move n bytes backward.
SEEK_SET
Move to an absolute position from the beginning of file.
lseek(n, 5, SEEK_SET) → position 5.
SEEK_END
Move relative to the end of file.
Usually used with a negative offset to go backwards from end.
/* lseek Syntax */
lseek(file_descriptor, offset, whence);

/* Parameters */
file_descriptor → integer returned by open() system call (e.g., n)
offset → number of bytes to move
whence → SEEK_CUR | SEEK_SET | SEEK_END
lseek() returns the resulting offset (byte position from beginning of file) on success, or -1 on error.

🎯 Order matters with lseek!

Always solve step by step in the given order.

Topic 7 — System Calls

7A

User Mode vs Kernel Mode

Modern CPUs operate in two privilege levels. The OS kernel runs in kernel mode (privileged), while user applications run in user mode (restricted). To access hardware or OS resources, user programs must cross into kernel mode via a system call.

🖥️ User Mode vs Kernel Mode — Architecture
👤

User Mode (Unprivileged)

Applications, user programs, scripts run here. Cannot directly access hardware, files on disk, or OS internals. Examples: writing a C program, running Firefox, playing a game.

System Call (crosses the boundary)
⚙️

Kernel Mode (Privileged)

OS kernel runs here. Has direct access to all hardware (CPU, RAM, disk, devices). Executes privileged instructions. Manages all resources.

Example 1: printf("Hello") in C → printf() calls write() system call → crosses to kernel mode → OS sends text to monitor → returns to user mode.
Example 2: Open a file → open() system call → kernel checks permissions, locates file on disk → returns file descriptor to user space.
Every interaction with hardware, files, or OS services MUST go through a system call. APIs like printf() are wrappers that internally invoke system calls.
7B

What Is a System Call?

📌 Definition: A system call is the programmatic interface through which a user-mode application requests a service from the OS kernel. It is the only legal way to transition from user mode to kernel mode. Every OS operation — file access, device I/O, process creation — happens through a system call.

⚠️ printf() is NOT a system call. It is a C library function. But it internally calls the write() system call to output text to the monitor. Similarly, scanf() → read(), fopen() → open(), fclose() → close(). The library functions are user-friendly wrappers around raw system calls.
Linux exposes ~300 system calls. Windows 7 has ~700 system calls. You never call most directly — APIs handle that for you.
7C

6 Categories of System Calls

1

File-Related System Calls

Create, open, read, write, close, delete files

Used whenever a process wants to perform any operation on a file stored on the hard disk (secondary memory). All file I/O goes through these calls.

open(path, flags)
Opens a file. Returns a file descriptor (integer) used in subsequent calls.
read(fd, buffer, count)
Reads 'count' bytes from file descriptor fd into buffer.
write(fd, buffer, count)
Writes 'count' bytes from buffer to file descriptor fd.
close(fd)
Closes the file descriptor. Frees OS resources for that file.
lseek(fd, offset, whence)
Repositions read/write head. Used for random access. (SEEK_CUR, SEEK_SET, SEEK_END)
unlink(path)
Removes (deletes) a file from the filesystem.
2

Device-Related System Calls

Access, control, and manage hardware devices

Used to interact with hardware devices — hard disk, printer, monitor, keyboard. Users cannot access devices directly; all device interactions go through the OS kernel via these calls.

read() / write()
Read data from a device OR write data to it (same calls as file, different fd targets).
ioctl(fd, request, ...)
I/O Control — configure device parameters, set baud rate, screen size, etc. Very versatile.
lseek()
Reposition within a device that supports seeking (e.g., disk drives, tape drives).
Get/Set attributes
Read or modify a device's attributes (device type, speed, buffer size, etc.).
📌 ioctl() is the most powerful and flexible device system call. It handles all device-specific operations that don't fit neatly into read/write. Examples: getting terminal window size, configuring serial port settings, controlling network interface cards.
3

Information-Related System Calls

Get metadata about processes, files, system

Used to query metadata — information about data. For example, a file's size, creation date, permissions (its attributes), or a process's ID, state, and priority.

getpid()
Returns the Process ID (PID) of the currently running process.
getppid()
Returns the Parent Process ID — the PID of the process that created this one (fork parent).
time() / gettimeofday()
Get current system time and date. Used to timestamp events.
stat(path, buf)
Get detailed file attributes: size, permissions, owner, modification time, inode number.
4

Process Control System Calls

Create, manage, and terminate processes

Used to manage the entire lifecycle of processes — loading, executing, creating child processes, waiting, and terminating. These are the most fundamental system calls in OS.

fork()
Creates a child process — an identical copy of the calling (parent) process. Both parent and child run concurrently. Used for multi-processing.
exec()
Replaces the current process image with a new program. Used after fork() to run a different program in the child.
wait()
Parent process waits for child to finish. Used for synchronization between parent and child.
exit() / abort()
Terminate a process normally (exit) or abnormally (abort). Frees all resources.
fork() — How it creates multiprocessing:
Parent calls fork() → OS creates child process (exact copy of parent). fork() returns 0 to child, child PID to parent. Both run independently. Child does its work; parent does its work simultaneously. This is how servers handle multiple client requests.
5

Communication System Calls (IPC)

Inter-process communication — pipe, shared memory, sockets

Used for Inter-Process Communication (IPC) — allowing two or more processes to exchange data. Two main IPC mechanisms: Pipes (unidirectional byte stream) and Shared Memory (fastest IPC method).

pipe(fd[2])
Creates a pipe — a unidirectional data channel. Data written to fd[1] can be read from fd[0]. Used between parent and child processes.
shmget()
Get/create a shared memory segment. Multiple processes can map it into their address space and read/write directly. Fastest IPC.
socket() / connect()
Create network connections. Used for communication between processes on different machines (client-server communication).
send() / recv()
Send and receive messages through a socket or message queue. Used in network programming.
6

Protection & Security System Calls

chmod, umask, access control, privileges

Used to manage access control and security — who can access what, with which permissions. These prevent unauthorised access and enforce the OS's security model.

chmod(path, mode) umask(mask) chown(path, uid, gid) access(path, mode) setuid(uid) getuid()
chmod(path, mode) — the system call behind the chmod UNIX command. Changes file permission bits.
umask(mask) — sets default permission mask for newly created files (subtracts permissions from default).
chown() — change file owner and group.
access() — check if the process has permission to access a file.

lseek System Call
Purpose
Move the read/write head to a specific position in an open file. Enables random access (not just sequential).
Default position
0 (index 0 = beginning of file) when a file is first opened.
Syntax
lseek(fd, offset, whence) → returns new position from start of file.
SEEK_CUR
Move RELATIVE to current position. +n = forward, –n = backward. New pos = current + offset.
SEEK_SET
Move to ABSOLUTE position from beginning. Ignores current position. New pos = offset exactly.
SEEK_END
Move relative to END of file. Usually used with negative offset (–n = move n bytes from end).
Exam question
lseek(n,10,SEEK_CUR) then lseek(n,5,SEEK_SET) → final position = 5. SEEK_SET always sets absolute.
Order matters
SEEK_SET first (pos=5), then SEEK_CUR +10 → final = 15. Always solve step by step in given order.

Programmatic interface to switch from user mode to kernel mode. The only legal way to access OS/hardware services.
User mode
Applications run here. Cannot directly access hardware, files on disk, or OS internals.
Kernel mode
OS kernel runs here. Has full hardware access. Executes privileged instructions.

printf() is a C library function. It internally invokes the write() system call. APIs are user-friendly wrappers.
Linux system calls
