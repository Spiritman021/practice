STOP AND WAIT ARQ

Send one frame, stop, wait for acknowledgement, then send the next frame. This lecture covers ACK meaning, timeout, retransmission, sequence numbers 0 and 1, and the lost-frame / lost-ACK cases.

GATEUGC NETData Link LayerWindow Size 1
1

Basic Idea: Send One, Then Wait

In Stop and Wait, the sender transmits only one frame at a time. After sending that frame, the sender stops and waits until an ACK returns from the receiver. Only after the ACK arrives does the sender transmit the next frame.

🔁 Normal Stop-and-Wait Flow
ACK number means the next expected frame, not the same frame number.
ACK meaning: An acknowledgement tells the sender that the frame reached successfully, so the next frame can be sent.
2

Window Size and Sequence Numbers

In Stop and Wait, both sides handle only one frame at a time. Therefore, the sender window size = 1 and the receiver window size = 1. Because the total movement is small, only two sequence numbers are required: 0 and 1.

Sender Window

How many frames can be sent without waiting for ACK?

1 frame

Receiver Window

How many frames can be accepted at a time?

1 frame

Sequence Numbers

Frames alternate between sequence number 0 and 1.

0, 1
Most asked point: Stop and Wait uses only two sequence numbers because sender window size and receiver window size are both 1.
3

Timeout and Retransmission

If the ACK does not arrive, the sender cannot know whether the original frame was lost or the ACK was lost. So it waits for a fixed timeout time. If the timer expires, the sender retransmits the same frame.

⏱ Lost Frame Case
If a frame is lost, receiver sends no ACK. Sender waits until timeout, then resends the same frame.
Timeout idea: It is chosen long enough to cover the frame going to the receiver and ACK coming back, plus practical delay margin.
4

Lost ACK and Duplicate Frame

Sometimes the frame reaches the receiver successfully, but the ACK is lost. The sender times out and sends the same frame again. The receiver uses the sequence number to detect that this is a duplicate frame, discards it, and sends the ACK again.

🔄 Lost ACK Case
Sequence numbers protect the receiver from accepting the same frame twice.
Important: If receiver is expecting frame 1 and receives frame 0 again, it understands that the ACK for frame 0 was probably lost. It discards duplicate frame 0 and resends ACK 1.
5

ACK Number Means Next Expected Frame

In Stop and Wait, the receiver does not send ACK for the same frame number. It sends ACK for the next expected frame. If frame 0 is accepted, the receiver sends ACK 1. If frame 1 is accepted, it sends ACK 0.

✅ Last-Minute Summary

  • Stop and Wait = send one frame, then wait peacefully for ACK.
  • ACK confirms successful frame delivery and permits the next frame.
  • Sequence numbers are needed because frames or ACKs can be lost.
  • Only 0 and 1 are required because both windows are size 1.
  • ACK always indicates the next expected frame.
  • Timeout expiry means retransmit the same frame.
