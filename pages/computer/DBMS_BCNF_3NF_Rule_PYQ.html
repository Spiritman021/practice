Rule Based PYQ

A rule-first explanation of the GATE 2018 normalization question: use candidate keys, prime attributes, and dependency checks to identify the schema in 3NF but not BCNF.

1

Core Rule Difference

The question asks which relation is in 3NF but not BCNF. So first lock the rule difference in mind.

BCNF Rule

For every FD X → Y, the left side X must be a candidate key or super key. RHS does not save the dependency.

3NF Rule

For every FD X → Y, either X is a key/super key, or Y is a prime attribute.

A relation becomes “3NF but not BCNF” when an FD has non-key LHS, but its RHS is prime.
