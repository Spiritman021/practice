What is a Super Key?

A Super Key is any attribute, or combination of attributes, that can uniquely identify each tuple in a table. It may contain extra attributes, but it must contain enough information to uniquely identify rows.

Core rule: Every Candidate Key is a Super Key, but every Super Key is not necessarily a Candidate Key.
2

Student Table Example

Suppose a Student table has three attributes: Roll_No, Name, and Age. If Roll_No is a Candidate Key, then it can uniquely identify every student. Any set that contains Roll_No also becomes a Super Key.

Candidate Key
A minimal unique identifier. No extra attribute is present. Example: {Roll_No}.
Super Key
A unique identifier that may include extra attributes. Example: {Roll_No, Name}.

Counting Super Keys

In exam numericals, relation R(A1, A2, A3, ..., An) is often given with one or more Candidate Keys. To count Super Keys, keep the Candidate Key fixed and choose any combination of the remaining attributes.

Case 1: A1 is Candidate Key

A1 must appear in every Super Key. Remaining attributes are n - 1, each with two choices: take or leave.

2n-1

Example sets: {A1}, {A1,A2}, {A1,A3}, {A1,A2,A3}.

Case 2: A1 and A2 are separate Candidate Keys

Count Super Keys from A1 and from A2, then subtract the common sets containing both.

2n-1 + 2n-1 - 2n-2

The subtraction avoids double-counting sets like {A1,A2}.

Case 3: Composite Candidate Key A1A2

If {A1,A2} together is one Candidate Key, both are fixed in every Super Key made from it.

2n-2

Remaining attributes are n - 2, so each can be included or excluded.

Case 4: A1A2 and A3A4 are Candidate Keys

Both composite keys produce Super Keys. Subtract common sets where all four fixed attributes appear.

2n-2 + 2n-2 - 2n-4

Common part contains A1,A2,A3,A4, so four attributes are fixed.

How to Solve Numericals

Super Key Counting Method
1
Identify all Candidate Keys given in the question.
2
For each Candidate Key, fix all attributes inside that key.
3
For the remaining attributes, use power-set choices: each attribute can be selected or not selected.
4
If multiple Candidate Keys exist, add their counts and subtract common duplicate Super Keys.
Final memory line: Super Key = Candidate Key + zero or more extra attributes.
1
Core Concept
Super Key:Any set of attributes that uniquely identifies tuples in a table.
Candidate Key:Minimal Super Key. It has no unnecessary extra attribute.
Memory:Candidate Key is always Super Key; Super Key may not be Candidate Key.
Shortcut:Super Key = Candidate Key plus any combination of remaining attributes.
