Basic Identity & OSI Layer

A hub is one of the most fundamental networking devices. It is pure hardware — no software of any kind — and operates exclusively at the Physical Layer (Layer 1). Its job is simply to receive a signal on one port and repeat it out of every other port.

💡 One-liner definition: A Hub = a multiport repeater. Everything a repeater does (regenerate and forward signals at Layer 1), a hub does — but across many ports instead of just two.
2

Hub vs Repeater — What Makes a Hub Different

Both devices are pure Layer 1 hardware with no software. The key distinction is the number of ports and a small set of extra diagnostic features that hubs provide.

🔁 Repeater

📌 2 ports only — connects two cable segments
📌 Regenerates signal from Port 1 → Port 2
📌 No diagnostics — if a wire is cut, it gives no indication
📌 Used to extend a single cable run
📌 Very simple; bare-minimum physical device

💠 Hub (Multiport Repeater)

📌 Multiple ports (4, 8, 16, 24, 26…)
📌 Repeats signal from any port out to all other ports
📌 Has LED indicator lights per port — blinks when active, shows if a wire is broken or disconnected
📌 Used in star-topology LANs to connect multiple devices
📌 More capable diagnostically but same logic as a repeater

Key Functionalities — Forwarding, Filtering & Broadcasting

📤
Forwarding
A hub always forwards data. When any device sends a signal into the hub on any port, the hub regenerates that signal and sends it outward. This is unconditional and automatic — no decision is made about where to send it.
✓ YES — Hub CAN forward
🚫
Filtering
A hub cannot filter. It has no software, no MAC address table, and no awareness of which device lives on which port. It is physically impossible for a hub to decide "this frame is only for Device B, so I will only send it to Port 2." It has no such knowledge.
✕ NO — Hub CANNOT filter
📢
Broadcasting (Consequence of No Filtering)
Because a hub cannot filter, it broadcasts every frame to every port. If Device A sends a message meant only for Device B, the hub also sends it to Devices C, D, and all others connected. Every device receives every frame — this is highly inefficient and creates excessive traffic.
📢 Broadcasts to ALL ports, always
📢 Hub Broadcast Behaviour — A sends to B, but everyone receives
Device A wants to send data only to Device B
Device A intended to send to Device B only. But the hub — having no MAC table — broadcasts the frame to all ports. Devices C and D receive data not meant for them.
4

Traffic, Collisions & Collision Domain

Because every message is broadcast to every port, a hub generates very high network traffic. Every device receives every frame whether it needs it or not. This also means collisions are common and frequent.

💥
Collision Domain = N (total connected devices)
All devices connected to a hub belong to a single collision domain. The maximum number of possible collisions equals N (the total number of devices connected). If multiple devices transmit at the exact same time, their signals collide inside the hub — because the hub has no buffer or memory to queue packets.
⚠ Collision Domain = N — one large shared domain
📊 Why is traffic so high? Imagine 8 devices on a hub. Device A sends 1 frame. All 7 other devices receive it. For every frame sent, 7 unnecessary copies travel the network. With N devices, each frame consumes (N−1) unnecessary bandwidth slots. This is why hubs were replaced by switches.

Cost Comparison — Networking Devices

Hubs and repeaters are the cheapest networking devices because they are pure hardware with no software. More intelligent devices (Bridge, Switch, Router) cost more because they have embedded software (firmware/OS) that handles MAC tables, routing tables, and filtering logic.

🔑 Rule of thumb: The higher the OSI layer a device operates at, the more software it has, and the more expensive it is. Hub/Repeater (L1) < Bridge/Switch (L2) < Router (L3).

IDENTITY & BASICS
OSI Layer & Type
  • Layer: Physical Layer (L1) ONLY
  • Type: Pure Hardware — zero software
  • Also called: Multiport Repeater
  • No MAC or IP address awareness
  • Cost: Lowest (no software = cheap)
Hub vs Repeater
  • Repeater: 2 ports only
  • Hub: 4 / 8 / 16 / 24 / 26+ ports
  • Hub adds LED indicator lights per port
  • Hub: broken wire → LED shows fault; repeater: no indication
  • Both: same Layer 1 logic, no filtering, no software
FORWARDING, FILTERING & BROADCASTING
📤 Forwarding
  • YES — always and unconditionally
  • Receives signal on one port → repeats out all others
  • No decision made about destination
🚫 Filtering
  • NO — cannot filter any traffic
  • Has no MAC table, no port-device mapping
  • Cannot distinguish between intended and unintended recipients
📢 Broadcasting
  • Always broadcasts to every port
  • A → B sends data? Hub delivers to B, C, D … everyone
  • Creates very high traffic on the network
  • Direct consequence of no filtering ability
COLLISION DOMAIN & COST
💥 Collision Domain
  • All N devices = ONE shared collision domain
  • Max collisions = N (all devices transmit simultaneously)
  • No buffer/memory → signals literally crash into each other
  • Switch solves this: 1 collision domain per port
💰 Cost Ranking (Cheap → Expensive)
  • Repeater / Hub → cheapest (pure HW)
  • Bridge → medium (HW + SW)
  • Switch → medium-high (HW + SW)
  • Router → most expensive (HW + full OS)
ONE-LINERS FOR EXAM
Must-Know Facts
  • Hub = Multiport Repeater
  • Hub operates at L1 Physical Layer
  • Hub can forward data
  • Hub cannot filter data
  • Hub broadcasts to all ports
  • Collision domain = N
  • Hub cheaper than Bridge/Switch/Router
Comparison at a Glance
  • Hub vs Repeater: ports (N vs 2) + LEDs
  • Hub vs Bridge: Hub=L1 no filter; Bridge=L2 uses MAC
  • Hub vs Switch: Hub=broadcast all; Switch=send to specific port
  • Hub vs Router: Hub=L1; Router=L3 uses IP, separates broadcast domains
