SRTF — Shortest Remaining Time First

🟩 SRTF — Algorithm at a Glance
Also CalledPreemptive SJF — SJF with preemption enabled
Selection CriteriaRemaining Burst Time — smallest remaining BT among arrived processes
ModePreemptive — CPU can be snatched if a shorter process arrives
StarvationYes — long processes may starve indefinitely
RT vs WTRT ≠ WT — first CPU time may differ from total wait time
OptimalYes — minimum average WT among all algorithms (proven)
▶ DIAGRAM 2.5-A — SRTF vs SJF: How Preemption Works
🔵 SJF (Non-Preemptive)
Once a process gets CPU → runs to completion
New arrivals wait until current process finishes
Decision only at completion points
RT = WT always
🟩 SRTF (Preemptive)
Every time a new process arrives → check burst times
If new process has smaller remaining BT → preempt!
Decision at every arrival event
RT ≠ WT (can differ in preemptive)
⚠ Key Rule: In SRTF, compare REMAINING burst times (not original BT). The remaining time decreases as the process runs. Track this carefully!
SRTF checks at every new arrival: should I preempt the current running process?
💡 Tie-Breaking in SRTF: Same as SJF — if remaining BT is equal: (1) pick lower Arrival Time, (2) if AT also equal: pick lower Process ID. Apply at every decision point.
⚠️ Completion Time (CT) Tricky! In SRTF, a process may appear multiple times in Gantt chart. CT = the time at the right edge of its LAST Gantt block. Don't use intermediate values!

t=9: Only P3 remains. No competition → run P3 fully from 9 to 13. P3 COMPLETES at t=13. CT(P3) = 13.

SRTF — Algorithm Basics
Full nameShortest Remaining Time First = Preemptive SJF
Selection criterionSmallest REMAINING Burst Time among arrived processes
ModePreemptive — CPU can be snatched at any new arrival
When to preemptWhen a new process arrives with BT < remaining BT of current running process
Tie-break (same rem BT)1st: lower AT | 2nd: lower Process ID
Starvation?Yes — long processes may never get CPU if shorter ones keep arriving
⭐ Optimal theoremSRTF gives the MINIMUM average WT among ALL scheduling algorithms
RT vs WTRT ≠ WT in preemptive. RT = first CPU time − AT. WT = TAT − BT.

★
Star Points & Common Mistakes
★ CT in preemptiveRight edge of LAST block in Gantt chart (a process appears multiple times!)
★ RT formulaFirst CPU time − AT. RT ≠ WT in preemptive scheduling.
★ WT formulaTAT − BT = CT − AT − BT
★ Remaining BT trackingKeep subtracting from BT each time the process runs. Compare REMAINING, not original BT.
★ Only 1 process?No competition → run to completion directly. No need to check 1 unit at a time.
★ SRTF optimalMinimum avg WT among ALL algorithms — preemptive and non-preemptive alike.
★ Context switchingSRTF causes more context switches (overhead) — impractical in real OS but important for theory exams.
