FLOW CONTROL PROTOCOLS

Three Protocols at a Glance

Data Link Layer flow control protocols decide how many frames can be sent before waiting for acknowledgement. The three important exam protocols are Stop-and-Wait, Go-Back-N, and Selective Repeat.

Efficiency Formulas

Efficiency means useful transmission time divided by total time. For Stop-and-Wait, only one frame is sent per cycle, so efficiency is lowest. Sliding-window protocols multiply the base efficiency by the sender window size.

Stop-and-Wait

Only one frame is transmitted, then sender waits for ACK.

1 / (1 + 2x)

Go-Back-N

Sender can transmit a full sender window before waiting.

(2^k - 1) / (1 + 2x)

Selective Repeat

Sender window is half the sequence-number space.

2^(k-1) / (1 + 2x)
x = propagation delay / transmission time. Stop-and-Wait is least efficient because it sends only one frame in the whole cycle.
3

Retransmission Load

Retransmission tells us how many frames may need to be sent again when a frame or acknowledgement is lost. Stop-and-Wait resends only one frame. Go-Back-N may resend the whole window. Selective Repeat resends only the missing or corrupt frame.

⏪ What Gets Retransmitted?
Go-Back-N has the highest retransmission load; Selective Repeat avoids unnecessary retransmission.
4

ACK Types and Out-of-Order Handling

ACK style and receiver behavior are favorite theory questions. Stop-and-Wait uses simple acknowledgement. Go-Back-N uses cumulative acknowledgement and rejects out-of-order frames. Selective Repeat can use cumulative, independent ACK, and NAK; it accepts out-of-order frames.

ProtocolACK StyleOut-of-Order FramesImplementation
Stop-and-WaitSimple ACKNot applicableVery easy
Go-Back-NCumulative ACKNot acceptedModerate
Selective RepeatCumulative + independent ACK + NAKAccepted and bufferedMost complex
Why Selective Repeat is complex: it needs searching for missing frames and sorting/buffering out-of-order frames before delivery to the upper layer.

Stop-and-Wait
  • Sends 1 frame at a time

  • Least efficiency, easiest implementation

  • Sliding window protocol

  • Receiver window = 1
  • Uses cumulative ACK
  • Out-of-order frames rejected

  • Sliding window protocol

  • Out-of-order frames accepted

✅ Last-Minute Summary

  • Least efficiency: Stop-and-Wait.
  • Highest retransmission: Go-Back-N.
  • Most complex implementation: Selective Repeat.
  • Go-Back-N receiver window is always 1, so out-of-order frames are rejected.
  • Selective Repeat receiver window is 2^(k-1), so out-of-order frames are accepted and buffered.
  • Available sequence numbers for Stop-and-Wait = sender window + receiver window = 2.
