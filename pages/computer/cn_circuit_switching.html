What is Circuit Switching?

Circuit switching is a communication technique in which a dedicated physical path is established between the sender and receiver before any data is transmitted. This path is reserved exclusively for that communication session from start to finish.

Circuit switching was originally designed for telephone networks — the classic PSTN (Public Switched Telephone Network). Before OSI layers and TCP/IP models were used in computer networks, telephone networks relied on circuit switching to connect callers. When you pick up a landline and dial a number, you are using circuit switching.

📞 Origin: Circuit switching was designed for telephone networks — long before TCP/IP and OSI models existed. It was the backbone of all voice communication for decades.

Why Physical Layer? — No OSI Layer Stack Needed

In modern computer networks we think in layers: Application → Transport → Network → Data Link → Physical. Data gets divided into segments, packets, frames as it travels down the stack.

In circuit switching, none of this happens. You pick up the phone, establish the connection, and start talking. The voice flows as a continuous electrical signal directly through the dedicated physical path. There is no concept of dividing data into packets, adding headers, or routing decisions at intermediate nodes. The entire connection works at the Physical Layer — raw signals, raw wires, physical circuits.

Dedicated Path — The Core Concept

The most fundamental property of circuit switching is the dedicated path. Once T1 dials T5 and the connection is established through the telephone exchange, a complete physical circuit is created from T1 all the way to T5.

This path is dedicated — no other user can share this particular circuit while T1 and T5 are talking. The ports inside the telephone exchange that connect T1 to T5 are exclusively reserved. Even if T1 is silent for 5 minutes, the path remains reserved — no one else can use it.

The telephone exchange acts like a multiplexer — it has many incoming lines from many phones, and it creates different dedicated circuits for different pairs of callers simultaneously. But each circuit, once made, is dedicated to that one pair.

📌 Point-to-Point Illusion: Although T1 and T5 are not physically connected by a direct wire, circuit switching makes them feel as if they are connected point-to-point — because the entire path through the exchange is reserved and no one else can interrupt it.
🔴 Dedicated Path — Ports Reserved Inside the Exchange
The orange ports inside the exchange are reserved exclusively for T1↔T5. Gray ports are free for other connections. This is the meaning of "dedicated path."

No Headers — Why? The Reservation at Setup

In packet switching (computer networks), every packet carries a header containing the source IP, destination IP, source port, destination port, MAC addresses, etc. This is needed because each packet might travel a different path and intermediate devices (routers) need to know where to send it.

In circuit switching, none of this is needed. Here is why: at setup time, the connection is established and resources are reserved —

📜 What Gets Reserved at Setup Time
🔌
Port
Specific port inside the exchange is reserved exclusively for this session
📡
Bandwidth
A fixed portion of the link's bandwidth is reserved for this circuit
💾
Buffer
Memory buffers at the exchange are allocated to hold this session's data
📏
Route
The complete path from source to destination is fixed at setup and never changes
Once all resources are reserved at setup, the data just flows. No one needs to read a header to figure out where to send it — the path is already locked in.
📌 Why no headers? Because the intermediate nodes (telephone exchange) don't need to make any routing decisions. The route is fixed. The port is fixed. The bandwidth is fixed. The data just travels through the pre-made channel. Contrast this with packet switching where each router reads the IP header and decides where to forward the packet — that's why headers are essential there.

Why Efficiency is Low — The Idle Reservation Problem

The biggest weakness of circuit switching is low efficiency. Once resources are reserved, they stay reserved for the entire duration of the connection — even when no data is being transmitted.

Imagine you are on a phone call. You say "wait for 2 minutes, I'll finish watching my show." You put the receiver on the side but do not hang up. For those 2 minutes — no voice is being transmitted, but the dedicated circuit remains reserved. The bandwidth, port, and buffer are locked up. No one else can use them. This is pure waste.

⚠ During the silence period, bandwidth is locked to T1–T5 but zero data is sent. Other users who need bandwidth cannot get it. This is why efficiency is low.
Circuit switching wastes bandwidth during idle periods. Packet switching solves this — bandwidth is shared and unused when no packet is being sent.

Total Time Formula — GATE Numericals

For GATE and UGC NET numerical questions, you may be asked: "How much total time does circuit switching take to send a message?" The total time has four components:

🎯 Total Time in Circuit Switching
Ttotal = Tsetup + Ttransmission + Tpropagation + Tteardown
Ttransmission = Message Size (bits) ÷ Bandwidth (bps)  |  Tpropagation = Distance ÷ Speed of Signal
♿
Phase 1: Setup Time (Tsetup)
Given in problem (not calculated from formula)
The time required to establish the dedicated connection — dialing the number, the exchange reserving ports, bandwidth, and buffers, and the call being connected. This is the reservation phase. In GATE problems, setup time is always given to you as a value; you cannot derive it from other parameters.
📡
Phase 2: Transmission Time (Ttransmission)
Ttx = Message Size (bits) ÷ Bandwidth (bps)
The time for the sender to push all the bits of the message onto the wire. This is purely how fast the sender can transmit the data — determined by the link bandwidth.
Example: Message = 1,000 bits, Bandwidth = 100 bps → Ttx = 1,000 ÷ 100 = 10 seconds
🌊
Phase 3: Propagation Delay (Tpropagation)
Tprop = Distance ÷ Speed of Signal
The time for the signal to physically travel from source to destination through the medium. Speed of signal in copper wire ≈ 2×108 m/s; in fibre ≈ 3×108 m/s (speed of light).
Example: Distance = 2,000 km, Speed = 2×108 m/s → Tprop = 2×106 ÷ 2×108 = 0.01 seconds
🚫
Phase 4: Teardown Time (Tteardown)
Given in problem (not calculated from formula)
The time required to release all reserved resources — freeing the ports, bandwidth, and buffers so others can use them. This happens after the session ends (hanging up the phone). Like setup time, teardown time is given in the problem; you don't derive it.
🎯 GATE Numerical Tip: In circuit switching problems, you will always be given setup time and teardown time directly. You calculate transmission time (message ÷ bandwidth) and propagation delay (distance ÷ speed) yourself. There is no store-and-forward queuing delay at intermediate nodes — the circuit is dedicated, so packets don't wait.
✎ Worked Example — Total Time Calculation
Problem: T1 sends a 500 KB message to T5 over a circuit-switched network.
GIVEN VALUES
Message = 500 KB = 500 × 8 × 1024 = 4,096,000 bits
Bandwidth = 1 Mbps = 106 bps
Distance = 3,000 km = 3×106 m
Signal Speed = 2×108 m/s
Setup Time = 0.2 s (given)
Teardown Time = 0.1 s (given)
SOLUTION
Ttx = 4,096,000 ÷ 106 = 4.096 s
Tprop = 3×106 ÷ 2×108 = 0.015 s
Ttotal = 0.2 + 4.096 + 0.015 + 0.1
= 4.411 seconds
📌 Note: There is NO queuing delay or processing delay at intermediate nodes because the circuit is dedicated — data flows straight through without being stored or buffered at each hop.
