Multiple Access Protocols Classification
⏳ Pending Merge Section 16 — MAC Layer 🎯 IBPS SO Focus 📅 2026-07-11

Multiple Access Protocols (all three branches — Random-access, Controlled-access, Channelization) fall under the Data Link Layer of the OSI model — more specifically, its MAC (Medium Access Control) sublayer.

Why Data Link Layer, and not something else?

Recall that in the OSI model, the Data Link Layer is split into two sublayers:

  1. LLC (Logical Link Control) sublayer — handles flow control, error control, and framing (interacts with the Network layer above it).
  2. MAC (Medium Access Control) sublayer — handles who gets to use the shared physical medium and when. This is exactly the job of ALOHA, CSMA, CSMA/CD, CSMA/CA, Reservation, Polling, Token Passing, FDMA, TDMA, CDMA.

Since these protocols exist purely to decide which station transmits over the shared channel at a given moment (avoiding or managing collisions), they naturally belong to the sublayer that manages access to the medium — the MAC sublayer, which sits within the Data Link Layer.

Quick memory hook

  • Physical Layer → deals with raw bits, signals, cables (below this).
  • Data Link Layer → MAC sublayer → decides who transmits when (this is where Multiple Access Protocols live).
  • Data Link Layer → LLC sublayer → handles framing, flow control, error control.
  • Network Layer → routing (above this).

MCQ trap points

⚠️
  1. "CSMA/CD belongs to which layer?" → Data Link Layer (MAC sublayer) — one of the most repeated direct questions.
  2. Don't confuse this with TDMA/FDMA/CDMA in mobile communication contexts — in cellular networks these are sometimes discussed as Physical Layer techniques (since they deal with actual frequency/time/code allocation of the radio spectrum). But in the standard computer networks syllabus (Forouzan-based, which IBPS SO follows), all multiple access protocols are taught under Data Link Layer → MAC sublayer — stick with this for your exam.
  3. MAC address (physical/hardware address) is also handled at this same sublayer — a nice connecting fact if a question links MAC addressing with multiple access protocols.

This is the top-level classification of Multiple Access Protocols in Computer Networks — the "family tree" that ALOHA, CSMA, CSMA/CD, and CSMA/CA (which we've already covered) belong to. For IBPS SO, this classification itself is a very common one-line MCQ ("Which of these is NOT a random-access protocol?" type questions), so let's map out all three branches clearly.

At the top: Multiple-access protocols — these decide how multiple stations share one common communication channel. They split into 3 main categories:

1. Random-Access Protocols

Core idea: No station has a fixed turn or priority — any station can try to transmit whenever it wants, and the protocol just manages what happens when collisions occur.

  • ALOHA — transmit whenever ready, no sensing at all (Pure ALOHA / Slotted ALOHA — we covered this).
  • CSMA — "sense before you send" to reduce collision chances (we covered this).
  • CSMA/CD — CSMA + Collision Detection — used in wired Ethernet; detects a collision mid-transmission and stops immediately (we covered this).
  • CSMA/CA — CSMA + Collision Avoidance — used in WiFi; avoids collisions upfront since wireless can't detect them (we covered this).
💡
Exam memory hook: notice the natural evolution — ALOHA (no sensing) → CSMA (sensing added) → CSMA/CD (sensing + detection, wired) → CSMA/CA (sensing + avoidance, wireless). Each step fixes a limitation of the previous one.

2. Controlled-Access Protocols

Core idea: Stations don't transmit freely — they need permission from either another station or a central coordinator before sending. This avoids collisions almost entirely, but adds coordination overhead.

  • Reservation: Before sending data, a station must reserve the channel in advance for a specific time slot. Time is divided into intervals; in each interval, there's a "reservation frame" where stations state their intent to send, then the actual data transmission happens in the reserved order. No collision because everyone knows their assigned slot in advance.
  • Polling: Works only if there's one central controller (primary station) and the rest are secondary stations. The primary station asks ("polls") each secondary station in turn — "do you have data to send?" Only the station being polled can transmit at that moment. Simple and collision-free, but the whole system depends on the primary station — if it fails, communication stops (single point of failure, a common MCQ trap).
  • Token Passing: Stations are arranged in a logical ring. A special small frame called a token circulates around the ring. Only the station currently holding the token is allowed to transmit; after finishing, it passes the token to the next station. No collisions since only the token-holder can send. Used in Token Ring and Token Bus networks — another classic exam pairing.

3. Channelization Protocols

Core idea: Instead of letting stations compete for the whole channel, the available bandwidth is divided among stations, so each one gets its own guaranteed share — no competition, no collision at all.

  • FDMA (Frequency Division Multiple Access): The available bandwidth is divided into frequency bands, and each station gets a fixed frequency band to use for the entire duration. Like giving each station its own separate radio frequency — they can all transmit simultaneously without interfering, since they're on different frequencies.
  • TDMA (Time Division Multiple Access): The available bandwidth (a single frequency) is divided into time slots, and each station gets a fixed time slot in a repeating cycle to transmit. Everyone shares the same frequency but takes turns based on time.
  • CDMA (Code Division Multiple Access): All stations transmit simultaneously over the same frequency and at the same time, but each is assigned a unique code. The receiver uses this code to separate and extract the intended signal from the mixed transmission. This is the most advanced of the three and used in mobile/cellular communication.

Quick comparison table (exam gold)

Category Basis Examples Collision?
Random-access No fixed turn, contention-based ALOHA, CSMA, CSMA/CD, CSMA/CA Possible
Controlled-access Needs permission/coordination Reservation, Polling, Token Passing None
Channelization Bandwidth is pre-divided FDMA, TDMA, CDMA None

MCQ trap points

⚠️
  1. "Which protocol has the highest chance of collision?" → Random-access category (especially Pure ALOHA).
  2. "Which protocol uses a token?" → Token Passing (controlled-access) — don't confuse with time slots in TDMA.
  3. "Which divides the channel by frequency vs time vs code?" → FDMA / TDMA / CDMA respectively — a very frequently repeated direct-match question.
  4. Polling requires a primary/central station — single point of failure; Token Passing has no central controller (fully distributed) — this distinction is a favorite trap.
  5. CSMA family (all 4 protocols) falls under Random-access, not Controlled-access — students sometimes misplace CSMA/CD under channelization by mistake.

ALOHA Protocols — Pure & Slotted
✅ Merged Section 16 — MAC Layer 🎯 GATE Important 📅 2026-07-10

The Basic Problem

Imagine a shared communication channel (like a radio or satellite link) that multiple stations want to use to send data. If two stations transmit at the same time, their signals collide, and both frames get corrupted. ALOHA protocols are early solutions to this collision problem.

ALOHA was developed at the University of Hawaii in the 1970s. It has two versions: Pure ALOHA and Slotted ALOHA.

1. Pure ALOHA

The rule is simple: whenever a station has data ready, it transmits immediately — no waiting, no coordination with other stations.

The problem: since any station can transmit at any moment, two frames can collide even if their transmissions only partially overlap — even a small overlap corrupts the entire frame.

Vulnerable period (shown in the diagram above):
If a frame's transmission time is Tt, then any other frame that starts transmitting within Tt before the current frame starts, or Tt after it starts, will collide with it.

Vulnerable Period = 2 × Tt

Throughput formula (important for the exam):

S = G · e^(-2G) S = throughput (successfully transmitted frames) G = offered load (total attempted transmissions, including retransmissions)

Maximum throughput: Maximizing this formula (setting dS/dG = 0) gives a peak at G = 0.5:

S_max = 1/(2e) ≈ 0.184 (18.4%)
💡
Key takeaway: Pure ALOHA's maximum efficiency is only 18.4% — meaning only 18.4% of channel capacity is ever used for successful transmission; the rest is wasted on collisions.

2. Slotted ALOHA

This is an improvement — time is divided into fixed-size slots, where each slot equals one frame transmission time (Tt).

Rule: a station can only start transmitting at the beginning of a slot — not at an arbitrary moment.

Benefit: now a collision only happens if two stations choose the same slot. Partial overlaps are impossible, since every transmission starts exactly at a slot boundary.

Vulnerable period:

Vulnerable Period = 1 × Tt (Half of Pure ALOHA's — which is why performance doubles.)

Throughput formula:

S = G · e^(-G)

Maximum throughput: peaks at G = 1:

S_max = 1/e ≈ 0.368 (36.8%)

MCQ Trap Points to Watch For

⚠️
  1. Efficiency doubles in Slotted ALOHA — this follows directly from the vulnerable period being halved, not a coincidence.
  2. The formulas involve e (Euler's number ≈ 2.718) — questions often directly ask for "S_max = 1/2e" or "1/e".
  3. The value of G at which maximum throughput occurs — G=0.5 for Pure ALOHA, G=1 for Slotted ALOHA — is a common direct MCQ.
  4. Slotted ALOHA requires a synchronized clock across all stations — a frequent true/false statement.
  5. Both are random access protocols, predecessors to the CSMA family; historically used in satellite communication and early Ethernet.
  6. Numerical questions: if G is given (e.g., G=1 for Pure ALOHA), you may be asked to calculate S — just substitute into the formula.

CSMA (Carrier Sense Multiple Access)
⏳ Pending Merge Section 16 — MAC Layer 🎯 GATE Important 📅 2026-07-10

What is CSMA?

Pure ALOHA and Slotted ALOHA had one big flaw — stations transmit blindly, without checking whether the channel is already busy. CSMA (Carrier Sense Multiple Access) fixes this with one simple rule:

"Listen before you talk." Before transmitting, a station first senses the channel. If it's busy, it doesn't transmit; if it's idle, only then does it try to send.

This reduces collisions significantly compared to ALOHA — but here's the exam-important catch: CSMA reduces collisions, it does NOT eliminate them.

Why collisions still happen — Vulnerable Time

Even though a station "senses" the channel, it can still get it wrong. Here's why (matches the diagram above and your slide's example):

Suppose station A transmits a frame at time t1. This signal doesn't reach every station instantly — it takes real physical time to travel along the medium (electrical/optical signal has finite speed). This travel time is called propagation time (Tp).

Now imagine station B, sitting somewhere in the middle, senses the channel before A's signal has reached it. B finds the channel idle (because A's signal simply hasn't arrived yet) and starts transmitting too. Collision.

So the vulnerable time in CSMA = propagation time (Tp) — much smaller than ALOHA's vulnerable time (which was based on frame transmission time Tt), which is exactly why CSMA performs far better than ALOHA.

💡
Key insight for MCQs:
  • Vulnerable time depends on distance between stations and signal propagation speed, not on frame size.
  • The farther two stations are apart, the longer it takes for one's signal to reach the other, so the higher the chance of collision.
  • A station right next to the sender has almost no vulnerable window; a station far away has a bigger window.

Numeric example (like your slide with distances 3, 5, 2): If A sends at 7:00, and it takes 3 minutes (time units) for the signal to reach B, then B could sense idle and transmit anywhere in that 3-minute gap — collision risk. Same logic for C and D based on their distances from A. The total propagation time to the farthest station (D) defines the full vulnerable window for the whole network.

Persistence methods — what to do after sensing?

CSMA tells you to sense before sending, but it doesn't specify what a station should do if the channel is busy, or exactly how to act once it's idle. That's what the three persistence methods solve.

1. 1-Persistent method
  • Station continuously senses the channel.
  • The moment it finds the channel idle, it transmits immediately (with probability 1 — hence "1-persistent").
  • Drawback: if multiple stations are waiting for the channel to become free, all of them jump in the instant it's idle → very high chance of collision.
  • Used in: classic Ethernet (CSMA/CD).
2. Non-persistent method
  • Station senses the channel.
  • If idle → transmit immediately.
  • If busy → don't keep sensing continuously; instead wait a random amount of time, then sense again.
  • Advantage: fewer collisions (since stations back off by different random times, unlikely to retry simultaneously).
  • Drawback: channel may sit idle even though a station has data ready, because it's still in its random wait — this wastes channel time, lowering efficiency.
3. P-persistent method (best of both worlds)
  • Used in slotted channels (time divided into slots, each slot ≥ max propagation time).
  • When channel is found idle:
    • With probability p → transmit.
    • With probability q = 1−p → wait for the next slot and sense again.
  • If it senses busy at any point → treats it like a collision and runs the backoff procedure.
  • Advantage: balances 1-persistent's "everyone jumps in" problem and non-persistent's "wasted idle time" problem — reduces collisions and keeps efficiency high.

Quick comparison table (exam-ready)

Method When channel busy When channel idle Collision chance Efficiency
1-persistent Keep sensing continuously Transmit immediately (p=1) Highest Can be low under load
Non-persistent Wait random time, sense again Transmit immediately Lower Reduced (channel idles unnecessarily)
P-persistent Backoff like a collision occurred Transmit with probability p, else wait a slot Balanced Best overall

MCQ trap points

⚠️
  1. CSMA reduces but never eliminates collisions — a very common true/false trap.
  2. Vulnerable time in CSMA = propagation delay (Tp), not frame transmission time. Don't confuse this with ALOHA's Tt-based vulnerable period.
  3. 1-persistent has the highest collision probability among the three — many students wrongly assume "1" means "best."
  4. Non-persistent avoids the 1-persistent collision problem but wastes bandwidth due to unnecessary idle time.
  5. P-persistent requires the channel to be divided into time slots — this detail ("slotted channel") is often tested directly.
  6. CSMA is the base for CSMA/CD (Collision Detection — used in wired Ethernet) and CSMA/CA (Collision Avoidance — used in wireless/WiFi) — natural next topics, and IBPS SO loves asking "CD is for wired, CA is for wireless."

CSMA/CD (Collision Detection)
⏳ Pending Merge Section 17 — CSMA/CD 🎯 IBPS SO Focus 📅 2026-07-11

For the IBPS SO (IT Officer) exam, you do not need the deep mathematical treatment of CSMA/CD that is required for GATE. You should focus on:

  • What it is
  • Why it is needed
  • How it works (step by step)
  • Ethernet association
  • Difference from CSMA/CA
  • Common MCQ points

Let's learn it from scratch.

Before CSMA/CD, understand the problem

Imagine there is only one microphone in a classroom.

There are 5 students.

Student A Student B Student C Student D Student E 🎤

Everyone wants to speak using the same microphone.

If only one student speaks,

No problem.

But suppose A says

"Sir..."
and exactly at the same time, B also says
"Sir..."

Both voices mix. Nobody understands anything. This is called a Collision.

The same thing happens in Ethernet.

What is CSMA/CD?

CSMA/CD stands for:

  • C → Carrier
  • S → Sense
  • M → Multiple
  • A → Access
  • C → Collision
  • D → Detection

Let's understand each word.

Carrier Sense (CS)

Carrier means

The communication channel (wire/cable).

Sense means

Listen before speaking.

Think about a phone conversation. Before speaking, you check "Is the other person already speaking?"

Similarly, a computer first checks whether the network cable is busy.

  • If busy → Wait.
  • If free → Transmit.

This is called Carrier Sense.

Multiple Access (MA)

Multiple Access means many computers share the same communication medium.

Example:

PC1 PC2 PC3 PC4 │ │ Shared Ethernet Cable

Everyone uses the same cable.

Collision Detection (CD)

Even after listening, sometimes two computers may decide to transmit at almost exactly the same moment.

Example:

Time → A : Listen ✓ Send B : Listen ✓ Send

Both thought "The cable is free." Both start together. Collision occurs.

CSMA/CD detects this collision while transmitting.

Why can collision happen even after listening?

This is the biggest doubt students have.

Imagine two people standing at opposite ends of a long hallway.

A ---------------------------- B

Both look. Nobody is speaking. Both start speaking together.

They couldn't know the other had just started because the sound hadn't reached them yet.

The same thing happens in Ethernet. Signals take a small amount of time to travel through the cable. So both computers may think "The line is free." This delay causes collisions.

How CSMA/CD Works

There are 6 easy steps.

Step 1: Computer has data

Suppose Computer A wants to send data.

Computer A Data Ready
Step 2: Listen to the channel

It first checks "Is someone already transmitting?"

Cable Busy? No

If busy → Wait. If free → Go ahead.

Step 3: Start transmitting

Now A starts sending.

A ======= Data =======>
Step 4: Keep listening while transmitting

This is very important. Unlike ALOHA, CSMA/CD does not stop listening after transmission starts. It keeps monitoring the cable.

Think of it like talking while also listening to see if someone interrupts you.

Step 5: Collision detected

Suppose Computer B also starts sending.

A --------> X B -------->

Signals collide. Computer A notices "What I'm receiving is different from what I'm transmitting." So it concludes

Collision!

Step 6: Stop transmission immediately

The computers stop sending data. Then they send a Jam Signal.

What is a Jam Signal?

Think of a referee blowing a whistle.

Whistle !! Everyone Stop !!

Similarly, a jam signal tells every computer:

Collision happened. Stop transmitting.

After sending the jam signal, everyone stops.

What happens after the collision?

Now both computers wait for a random amount of time.

Example: Computer A waits 2 ms, Computer B waits 5 ms.

Since they wait for different times, they are less likely to collide again.

Binary Exponential Backoff

This is one of the favorite IBPS interview/MCQ topics.

Suppose collision happens once. Random waiting is chosen from 0 or 1.

If another collision happens, choose from 0 1 2 3.

Next collision: 0 to 7.

Next: 0 to 15.

Next: 0 to 31.

The waiting range doubles after every collision, reducing the chance of repeated collisions. This method is called Binary Exponential Backoff.

Complete Flow

Data Ready ↓ Listen ↓ Busy? ↓ YES ---------------------> Wait ↓ NO ↓ Transmit ↓ Collision? ↓ NO ---------------------> Success ↓ YES ↓ Send Jam Signal ↓ Stop Transmission ↓ Random Backoff ↓ Try Again

Real-Life Analogy

Imagine two friends using walkie-talkies.

Friend A: "Hello..."
Friend B: "Hello..."

Both speak together. Nobody understands. Now they both hear the interference. They stop. One waits 2 seconds. The other waits 5 seconds. Then one speaks. Perfect. That's CSMA/CD.

Where is CSMA/CD used?

Used in:

  • Traditional shared Ethernet
  • Bus topology
  • Hub-based Ethernet
  • Half-duplex Ethernet

Not used in:

  • Modern switched Ethernet (full-duplex), because each device has its own dedicated link, so collisions do not occur.

Why is CSMA/CD not used in Wi-Fi?

In wireless networks, a device cannot reliably transmit and detect a collision at the same time because its own transmitted signal overwhelms any incoming signal. So Wi-Fi tries to avoid collisions instead of detecting them. It uses CSMA/CA (Collision Avoidance) rather than CSMA/CD (Collision Detection).

CSMA/CD vs ALOHA

Feature ALOHA CSMA/CD
Listen before sending? ❌ No ✅ Yes
Collision detection during transmission? ❌ No ✅ Yes
Efficiency Lower Higher
Used in Early radio networks Ethernet LANs

CSMA/CD vs CSMA/CA

CSMA/CD CSMA/CA
Collision Detection Collision Avoidance
Wired Ethernet Wireless (Wi-Fi)
Detects collision after it occurs Tries to prevent collision before it occurs
IEEE 802.3 IEEE 802.11

IBPS SO Important Points (Memorize)

  • CSMA/CD = Carrier Sense Multiple Access with Collision Detection
  • Operates at the MAC (Media Access Control) sublayer of the Data Link Layer.
  • Used in shared, half-duplex Ethernet (IEEE 802.3).
  • Listen before transmitting (Carrier Sense).
  • If the channel is busy, wait.
  • If a collision occurs, detect it, send a Jam Signal, stop transmission, wait using Binary Exponential Backoff, and retransmit.
  • Modern switched full-duplex Ethernet does not use CSMA/CD, because collisions are effectively eliminated.
💡
A simple memory trick:
Remember the sequence: L → T → D → J → B → R
  • L = Listen
  • T = Transmit
  • D = Detect Collision
  • J = Jam Signal
  • B = Binary Exponential Backoff
  • R = Retransmit
If you can recall this sequence, you'll be able to answer almost every CSMA/CD question that appears in the IBPS SO exam.

Binary Exponential Backoff (BEB)
⏳ Pending Merge Section 17 — Backoff Algorithm 🎯 IBPS SO Focus 📅 2026-07-11

Excellent! Binary Exponential Backoff (BEB) is one of the most confusing topics for beginners, but once you understand the logic behind it, it's actually very simple.

First, why do we need Binary Exponential Backoff?

Imagine two people, Rahul and Aman, are using walkie-talkies.

Time = 0 seconds

Rahul starts speaking. At the exact same time, Aman also starts speaking.

Rahul: Hello.... Aman : Hello.... COLLISION 💥

Neither message is understood. Now both stop. The question is:

When should they try again?

Problem 1: Retry immediately

Suppose both think, "Let's try again immediately."

Time 0 Rahul ---> Hello Aman ----> Hello COLLISION 💥

Again, collision.

Problem 2: Wait exactly 2 seconds

Now suppose both decide, "Let's wait exactly 2 seconds."

Time 2 sec Rahul ---> Hello Aman ----> Hello COLLISION AGAIN 💥

Still a collision.

So what should we do?

Instead of choosing the same waiting time, let each computer choose a random waiting time.

Example:

Rahul waits 1 second Aman waits 4 seconds

Now:

1 sec Rahul ---> Hello ✅ 4 sec Aman ---> Hello ✅

No collision. This is the basic idea behind Backoff.

What does "Binary Exponential Backoff" mean?

Let's understand each word.

Binary

Means powers of 2.

2^0 = 1 2^1 = 2 2^2 = 4 2^3 = 8 2^4 = 16
Exponential

Means the range doubles after every collision.

Example:

1 2 4 8 16 32

Notice every value doubles.

Backoff

Wait for some time before trying again.

How does it actually work?

Suppose Rahul and Aman collide. This is the first collision.

First Collision

The computer randomly chooses 0 or 1. Only these two values. Think of it as:

Random = {0,1}

Suppose Rahul gets 0, Aman gets 1. Then:

Rahul sends immediately. Aman waits one slot.

Success.

What if they collide again?

Now collision count becomes 2. Now the random number is chosen from 0, 1, 2, 3.

Why? Because 2² = 4 values. Range: 0 to 3.

Example: Rahul chooses 3, Aman chooses 1. Aman sends first. Rahul waits. No collision.

Third Collision

Suppose they are very unlucky. Another collision happens. Now choose from 0, 1, 2, 3, 4, 5, 6, 7.

Why? Because 2³ = 8 values. Range: 0 to 7. Now the chance that both pick the same number is much smaller.

Fourth Collision

Choose from 0 to 15. Because 2⁴ = 16 values.

General Formula

If n = Number of collisions, then the random number is selected from:

0 to (2^n − 1)

This is the most important formula for exams.

Examples:

Collision Number (n) Random Range
1 0 – 1
2 0 – 3
3 0 – 7
4 0 – 15
5 0 – 31
6 0 – 63

Notice how the range doubles after each collision.

Why does doubling help?

Imagine only two choices: 0, 1. Rahul and Aman have a 50% chance of choosing the same value.

Now increase the choices: 0, 1, 2, 3. The probability of both choosing the same value becomes lower.

Now increase further: 0, 1, 2, 3, 4, 5, 6, 7. Now it's even less likely. So increasing the waiting range reduces repeated collisions.

Real-Life Analogy

Imagine two friends trying to enter a narrow door. Initially, they both try together. 💥 Collision.

The teacher says:

"One of you wait randomly between 0–1 seconds."

If they still collide, the teacher says:

"Now wait randomly between 0–3 seconds."

Still collide? The teacher says:

"Now wait randomly between 0–7 seconds."

Eventually, one goes first, then the other. This is exactly how Binary Exponential Backoff works.

Visualization

Collision #1 Choose 0 1 ↓ Collision #2 Choose 0 1 2 3 ↓ Collision #3 Choose 0 1 2 3 4 5 6 7 ↓ Collision #4 Choose 0 ... 15 ↓ Collision #5 Choose 0 ... 31

The waiting choices keep expanding.

How does Ethernet measure the waiting time?

The random number is not in seconds. It is in slot times.

For example, suppose the random number selected is 3.

Wait: 3 × Slot Time Then retransmit.

A slot time is a fixed unit of time defined by the Ethernet standard. For the IBPS SO exam, you usually don't need to memorize its exact duration—just know that computers wait for a random number of slot times before retransmitting.

IBPS SO Exam Points

  • Binary Exponential Backoff is used after a collision in CSMA/CD.
  • After every collision, the waiting range doubles.
  • The random number is chosen from: 0 to (2^n - 1) where n is the number of collisions.
  • The purpose is to reduce the probability of repeated collisions.
  • It is a key mechanism used in Ethernet (IEEE 802.3).
💡
Easy Memory Trick:
Think of a child repeatedly bumping into someone in a hallway.
  • First time: "Wait a little." (0-1)
  • Second time: "Wait a bit longer." (0-3)
  • Third time: "Wait even longer." (0-7)
  • Fourth time: "Wait even longer." (0-15)
  • Fifth time: "Wait even longer." (0-31)
Just remember the pattern: 2, 4, 8, 16, 32... (powers of 2). That's why it's called Binary Exponential Backoff:
  • Binary → based on powers of 2.
  • Exponential → the range doubles after each collision.
  • Backoff → wait before trying again.

CSMA/CA (Collision Avoidance)
⏳ Pending Merge Section 17 — CSMA/CA 🎯 IBPS SO Focus 📅 2026-07-11

Why does WiFi need Collision Avoidance?

In a wireless network, all devices in the same area share one RF channel. The core problem is interference: when multiple devices transmit at the same time, their RF signals interfere with each other, and this problem grows as more devices join the network. When two signals overlap, they cancel each other out and the receiver gets corrupted data.

This is essentially the same collision problem you saw in wired Ethernet with a hub — all devices connected to a hub share one collision domain, so every host must operate in half-duplex mode, meaning it can't send and receive at the same time.

The fix is the same idea humans use at a table — only one person talks while everyone else listens. That's exactly what CSMA/CA enforces for wireless devices.

Breaking down the name — CSMA/CA

  • Carrier Sense (CS): a device may transmit only if the channel is free, so every device continuously monitors the RF spectrum to sense whether someone else is transmitting.
  • Multiple Access (MA): the wireless medium is shared by all devices using the same frequency, so they must all follow a common set of access rules.
  • Collision Avoidance (CA): devices use a backoff mechanism to reduce the chance that two of them transmit at the same time.

So CSMA/CA's whole goal is: make sure only one device transmits at a time — by avoiding collisions upfront, not by fixing them after they happen.

The single biggest exam point: WHY CA and not CD for WiFi?

This is the #1 conceptual question IBPS SO can ask, so lock this in:

  • In wired networks, a device can detect a collision because it can listen for electrical signals on the cable while it's transmitting — if it hears an unexpected signal, it knows a collision happened.
  • In wireless networks, detecting a collision is impossible, because a device can't transmit and listen on the same frequency at the same time — its own signal is far stronger than any incoming one, drowning out anything else.

That's the whole reason two different algorithms exist:

  • CSMA/CA (wireless) tries to prevent collisions before they happen by having devices check the channel is clear before transmitting.
  • CSMA/CD (wired) lets collisions happen, then detects and resolves them by stopping and resending the data.
💡
One-line memory trick: CD = "collision already happened, catch it." CA = "stop it from happening at all."

How CSMA/CA actually works (matches the flowchart below)

  1. A device wants to send data → it senses the channel.
  2. Channel free? → it transmits.
  3. Channel busy? → it waits a random amount of time called the backoff period, then checks the channel again.
  4. Once the backoff timer hits zero and the channel is clear, it sends the data.
  5. After receiving the transmission, the receiver sends back an acknowledgment (ACK) to confirm the delivery was successful.

There's also an optional RTS/CTS mechanism (Request to Send / Clear to Send): a device first asks the access point for permission before sending; if granted, it transmits, which ensures no other device transmits at the same time. This is a nice extra layer of avoidance for busier networks.

Two ways a device "senses" the channel

  1. Physical Carrier Sense (Clear Channel Assessment - CCA): the device listens to the channel when it's not transmitting; if it detects any signal, it assumes the channel is busy.
  2. Virtual Carrier Sense (Network Allocation Vector - NAV): every transmitted frame carries a duration value telling other devices how long the channel will stay occupied; devices that hear this store it in a NAV timer and wait until it counts down to zero before trying to transmit.

Why two methods? Even with NAV, if multiple devices detect the channel is free at exactly the same moment, they may all try to transmit simultaneously, causing a collision anyway. That's exactly why the random backoff timer exists as a third safety layer.

The Random Backoff Timer (very likely MCQ detail)

When a client wants to transmit, it picks a random number of timeslots between 0 and 31 as its backoff timer, which reduces the chance that multiple clients transmit at the moment. Only once both the NAV timer and the backoff timer reach zero does the device get to compete for the channel.

Frame Acknowledgements — how does a sender know it worked?

Since wireless can't detect collisions in real time, it needs a different way to confirm success. When a client receives a unicast frame, it must send back a unicast acknowledgment (ACK) to confirm successful delivery. The 802.11 standard requires an ACK for every frame, except in 802.11n and WMM (802.11e), where a single ACK can confirm multiple frames sent in a burst.

If no ACK arrives, the sender assumes the frame was lost (possibly due to a collision) and retransmits after another random backoff.

Quick comparison: CSMA/CD vs CSMA/CA

Feature CSMA/CD CSMA/CA
Used in Wired Ethernet (old, bus topology) Wireless (802.11 WiFi)
Approach Detects collision after it happens Avoids collision before it happens
Can it detect collisions? Yes (listens while transmitting) No (can't listen and transmit at once)
Confirmation method No ACK — relies on absence of collision Uses ACK for every frame
Extra mechanism Jam signal + backoff NAV + random backoff (+ optional RTS/CTS)
Duplex mode Half-duplex over shared bus Half-duplex over shared RF channel

MCQ trap points

⚠️
  1. CD = wired, CA = wireless — the single most repeated fact; know it cold.
  2. The core reason CA exists: a wireless radio can't listen and transmit on the same frequency simultaneously — its own transmission overpowers any incoming signal.
  3. Backoff timer range in WiFi: 0 to 31 slots — a specific number that examiners like to test.
  4. CSMA/CA uses ACK-based confirmation, unlike CSMA/CD which infers success from "no collision detected."
  5. RTS/CTS is optional, not mandatory — used to further reduce collisions, especially useful for the "hidden node problem" (a classic follow-up topic).
  6. NAV (Network Allocation Vector) is the "virtual" sensing method based on a duration field in the frame; CCA is the "physical" sensing method based on actually listening to the RF channel — these two terms are often swapped in tricky MCQs, so keep them straight.

Controlled-Access Protocols

Controlled-Access Protocols — overview

Unlike random-access (ALOHA/CSMA family) where stations transmit freely and risk collisions, in controlled access, stations must consult each other or a controller first — a station cannot send data unless it has been authorized. This trades off a bit of speed/simplicity for guaranteed collision-free transmission. There are 3 methods: Reservation, Polling, Token Passing.

1. Reservation

Core idea: Before actual data is sent, stations first reserve their turn in advance.

How the timeline is structured (see diagram below):

  • Reservation Interval: a fixed-length time period, divided into M slots if there are M stations — each station gets exactly one dedicated slot.
  • Data Transmission Period: a variable-length period that follows, where actual data frames are sent.

Step by step:

  1. During the reservation interval, if a station has data to send, it marks its own slot with a 1-bit signal (like raising a hand). If it has nothing to send, its slot stays 0.
  2. No other station is allowed to touch someone else's slot — this keeps the reservation phase itself collision-free.
  3. Once all M slots have been checked, every station now knows exactly which stations want to transmit and in what order.
  4. The stations that reserved a slot then transmit their actual data frames one by one, in the agreed order — no collisions, because everyone already agreed on the sequence.
  5. After the data transmission period ends, a fresh reservation interval begins again, and the whole cycle repeats.

Exam-friendly example (matches your slide): If Station 1, Station 3, and Station 4 mark their slots as 1 during the reservation interval, then only these three transmit data in that round, in slot order (1 → 3 → 4), while Stations 2 and 5 just wait for the next cycle.

Key point for MCQs: Reservation is best suited when the number of stations is fixed and known in advance, since the reservation interval is divided exactly by M.

2. Polling

Core idea: One device is the primary station (controller), and all others are secondary stations. All communication passes through the primary — even if the actual destination is another secondary device, data goes via the primary first. The primary decides who gets to use the channel and when.

There are two functions the primary uses: Select and Poll.

a) Select — when primary wants to SEND data
  • The primary has data to send to a secondary, but doesn't know if that secondary is ready to receive.
  • So the primary creates and sends a SEL (select) frame, which includes the address of the intended secondary — essentially saying "hey, are you ready?"
  • The secondary responds with an ACK confirming it's ready.
  • The primary then sends the Data, and the secondary responds with another ACK confirming receipt.
  • (Sequence: SEL → ACK → Data → ACK)
b) Poll — when primary wants to RECEIVE data
  • The primary asks ("polls") each secondary in turn: "do you have anything to send?"
  • If that secondary has nothing, it replies with a NAK (negative acknowledgment) — so the primary moves to poll the next secondary.
  • If that secondary does have data, it responds directly with the data frame itself (instead of just saying yes) — the primary then reads it and sends back an ACK to confirm receipt.
  • (Sequence: Poll → NAK → Poll (next) → Data → ACK)
💡
Key point for MCQs:
  • Polling requires a primary/central controller — this is a single point of failure: if the primary goes down, the whole network communication stops. This is one of the most repeated trap questions.
  • SEL is for the primary sending data; Poll is for the primary receiving data — these two are commonly swapped in exam options, so keep the direction clear: SEL = "I have something for you," Poll = "do you have something for me?"

3. Token Passing

Core idea: Stations are arranged in a logical ring (physically they might be in a bus, star, or ring topology — but logically they form a ring). A special small packet called a token circulates around this ring, and only the station currently holding the token is allowed to transmit.

How it works:

  1. Every station has a predecessor (the station before it in the ring) and a successor (the station after it).
  2. The token keeps moving from one station to its successor, continuously circulating around the ring.
  3. A station that has data to send waits until it receives the token from its predecessor.
  4. Once it has the token, it transmits its data. After finishing, it releases the token to its successor.
  5. If a station receives the token but has nothing to send, it just immediately passes the token forward without any delay.
  6. If a station has already passed the token in the current round, it must wait for the next round to get another chance — this ensures fairness (no station can hog multiple turns).

Token Management — 3 important aspects (good for MCQs):

  • Time control: how long a station is allowed to hold the token must be limited, so no single station can monopolize the channel.
  • Token monitoring: the network needs to detect if the token gets lost (e.g., the station holding it crashes) — otherwise the whole ring stalls forever.
  • Priority assignment: the system can assign priorities so that critical/important data gets transmitted before less important data.

Handling failures: if the station holding the token fails, the token must be regenerated or reassigned, or the entire network communication halts — this is a key robustness concern examiners like to test ("what happens if token is lost?").

Physical implementations (from your slide's diagrams): Token passing can be implemented over different physical topologies — physical ring, dual ring (for redundancy), bus (Token Bus), or even a star topology using a hub (Token Ring via hub) — even though logically it's always a ring.

Real-world examples: Token Ring and Token Bus networks are the classic examples — a very common exam pairing question.

Quick comparison table

Method Who controls access Collision risk Weak point (exam favorite)
Reservation Pre-agreed slot ownership None Needs fixed M stations known in advance
Polling Primary/central station None Primary failure = single point of failure
Token Passing Token itself (distributed) None Token loss halts the network; needs monitoring

MCQ trap points

⚠️
  1. Polling has a single point of failure (primary); Token Passing is decentralized (no central controller) — the token itself carries the authority. This contrast is asked very often.
  2. In Reservation, if there are M stations, the reservation interval has exactly M slots — a small but frequently tested numeric detail.
  3. SEL vs Poll in Polling — SEL is primary-initiated sending; Poll is primary-initiated receiving. Don't mix them up.
  4. A secondary responding with NAK in polling means "I have nothing to send," not an error — this phrasing sometimes confuses students who assume NAK always means failure.
  5. Token Ring / Token Bus are real IEEE-standard implementations of token passing — good to remember as concrete examples if asked "give an example of token passing."
  6. All three (Reservation, Polling, Token Passing) guarantee zero collisions, unlike the random-access family — a solid one-line differentiator if a question asks to compare categories.

Channelization Protocols

Channelization Protocols — the big picture

Unlike Random-access (compete for channel) and Controlled-access (take turns via permission), Channelization protocols divide the available bandwidth itself among stations in advance — so each station gets a guaranteed share, and there's no competition, no collision at all. There are 3 methods: FDMA, TDMA, CDMA.

1. FDMA (Frequency-Division Multiple Access)

Core idea: The available bandwidth is split into separate frequency bands, and each station is permanently allocated one band.

  • Each band belongs to one station all the time — meaning that station can transmit whenever it wants, but only within its own frequency range.
  • Since every station uses a different frequency, they can all transmit simultaneously without interfering with each other.

Real-world example (from your slide): FM radio is a perfect real-life FDMA example — Radio Mirchi (98.3), Red FM (93.5), Big FM (92.7) all broadcast at the same time, but each on its own fixed frequency. Your radio tunes into one specific frequency to pick just that station's signal, ignoring all others.

Key point for MCQs: In FDMA, guard bands (small unused frequency gaps) are usually kept between adjacent bands to prevent stations from interfering with each other's edges — a good add-on fact if the question probes deeper.

2. TDMA (Time-Division Multiple Access)

Core idea: Instead of splitting frequency, TDMA gives every station the full bandwidth, but only for a fixed time slot in a repeating cycle.

  • All stations use the same frequency, but take turns based on time.
  • Each station transmits during its own slot, then waits for its next turn to come around again.

How it's different from FDMA (see the diagram below): In FDMA, a station's band is exclusively theirs permanently — like having your own dedicated lane on a highway all day. In TDMA, everyone shares the same lane (frequency), but each gets a scheduled turn to drive through it — like a shared conference room booked in time blocks.

Real-world example: Old GSM mobile networks (2G) used TDMA — multiple calls could share the same frequency channel by taking turns in tiny time slots, fast enough that it feels seamless to the user.

Key point for MCQs: TDMA requires precise synchronization among stations — everyone must agree exactly on when their time slot starts and ends, otherwise slots overlap and data corrupts.

3. CDMA (Code-Division Multiple Access)

Core idea: This is the most advanced of the three. In CDMA, all stations transmit at the same time, over the same frequency — no separation by time or frequency at all. Instead, each station is assigned a unique code, and the receiver uses that code to extract only the intended signal from the mixed jumble of all transmissions.

Analogy that makes this click: Imagine a room full of people, all speaking different languages simultaneously (that's the "same time, same frequency" part). If you only understand, say, French, your brain automatically picks out the French conversation and ignores the rest — your "code" is your language. That's exactly how CDMA receivers work — they're tuned to a specific code and extract only that signal, treating everything else as background noise.

How it differs from the other two (this is a very common direct exam question):

  • Differs from FDMA: in FDMA, only a fraction of the bandwidth belongs to one station at a time; in CDMA, one single channel occupies the entire bandwidth of the link.
  • Differs from TDMA: in TDMA, stations take turns over time; in CDMA, all stations send data simultaneously — there's no timesharing at all.

Real-world example (from your slide): CDMA was the technology behind mobile operators like Tata DOCOMO/Tata Indicom in India (pre-4G era) — CDMA phones used unique codes assigned by the network to separate each user's call from everyone else's, even though they shared the same frequency and time.

Key point for MCQs: CDMA is generally considered more spectrum-efficient and secure than FDMA/TDMA (since without knowing the exact code, it's very hard to eavesdrop or decode a specific transmission) — this is a common "advantage of CDMA" question.

Quick comparison table (exam gold, ties the whole Channelization branch together)

Protocol What's divided How stations are separated Real-world example
FDMA Frequency Each station = own frequency band, all the time FM Radio
TDMA Time Each station = full frequency, own time slot 2G GSM calls
CDMA Neither (uses codes) Unique code per station, same time + frequency CDMA mobile networks (Tata DOCOMO)

MCQ trap points

⚠️
  1. FDMA → frequency bands; TDMA → time slots; CDMA → unique codes — the most repeated direct-match question across this entire chapter.
  2. CDMA is the only one of the three where all stations transmit truly simultaneously without any division of frequency or time — a favorite "which statement is true about CDMA" trap.
  3. TDMA needs tight time synchronization; FDMA needs **guard bands** between frequencies — both are common "limitation of X" questions.
  4. All three Channelization protocols are collision-free by design (same category-level property as Controlled-access, but achieved differently) — good if a question compares all 3 top-level categories together.
  5. Don't confuse CDMA (Channelization, no timesharing) with CSMA/CA (Random-access, collision avoidance) — similar-sounding acronyms, completely different mechanisms; this exact confusion is a classic distractor in MCQs.

Flow Control, Error Control & Stop-and-Wait ARQ
⏳ Pending Merge Topic 23 — Stop & Wait 🎯 IBPS SO Focus 📅 2026-07-15

Flow Control vs Error Control — the big picture

Both are Data Link Layer (and Transport Layer too) responsibilities, but they solve different problems:

Flow Control Error Control
Problem it solves Receiver getting overwhelmed Data getting corrupted/lost
Mechanism Buffer + feedback to sender to slow down Sequence numbers + ACK + retransmission (ARQ)
Flow Control
  • Purpose: the receiver has limited processing speed and memory — if the sender floods it with data faster than it can process, data gets dropped.
  • Buffer mechanism: the receiver stores incoming data temporarily in a buffer until it processes it.
  • Handling overflow: when the buffer is nearing capacity, the receiver signals the sender to pause or slow down — this is exactly the kind of feedback loop you see in TCP's sliding window.
Error Control
  • Purpose: to detect and correct errors introduced during transmission (noise, interference, etc.)
  • Error detection: identifies frames that are lost, out-of-order, or corrupted.
  • Retransmission (ARQ): if an error is found, the receiver essentially triggers the sender to resend the affected frame — this whole family of techniques is called Automatic Repeat Request (ARQ).

For noisy channels, there are 3 ARQ protocols (increasing order of complexity/efficiency):

  1. Stop-and-Wait ARQ
  2. Go-Back-N ARQ
  3. Selective Repeat ARQ

We'll cover Stop-and-Wait in full detail here (most fundamental, and the one your slides focus on), since Go-Back-N and Selective Repeat build directly on it.

Stop-and-Wait ARQ — how it works

Core idea: the sender sends one frame, then stops and waits for an acknowledgment (ACK) before sending the next one. Simple, but not very efficient (we'll see why via the math below).

Error handling
  • Corrupted frames: the receiver detects corruption (via checksum/CRC) and simply discards the frame silently — it doesn't send any ACK or NAK, it just stays quiet.
  • Lost frames: since the receiver stays silent for both a lost frame AND a corrupted frame, the sender can't tell the difference just from silence — so it needs another mechanism: a timer.
Sequence numbers — why only 0 and 1?
  • Every frame is given a sequence number so the receiver can uniquely identify it.
  • In Stop-and-Wait, since only one frame is ever "in flight" at a time, you only need 2 sequence numbers (0 and 1), cycling back and forth — this is a very commonly tested fact ("minimum sequence numbers needed for stop-and-wait = 2").
  • These numbers let the receiver detect duplicates — e.g., if an ACK gets lost and the sender resends the same frame, the receiver can recognize "I've already received sequence number 0" and discard the duplicate instead of processing it twice.
Acknowledgment + Timer mechanism (walking through the diagram below)
  1. Sender sends Frame 0, keeps a copy, and starts a timer.
  2. If ACK 1 arrives in time (ACK numbering indicates "I'm ready for the next frame," i.e., 1 comes after 0) → sender stops the timer and sends the next frame (Frame 1).
  3. If Frame 1 gets lost in transit → receiver never gets it, so it stays silent → sender's timer expires (time-out) → sender automatically resends Frame 1.
  4. This time it arrives fine → receiver sends ACK 0 (meaning "I got frame 1, now expecting frame 0 next") → sender proceeds to send Frame 0 again (next in the 0/1 cycle).
  5. If the ACK itself gets lost (not the frame) → the sender's timer still expires (since it heard nothing back) → it resends the same frame → the receiver, using sequence numbers, recognizes this as a duplicate and discards it, but still re-sends the ACK so the sender can move on.

This is exactly the beauty of combining sequence numbers + timeout + ACK — it handles lost frames, lost ACKs, AND corrupted frames, all with just 2 sequence numbers.

The Delay Components (crucial for numerical questions)

To calculate how efficient Stop-and-Wait actually is, you need to understand these delay types:

1. Transmission Delay (Tt)

Time to push all the bits of a packet onto the transmission line, one by one.

Tt = Packet Length (L) / Bandwidth (B)
2. Propagation Delay (Tp)

Time for a single bit to physically travel from sender to receiver over the medium (depends on distance and signal speed, not on data size).

Tp = Distance / Propagation Speed
3. Processing Delay

Time the destination host takes to receive a packet, strip the header, run error detection, and hand it off — usually treated as negligible/zero in standard numericals unless explicitly given.

4. Queuing Delay

Time a packet waits in a router's input/output queue before being processed — also usually treated as zero in standard Stop-and-Wait numericals.

Measuring Total Performance — building the formula step by step

Full formula (theoretical):

Total Time = Tt(data) + Tp(data) + Delay_que + Delay_pro + Tt(ack) + Tp(ack)

Simplifications typically made in exam problems:

Step 1 — drop queuing and processing delay (usually assumed 0):

Total Time = Tt(data) + Tp(data) + Tt(ack) + Tp(ack)

Step 2 — ACK size is tiny compared to data, so its transmission time is negligible:

Total Time = Tt(data) + Tp(data) + Tp(ack)

Step 3 — propagation delay for data and ACK is essentially the same (same physical link, same distance):

Total Time = Tt(data) + 2 × Tp

This final formula — Total Time = Tt + 2Tp — is THE most important formula for Stop-and-Wait numericals.

Important terminology: the term 2 × Tp (the round trip for the frame to reach the receiver and the ACK to come back) is also commonly called the Round Trip Time (RTT).

Efficiency (η) — the key exam formula

Look at the diagram above: out of the entire cycle time (Tt + 2Tp), only Tt was actual "useful" time spent sending data. The rest (2Tp) was just waiting — time during which the sender could have sent more frames but didn't (because Stop-and-Wait insists on waiting for ACK first).

Efficiency (η) = Useful Time / Total Cycle Time = Tt / (Tt + 2Tp)

Simplified form (divide numerator and denominator by Tt, and let a = Tp/Tt):

η = 1 / (1 + 2a)

Throughput / Effective Bandwidth:

Throughput = η × B (where B is the actual bandwidth/transmission rate of the link)

Worked numerical example (typical IBPS SO style)

📝
Question: A sender sends 1000-bit frames over a 1 Mbps link, with propagation delay 20 ms one-way. Find efficiency and throughput.

Solution:
  • Tt = L/B = 1000 / (1×10⁶) = 1 ms
  • Tp = 20 ms
  • a = Tp/Tt = 20/1 = 20
  • η = 1/(1 + 2×20) = 1/41 ≈ 2.44%
  • Throughput = η × B = 0.0244 × 1 Mbps ≈ 24.4 kbps

Notice how badly Stop-and-Wait performs when propagation delay is large relative to transmission time — this is exactly why Go-Back-N and Selective Repeat ARQ exist (they let the sender keep transmitting multiple frames without waiting for each individual ACK, dramatically improving efficiency in high-latency links). That's the natural next topic.

MCQ trap points

⚠️
  1. Minimum sequence numbers needed for Stop-and-Wait ARQ = 2 (just 0 and 1) — extremely common one-liner.
  2. Corrupted frames are handled by silence + timeout, NOT by sending a NAK — Stop-and-Wait doesn't use NAKs in the standard version.
  3. Total cycle time = Tt + 2Tp, and 2Tp = RTT — both the formula and the terminology are frequently tested together.
  4. η = 1/(1+2a) where a = Tp/Tt — a classic "calculate efficiency given these values" numerical.
  5. Throughput = η × Bandwidth — don't confuse throughput (actual useful data rate) with bandwidth (the link's raw capacity).
  6. As Tp increases relative to Tt (i.e., a increases), efficiency drops — this inverse relationship is often tested as a conceptual (non-numeric) question too.
  7. Stop-and-Wait is simple but **inefficient for high-bandwidth-delay-product links** (satellite links, long-distance fiber) — this is the setup line for why Go-Back-N/Selective Repeat exist.
