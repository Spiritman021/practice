GO-BACK-N ARQ

A sliding-window protocol where the sender can send multiple frames, the receiver accepts only the next expected frame, ACKs are cumulative, and a lost frame makes the sender go back and retransmit from that frame onward. Now includes a clear receiver-side explanation and sequence-number rules.

GATEUGC NETSender Window 2^m - 1Receiver Window 1
1

Why It Is a Sliding Window Protocol

Go-Back-N is called a sliding window protocol. In Stop-and-Wait, the window size was only 1, so the sender could send just one frame and then had to stop. In Go-Back-N, the sender window is bigger, so the sender can send multiple frames continuously before all acknowledgements come back.

The sender window keeps sliding forward as acknowledgements arrive. If the current sender window contains frames 0, 1, and 2, the sender can send these frames. When frame 0 is acknowledged, 0 goes out of the window and frame 3 enters. This movement of the window is the main idea of a sliding window.

But the receiver side is still strict: the receiver window size is 1. That means the receiver accepts only the next expected frame. If it is waiting for frame 1, then frame 2 or frame 3 will not be accepted before frame 1.

🪑 Sliding Sender Window
The window moves forward as earlier frames are acknowledged.
2

Window Size and Sequence Number Formula

If m bits are used to represent sequence numbers, then total sequence numbers are 2^m. Here m does not mean the sequence number itself. It means how many bits are used to represent the sequence number.

For example, if m = 2, two bits can represent 00, 01, 10, 11, which correspond to sequence numbers 0, 1, 2, 3. So total sequence numbers are 2^2 = 4. If m = 3, the range becomes 0 to 7. If m = 4, the range becomes 0 to 15.

In Go-Back-N, the sender window size must be 2^m - 1, and the receiver window size is always 1.

3

Why Sender Window Is Not 2^m

The sender window must be less than the total sequence-number space. If it becomes equal to 2^m, old retransmitted frames can look like new frames after sequence numbers wrap around. This is the biggest theory point in the lecture.

Safe case: If m = 2, total sequence numbers are 0, 1, 2, 3. Correct sender window size is 2^2 - 1 = 3, so sender sends 0, 1, 2. Suppose all acknowledgements are lost. Receiver has accepted 0, 1, 2 and is now waiting for 3. When sender times out and resends 0, receiver will discard it because it wants 3. No confusion occurs.

Wrong case: If sender window size is wrongly taken as 2^m = 4, sender sends 0, 1, 2, 3 together. Receiver accepts all and wraps around; now it expects 0 again. If all ACKs are lost and sender retransmits old 0, receiver may accept old 0 as if it were a new 0. That is the problem.

⚠ Ambiguity When Window = 2^m
Keeping one sequence number outside the sender window prevents old/new frame confusion.
Exam trap: For Go-Back-N, do not write sender window = 2^m. The answer is 2^m - 1.
4

Cumulative Acknowledgement

Go-Back-N uses cumulative acknowledgement. Cumulative means combined. Instead of depending on every individual ACK, one later ACK can confirm multiple earlier frames.

For example, receiver accepts frame 0 and sends ACK 1. Then it accepts frame 1 and sends ACK 2. Suppose ACK 2 is lost. Then receiver accepts frame 2 and sends ACK 3. When ACK 3 reaches the sender, the sender understands that frames 0, 1, and 2 have been accepted. So ACK 3 also covers the lost ACK 2.

Just like Stop-and-Wait, ACK number means the next expected frame. ACK 4 means "send frame 4 now", and it also implies that frames up to 3 were accepted.

📨 Cumulative ACK Example
Cumulative ACK reduces the need to receive every individual ACK.
5

Lost Frame and Out-of-Order Discard

The receiver window is 1, so it accepts only the next expected frame. This creates the main limitation of Go-Back-N: out-of-order packets are not accepted.

Take the teacher's m = 3 example. Sequence numbers are 0 to 7 and sender window size is 7, so sender may send 0, 1, 2, 3, 4, 5, 6. Suppose frame 0 reaches and receiver now wants 1. But frame 1 is lost. If frames 2 and 3 reach after that, receiver still says, "I want 1." It discards 2 and 3 even though they reached correctly.

After timeout, sender goes back to frame 1 and retransmits from there: 1, 2, 3 and so on. This forced retransmission of later frames is exactly why the protocol is named Go-Back-N.

⏪ Go Back From Missing Frame
This is why the protocol is called Go-Back-N: go back to the missing frame and resend from there.
Key point: Go-Back-N does not accept out-of-order packets. Selective Repeat handles that differently.
6

Receiver-Side Working

On the receiver side of Go-Back-N, the most important rule is WR = 1. That means the receiver has space for only one expected frame. It accepts a frame only if it is exactly the next expected sequence number.

If the expected frame arrives correctly, receiver accepts it, sends an ACK, and updates the expected sequence number. If an out-of-order frame arrives, receiver discards it and repeats the ACK for the last correctly received frame, indirectly saying, "I am still waiting for the missing one."

📥 Receiver Window = 1: Accept Only Expected Frame
Out-of-order frames are not buffered in Go-Back-N.
7

Efficiency, Throughput and Sequence-Number Rule

Let N be the sender window size. In the ideal case, when processing delay, queuing delay and ACK transmission delay are ignored, Go-Back-N efficiency is:

Ideal Efficiency

Useful transmission over total round-trip cycle.

N / (1 + 2a)

a value

Ratio of propagation delay to transmission delay.

a = Tp / Tt

Throughput

Actual data rate achieved on a channel of bandwidth B.

Efficiency x B

When extra delays are included, use the general idea: Efficiency = N x useful time / total time. Useful time is the data-frame transmission time. Total time includes data-frame transmission, round-trip propagation, processing delay, queuing delay and ACK transmission delay.

Symbols: Tp = propagation delay, Tt = transmission delay of data frame, B = bandwidth, N = sender window size.

For sequence numbers, the safe relation is Ws + WR <= ASN, where Ws is sender window size, WR is receiver window size and ASN is available sequence numbers. In Go-Back-N, WR = 1, so Ws + 1 <= ASN. Therefore, if sender window is N, minimum sequence numbers required are N + 1.

🔄 Why One Extra Sequence Number Is Needed
For sender window N, Go-Back-N needs at least N + 1 sequence numbers.
8

Teacher-Style Complete Summary

Step 1: Decide the number of bits m used for sequence numbering. If m = 2, sequence numbers are 0, 1, 2, 3. If m = 3, sequence numbers are 0 to 7.

Step 2: Sender window size becomes 2^m - 1. Receiver window size remains 1. So with m = 2, sender can send 3 frames at once, but receiver accepts only one expected frame at a time.

Step 3: ACKs are cumulative. ACK 3 means frames before 3 are accepted. ACK 4 means frames before 4 are accepted.

Step 4: If one frame is missing, receiver discards later out-of-order frames. Sender later goes back and resends from the missing frame onward.

Full forms: ARQ = Automatic Repeat reQuest. ACK = Acknowledgement. Go-Back-N means go back to the missing/unacknowledged frame and retransmit N frames from there onward.
