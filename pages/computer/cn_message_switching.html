Message Switching

The bridge between Circuit Switching and Packet Switching. Introduced the Store & Forward concept with hop-by-hop delivery in the 1960s — paving the way for modern packet networks.

🕐 Introduced ~1960 💾 Store & Forward 🚶 Hop-by-Hop 🚫 No Reservation ⏱ High Delay 📦 Predecessor of Packet Switching
1

What is Message Switching? — The Switching Evolution

Switching techniques evolved in a clear chronological order. Circuit Switching came first (telephone era), then Message Switching (~1960s), and finally Packet Switching (modern internet). Each solved limitations of the previous one.

Circuit Switching (Before 1960)

Dedicated physical path established before any data transfer. Designed for telephone networks. Setup → Transfer → Teardown. Bandwidth reserved even when idle → efficiency less.

Message Switching (~1960s) ← This Topic

No dedicated path. Messages sent hop-by-hop. Each intermediate switch stores the entire message, processes it, then forwards to the next hop. Store & Forward on hard disk originally.

Packet Switching (Modern Internet)

Message divided into small packets. Multiple packets flow in parallel across the network. Pipelining increases throughput, reduces delay. Packet Switching = Modified Message Switching.

🔔 Key Insight: Message Switching directly evolved into Packet Switching. The only difference: instead of sending one big message hop-by-hop, we divide it into small packets and send them parallelly. Message Switching is therefore the predecessor (parent) of Packet Switching.
2

Store & Forward — The Core Mechanism

Unlike circuit switching where a path is pre-established, in message switching A simply transmits its entire message towards D. There is no end-to-end connection setup. Each intermediate node (switch/router) receives the full message, stores it, processes it (finds the next route), and then forwards it to the next hop.

📩 Message Switching — Hop-by-Hop Store & Forward Delivery (A to D)
A sends the entire message to Switch 1. Switch 1 stores it (originally HDD, now RAM), processes (routing), then forwards to Switch 2 — until D receives the complete message. No path reserved between A and D.

Evolution of storage at intermediate nodes:

💾 Hard Disk (HDD) Original 1960s
Very slow — High delay
→
⚡ RAM (Modern) Much faster
Lower delay — Used today
💡 Why RAM replaced HDD: When messages were stored on hard disk at each intermediate node, the delay was enormous (HDD is slow). As technology evolved, RAM became the intermediate storage, dramatically reducing the store-and-forward delay at each hop.

Key Properties of Message Switching

Connection
Connectionless
No setup phase. A sends directly. No reserved path A→D.
Data Unit
Message (full)
The entire message is sent as ONE unit, not divided into packets.
Delivery
Hop-by-Hop
Message jumps from one intermediate node to the next, one at a time.
Storage
Store & Forward
Full message stored at each intermediate node before forwarding.
Delay
High Delay
Every hop introduces store + process + forward delay. Delay > Circuit Switching.
Efficiency
Better than CS
No resource reservation → bandwidth used only when message is transmitted.
Routing
Dynamic per hop
Each intermediate node independently decides where to forward the message next.
Headers
Headers Needed
Each message carries full address info (source + destination) as a header.
✅ No Reservation Advantage: Unlike circuit switching (where bandwidth, CPU, and buffer were reserved even during silence), message switching uses resources only when data is actually being transmitted. This makes the network more efficient overall.

Three-Way Comparison — Circuit vs Message vs Packet

⚡ Circuit Switching

  • Dedicated path reserved
  • Setup → Transfer → Teardown
  • No store & forward
  • Contiguous (stream) flow
  • No headers needed
  • Delay less
  • Efficiency less (idle wastes BW)
  • Best for voice calls

📩 Message Switching

  • No path reservation
  • Full message stored at each hop
  • Store & Forward (HDD→RAM)
  • Hop-by-hop delivery
  • Headers required
  • Delay high
  • Efficiency better than CS
  • ~1960s, predecessor of PS

📦 Packet Switching

  • No path reservation
  • Message divided into packets
  • Store & Forward per packet
  • Pipelining / parallelism
  • Headers on each packet
  • Delay less than MS
  • Efficiency highest
  • Internet (IP, TCP/UDP)
📄 Message vs Packet — Why Packets Are Better (Pipelining)
Message switching sends one large message hop-by-hop (sequential, slow). Packet switching divides it into small packets that flow in parallel — achieving pipelining and dramatically lower end-to-end delay.
5

Advantages & Disadvantages

✓ Advantages
  • No reservation — bandwidth, CPU, buffer not pre-allocated
  • Better efficiency than circuit switching
  • Dynamic routing — each hop chooses the best available path
  • Network usage — whole network used efficiently
  • No call setup time — no setup phase delay
  • Traffic management — switches can buffer during congestion
✗ Disadvantages
  • Very high delay — every hop stores the FULL message
  • Large storage needed — each switch must store complete message
  • Not suitable for real-time — voice/video calls cannot use this
  • No pipelining — one message at a time per route
  • Header overhead — full source + destination address per message
  • Superseded by Packet Switching — rarely used in modern networks

🎯 Most Likely Exam Points

  • Correct chronological order: Circuit Switching → Message Switching (~1960) → Packet Switching
  • Message Switching uses Store & Forward — full message stored at each intermediate node before forwarding
  • Originally messages stored on Hard Disk (HDD); later replaced by RAM (faster, less delay)
  • No connection setup in MS — connectionless (unlike circuit switching)
  • Delay is MORE in MS than CS because of store-and-process at every hop
  • Efficiency is BETTER than CS because no bandwidth/CPU/buffer is reserved
  • MS has NO pipelining; PS has pipelining — key advantage of PS over MS
  • MS is the predecessor / parent of PS — PS = MS + divide into packets + parallel transmission
  • Headers are required in both MS and PS (carry address per data unit)
  • MS is NOT suitable for real-time applications (voice/video) due to high delay
  • In MS, each intermediate node decides the route independently — dynamic routing
  • Three-way comparison table (CS vs MS vs PS) is a high-frequency exam question
