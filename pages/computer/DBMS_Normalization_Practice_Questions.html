Question 2: Count Candidate Keys

Given relation R(A, B, C, D, E, F, G, H) and these functional dependencies, find the number of candidate keys.

CH → GA → BCB → CFHE → AF → EG
3

Shortcut: Missing From RHS

List all attributes appearing on the right-hand side of FDs: A, B, C, F, G, H. Attribute D is missing from RHS.

If an attribute never appears on RHS, no other attribute can derive it. So D must be present in every candidate key.
TrialClosure resultConclusion
D+Only gives D.Not a key
AD+A gives B,C; B gives F,H; F gives E,G. So AD gives all attributes.Candidate key
4

Generate Remaining Keys

After finding AD, use determinant substitution. Since E → A, replace A with E. Since F → E, replace E with F. Since B → F, replace F with B.

Candidate Key Substitution Cycle

A starts the closure and D is compulsory.
ED

FD
F → E, so F can replace E.
BD
B → F, so B can replace F.
A → B, B → F, F → E, and E → A form a cycle. With D fixed, four candidate keys appear.

FDF derives E and G; E derives A; A and B chain gives all remaining attributes.
BDB derives C,F,H; F derives E,G; E derives A.
Correct answer for Question 2: 4 candidate keys = AD, ED, FD, BD
