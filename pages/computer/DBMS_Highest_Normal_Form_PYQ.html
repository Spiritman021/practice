Highest Normal Form
Solved Question

A step-by-step normalization question: find candidate keys, prime/non-prime attributes, then check BCNF, 3NF, 2NF, and finally 1NF.

Candidate KeysPrime Attributes BCNF to 1NFPartial Dependency
1

Question Data

Relation R(A, B, C, D, E, F) has these functional dependencies. Find the highest normal form.

AB → CC → DEE → FF → A
For highest-normal-form questions, start from the highest form: BCNF, then 3NF, then 2NF, and finally 1NF.
2

Step 1: Candidate Keys

Attribute B does not appear on any right-hand side, so every candidate key must contain B.

Check BCNF

BCNF condition: left-hand side of every FD must be a candidate key or super key.

FDLHS candidate/super key?BCNF status
AB → CYes, AB is candidate keyValid

Since even one invalid FD is enough, the relation is not in BCNF.
4

Check 3NF

3NF condition: for every FD, LHS is candidate/super key OR RHS is prime attribute.

FD3NF check3NF status
AB → CLHS AB is candidate key.Valid
C → DLHS C is not key and RHS D is non-prime.Invalid
C → ERHS E is prime.Valid
E → FRHS F is prime.Valid
F → ARHS A is prime.Valid
Because C → D fails, the relation is not in 3NF.
5

Check 2NF

2NF fails if a proper subset of any candidate key determines a non-prime attribute.

DependencyPartial dependency test2NF status
C → DC is a proper subset of candidate key CB, and D is non-prime.Partial dependency exists
Since partial dependency exists, the relation is not in 2NF.
6

Final Answer

Highest normal form = 1NF, assuming the relation is already atomic/first normal form.
If an exam option set includes BCNF, 3NF, 2NF, and 1NF, choose 1NF. If 1NF is not given and “None of these” is given, then choose according to the exact assumption in the question.
